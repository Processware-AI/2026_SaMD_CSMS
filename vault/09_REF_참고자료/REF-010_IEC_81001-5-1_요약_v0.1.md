## 1. 요약

IEC 81001-5-1 은 **health software**(SaMD 포함) 제조업자가 제품 라이프사이클 전반에 걸쳐 **사이버보안 활동**을 통합하는 방법을 규정하는 국제 표준. IEC 62304(의료기기 SW 라이프사이클)와 짝을 이루며, 보안을 SW 개발 프로세스에 내재화한다.

**핵심 철학**: Security by Design / Secure Product Lifecycle / Total Product Life Cycle(TPLC)

## 2. 표준 구조 (상위 Clause)

| Clause | 제목 | 요지 |
|---|---|---|
| 1 | Scope | Health software 의 security activities 정의 |
| 3 | Terms | 용어 (Asset, Vulnerability, Exploit, Threat 등) |
| 4 | General requirements | QMS 통합, 프로세스 요구사항 |
| 5 | Software development process | 개발 라이프사이클 내 보안 활동 |
| 6 | Software maintenance process | 유지보수 단계 보안 활동 |
| 7 | Security risk management process | 보안 위험 관리 (ISO 14971 확장) |
| 8 | Software configuration management process | 형상관리 — SBOM 관리 포함 |
| 9 | Software problem resolution process | 문제 해결 (취약점·침해 대응) |

## 3. 본 표준(SaMD-CSMS)과의 대응

| IEC 81001-5-1 Clause | 본 표준 조항 | 대응 Req |
|---|---|---|
| Clause 5 (Development process) | 제14조 개발 및 검증 활동 | MDCS-R-141~144 |
| Clause 6 (Maintenance) | 제10조 유지 관리, 제15조 레거시 관리 | MDCS-R-101~103, R-151~154 |
| Clause 7 (Security risk mgmt) | 제13조 위험 관리 활동 | MDCS-R-131~136 |

| Clause 9 (Problem resolution) | 제17~19조 침해대응, 제20~22조 취약점 관리 | MDCS-R-171~224 |

## 4. 핵심 활동 (요지)

### 4.1 Secure SDLC
- 보안 요구사항 정의 → 위협 모델링 → 보안 설계 → 보안 구현(보안 코딩) → 보안 검증(SAST/DAST/펜테스트) → 보안 릴리스

### 4.2 보안 위험 관리
- ISO 14971(의료기기 위험관리) + 보안 위험 = Safety-Security 융합
- Threat-Vulnerability-Impact 평가
- 잔여 위험 수용 프로세스

### 4.3 SBOM 관리 및 취약점 대응
- 컴포넌트 명세서 유지
- CVE/NVD 모니터링
- 취약점 발견 시 영향 평가 → 패치 → 배포 → 고객 통보

### 4.4 Post-market surveillance
- 시판 후 보안 모니터링
- 침해사고 대응 절차
- 고객(의료서비스제공자)과의 정보 공유
