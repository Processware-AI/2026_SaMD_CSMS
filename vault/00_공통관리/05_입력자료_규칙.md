# 입력자료(_inputs) 규칙

> 하네스 에이전트가 각 표준을 분석·설계할 때 참조하는 **외부 입력물**을 배치하는 규약. 표준 편입 전에 사용자가 미리 채워둠.

## 1. 위치

### 표준별 입력 (주)
```
vault/02_표준/{표준코드}/_inputs/
```
각 표준에 고유한 원문·법규·해설서·As-Is 문서를 투하.

### 전사 공통 입력 (선택)
```
vault/_inputs_common/
```
조직 공통 자산(기존 정책·조직도·용어집 외부본·회사 미션 등). 모든 표준 분석 시 참조.

## 2. 하위 폴더 구조 (권장)

```
_inputs/
├── 01_표준원문/         ← ISO/IEC/KS 원본 PDF·TXT (라이선스 보유 시)
├── 02_법규/             ← 국내 법령 PDF·MD (law.go.kr 발췌)
├── 03_해설서/           ← 인증기관·심사원 교재·산업협회 가이드
├── 04_AsIs/             ← 고객사 기존 정책·SOP·절차서
├── 05_산업가이드/       ← 업종별 best practice (NIST·ENISA·식약처 등)
└── README.md            ← 입력물 목록·출처·라이선스 상태 요약
```

> 서브폴더가 없어도 동작. 단, 혼재 시 에이전트가 분류 정확도 낮을 수 있음.

## 3. 에이전트 동작 규약

**모든 분석 에이전트는 _inputs/ 선행 스캔을 의무화**:

1. 표준별 `_inputs/` 존재 확인 → 없으면 `mkdir` + README stub 생성 후 사용자에게 경고
2. 있으면 전수 스캔 (PDF·MD·TXT 모두 읽기)
3. 각 파일을 카테고리로 분류 후 요약
4. 분석 결과에 근거로 인용 (source_citation 필수)
5. 입력물에 없는 사실 추정 시 `[_inputs 미제공 — LLM 추정]` 꼬리표

## 4. source_citation 스키마

모든 요구사항/산출물 본문에 근거를 명시:

```yaml
source_citation:
  - type: standard_original    # standard_original | law | guide | as_is | industry | llm_inference | web
    file: "_inputs/01_표준원문/ISO9001_2015.pdf"
    locator: "Section 7.5.3 p.18"
    retrieved_at: "2026-04-17"
    license: "ISO copyright — 내부 참조 only, 원문 인용 금지"
    paraphrase_only: true
  - type: law
    file: "_inputs/02_법규/개인정보보호법.md"
    locator: "제29조"
    retrieved_at: "2026-04-17"
    license: "공공저작물"
    paraphrase_only: false
```

본문 마크다운에서는 축약 가능: `(출처: ISO9001 §7.5.3)` 하되, 상응 frontmatter 또는 각주에 전체 스키마 보존.

## 5. 출처 우선순위

에이전트가 상충 정보를 받았을 때 신뢰도 순위:

```
1. 국내 법규 (법적 강제)
2. 라이선스 보유 표준 원문 (_inputs/01_표준원문/)
3. 공식 해설서·인증기관 가이드 (_inputs/03_해설서/)
4. 산업 가이드 (NIST, ENISA 등)
5. 고객사 As-Is (_inputs/04_AsIs/)  ← 존중하되 표준 대비 gap 지적
6. LLM 내부 지식
7. Web fetch (가장 마지막, 출처 URL 필수 기록)
```

## 6. 저작권 가드레일 (CRITICAL)

| 입력물 유형 | 가능 | 금지 |
|---|---|---|
| ISO/IEC 원문 | 요지 paraphrase, 조항번호·제목 인용 | **원문 문구 직접 복사** (침해) |
| KS 표준 | 동일 (라이선스 조건 확인) | 동일 |
| 국내 법규 | 원문 인용 가능 (공공저작물) | — |
| 공개 가이드(NIST 등) | 원문 인용 가능 (저작권 표기) | — |
| 고객사 As-Is | 외부 배포 불가, 내부 참조만 | 타 고객 산출물에 유출 |

→ qa-reviewer 가 생성물 내 `_inputs/01_표준원문/` 파일과 **20단어 이상 연속 일치** 검사, 탐지 시 부적합.

## 7. 사용자 워크플로우

```bash
# ① 표준 편입 전: _inputs/ 에 입력물 투하
mkdir -p vault/02_표준/ISO9001/_inputs/{01_표준원문,02_법규,03_해설서,04_AsIs,05_산업가이드}
cp ~/Downloads/KS_Q_ISO9001.pdf vault/02_표준/ISO9001/_inputs/01_표준원문/

# ② 편입 실행
/build-standard ISO9001

# ③ 결과물은 _inputs/ 를 근거로 생성되며 source_citation 포함
```

## 8. 검증 규칙 (qa-reviewer 체크 포인트)

- [ ] `_inputs/README.md` 에 각 파일 라이선스 상태 명시
- [ ] 생성된 POL/PRO/WI 중 `source_citation` 누락 문서 없음
- [ ] 라이선스 없는 표준 원문 문구 직접 복사 없음 (20단어 일치 검사)
- [ ] `[_inputs 미제공 — LLM 추정]` 꼬리표 있는 요구사항은 작업노트에 이슈로 등록

## 9. README 스텁 (자동 생성용)

에이전트가 빈 `_inputs/` 를 만날 때 자동 생성하는 `_inputs/README.md`:

```markdown
# {표준코드} 입력자료

## 필수 배치 (우선순위 순)
1. 표준 원문: `01_표준원문/`  — 라이선스 보유 시
2. 관련 법규: `02_법규/`      — law.go.kr 등 공공 소스
3. 해설서:   `03_해설서/`    — 인증기관·심사원 교재
4. As-Is:    `04_AsIs/`       — 고객사 기존 문서

## 현재 입력물 인벤토리
| 파일 | 유형 | 출처 | 라이선스 | 비고 |
|---|---|---|---|---|
| (입력물 추가 시 기록) | | | | |
```
