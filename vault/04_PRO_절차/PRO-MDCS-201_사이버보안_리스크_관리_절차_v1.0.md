# 사이버보안 리스크 관리 절차 (AI 보안 통합) (PRO-MDCS-201)

> 상위 정책: [[POL-MDCS-002_사이버보안_리스크관리_정책_v1.0]]
> 연계 정책: [[POL-MDCS-001_의료기기_사이버보안_기본방침_v1.0]]

## 1. 목적

디지털의료기기 수명주기 전반의 **사이버보안 위협·취약점을 식별·평가·통제**하고, **ISO 14971 안전 리스크 관리와 연계**하여 환자 안전에 미치는 영향을 최우선 기준으로 판단하며, **AI 적용 기기의 데이터·모델 위협**까지 통합 평가한다.

## 2. 적용 범위

- **설계·개발·검증·출시·운영·유지·레거시·폐기** 전 수명주기
- 자사 개발 + **제3자·오픈소스** 구성요소 (SBOM 기반)
- **AI 적용 기기**의 데이터·모델 공격, Fail-safe·대안 운영 모드 포함 (제12조)
- 시판 후(Post-market) **지속 감시·재평가**

## 3. 역할과 책임 (RACI)


| 단계                               | PSO   | R&D   | Safety/QA | PSC   | CEO              | DPO |
| -------------------------------- | ----- | ----- | --------- | ----- | ---------------- | --- |
| 위협 모델링                           | **R** | **R** | C         | A     | -                | -   |
| 취약점 평가 (스캐닝·펜테스트·SAST/DAST·SBOM) | **R** | C     | C         | A     | -                | -   |
| 위험 평가 (심각도·영향)                   | **R** | C     | **R**     | **A** | -                | C   |
| 통제 조치 설계·구현·검증                   | C     | **R** | C         | A     | -                | -   |
| 잔여 위험 수용                         | C     | -     | C         | **R** | **A** (Critical) | C   |
| 시판 후 지속 감시                       | **R** | C     | C         | A     | -                | -   |
| 연 1회 재평가                         | **R** | C     | C         | **A** | I                | -   |
| AI 데이터·모델 위협 평가                  | **R** | **R** | C         | A     | -                | C   |

## 5. 단계별 상세


| #   | 단계            | 설명                                                                  | 담당           | 입력        | 출력              |
| --- | ------------- | ------------------------------------------------------------------- | ------------ | --------- | --------------- |
| 1   | 자산·진입점 식별     | 공격 표면·자산·신뢰 경계·진입점 문서화                                              | PSO + R&D    | 아키텍처      | 자산 목록           |
| 2   | STRIDE 위협 모델링 | Spoofing/Tampering/Repudiation/Info Disclosure/DoS/Elevation 항목별 식별 | PSO + R&D    | 자산·아키텍처   | TMP-STRIDE 워크시트 |
| 3   | AI 위협 통합      | 데이터 중독·회피·모델 추출·반복 질의 등 AI 특화 위협 식별                                 | PSO + R&D    | AI 모델·데이터 | AI 위협 시나리오      |
| 4   | 취약점 평가        | 정기 스캐닝·펜테스트·SAST/DAST·SBOM 기반 CVE 추적                                | PSO + QA     | 제품 빌드     | 취약점 목록          |
| 5   | 위험 평가         | 심각도(CVSS) × 임상 영향 × 악용 가능성 × 탐지 용이성                                 | PSO + Safety | 위협·취약점    | 위험 평가 보고서       |
| 6   | 통제 조치         | 보안 코딩·보안 설계·암호화·접근통제·파일/입력 유효성·유지관리·물리보안 조치 설계·구현                   | R&D          | 평가 결과     | 통제 설계           |
| 7   | 잔여 위험 재평가     | 통제 후 잔여 위험 수준 재평가                                                   | PSO + Safety | 통제 구현 결과  | 잔여 위험 보고서       |
| 8   | 수용 결정         | 허용 가능 여부를 PSC 가 공식 수용, Critical 은 CEO                               | PSC/CEO      | 잔여 보고     | 수용 기록           |
| 9   | 종합 평가 보고      | 식별 위협·취약점·통제·잔여 수준을 망라한 종합 보고                                       | PSO          | 전 단계      | 종합 위험 보고서       |
| 10  | 시판 후 감시       | 최신 보안 동향 파악, 침해대응 계획·패치, MSP 정보공유 채널 운영                             | PSO          | 공개 정보·제보  | 감시 리포트          |
| 11  | 재평가           | 연 1회 정기 + 중대 신규 위협 발견 시 14일 이내 긴급                                   | PSO + PSC    | 변경·신규 위협  | 재평가 보고서         |
