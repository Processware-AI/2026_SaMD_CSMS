# 증적 수집 기준 업무지침 (WI-101-03)

> 상위 절차: [[PRO-MDCS-101_보안_문서화_관리_절차_v1.0]] · §5 단계 7

## 1. 업무 목적

SaMD-CSMS 제3~22조의 보안 활동 수행 사실을 식약처 심사·사내 감사 등 외부 검증 상황에서 재현 가능하도록, 증적의 수집 범위·보관 형식·변조 방지 수단·보존 기간을 단일 기준으로 규정한다.

## 2. 수행 주체

- **주 수행자**: 해당 보안 활동을 실제 수행한 부서의 담당자 (SecOps·개발팀·CSIRT·인프라팀 등)
- **검토자**: PSO (활동별 증적 목록 정합성 점검)
- **승인자**: 오동석 (보존 정책 예외 승인)

## 3. 범위

- 모든 보안 활동(POL·PRO·WI 에 정의된 단계)의 수행 증적
- 자동 생성 증적(시스템 로그·SAST/DAST 리포트·AV 스캔 로그) 과 수동 기록(훈련 수행 시나리오·회의록·서명본)을 모두 포함

## 4. 입력 자료 / 산출물

- **Input**
  - 보안 활동 수행 결과(로그·리포트·서명·사진 등)
  - [[TMP-DOC-002_증적_목록표_v0.1]] 기반 활동별 필수 증적 카탈로그
- **Output**
  - 변조 방지 상태로 보관된 REC(기록) 파일 세트
  - MAT-001 또는 증적 대장에 등록된 메타데이터

## 5. 수행 절차

### 5.1 사전 준비

1. 담당자가 자신이 수행하는 활동(PRO·WI 단위)의 필수 증적 카탈로그를 [[TMP-DOC-002_증적_목록표_v0.1]] 에서 확인한다.
2. 자동 증적은 수집 파이프라인(SIEM·로그 포워더)의 정상 동작을 점검하고, 수동 증적은 수집 양식(REC 양식)을 준비한다.
3. 수집 대상의 보존 기간을 확정한다. 기본은 최소 3년, 침해 관련 증적은 5년, 개인정보 관련 증적은 개인정보보호법 보존 기간을 따른다.

### 5.2 수행 단계

1. **증적 식별**
   - 활동 수행 직후 담당자가 활동 결과에 포함된 모든 산출물을 나열한다.
   - 카탈로그 대비 누락된 증적은 즉시 재수행 또는 사유 기록한다.
2. **변조 방지 처리**
   - 텍스트 증적: SHA-256 해시 생성 후 해시값을 대장에 함께 기록한다.
   - 중요 증적(침해대응·위험평가 보고서): 디지털 서명 또는 변조 불가 저장소(WORM) 로 이관한다.
   - 로그 증적: SIEM 의 무결성 기능(예: append-only index) 을 사용한다.
3. **보관 위치 지정**
   - REC 는 `vault/08_REC_기록/` 하위 연도별 폴더에 저장한다.
   - 저장 경로에 REC-{영역}-{YYYY}-{###} 명명 규칙을 적용한다.
4. **접근 권한 부여**
   - 기본: Doc Owner 읽기, PSO/QMR 읽기+감사.
   - 개인정보 포함 증적: DPO 포함, 접근 로그 기록.
5. **증적 대장 기록**
   - MAT-001 또는 증적 전용 대장에 Row 추가한다.
   - 기록 필드: REC 번호 / 상위 활동(WI) / 발생일 / 해시 / 보존기간 / 보관 위치.
6. **정기 감사**
   - PSO 가 분기 1회 샘플 10% 를 선정해 해시 재계산·접근 로그 검토를 수행한다.

### 5.3 완료 조건

- [ ] 활동별 필수 증적이 모두 수집되었다
- [ ] 해시값·서명이 확보되었거나 변조 방지 저장소에 이관되었다
- [ ] REC 파일명이 명명 규칙(REC-{영역}-{YYYY}-{###})에 맞게 지정되었다
- [ ] 보존 기간이 본 지침 §5.1 기준에 부합하게 설정되었다
- [ ] 증적 대장에 Row 가 등록되었다
- [ ] 개인정보 포함 여부가 명시되고 접근 권한이 적정하게 부여되었다

## 6. 인터페이스 부서

- **SecOps**: 자동 로그 수집 파이프라인 운영, SIEM 무결성 설정
- **DPO**: 개인정보 포함 증적 접근 권한 심의
- **IT팀**: WORM 저장소·Git 저장소 운영
- **감사팀**: 분기·연간 증적 감사

## 7. 주의사항 / 예외 처리

### 7.1 자동 증적 수집 실패 (SIEM 장애)

SIEM·로그 포워더 장애로 자동 증적 수집이 중단된 경우, SecOps 가 30분 이내 장애 티켓을 열고 수동 백업(서버 직접 로그 파일 복사 + 해시)으로 대체한다. 장애 기간 동안의 증적 공백 구간은 사후 보고서에 명시한다.

### 7.2 대용량 증적 (퍼징 로그·바이너리)

개별 증적 파일이 100MB 를 초과하는 경우 Vault 에 본문 저장하지 않고 WORM 저장소 URI 와 해시만 REC 에 기록한다. 본 예외는 분기 감사에서 실재 확인을 거친다.

### 7.3 보존 기간 조기 폐기 요청

법원 제출·영업 비밀 보호 등 특수 사유로 보존 기간 전 폐기 요청이 발생한 경우, 오동석 승인 + 법무 검토를 거쳐야 하며 폐기 사유·승인자·폐기 일시를 REC 에 최종 기록한다. 단 개인정보 관련 증적은 법정 보존 기간 내 폐기 금지.

### 7.4 해시 알고리즘 전환

SHA-256 이 향후 취약성 발견으로 폐지될 경우, KMS Admin 과 협의하여 신규 알고리즘(SHA-3 등) 으로 전환하고 기존 증적은 이중 해시(SHA-256 + SHA-3) 를 추가 기록한다.
