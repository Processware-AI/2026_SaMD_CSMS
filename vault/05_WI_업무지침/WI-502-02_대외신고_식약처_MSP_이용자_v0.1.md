# 대외 신고(식약처·MSP·이용자) 업무지침 (WI-502-02)

> 상위 절차: [[PRO-MDCS-502_침해사고_대응_및_신고_절차_v1.0]] · §5 단계 5, 11

## 1. 업무 목적

전자적 침해행위를 인지한 즉시 식품의약품안전처·의료서비스제공자·제품 이용 사용자에게 법정·내부 기준 시간 이내 초기 신고를 수행하고, 이후 조치 상황·추가 분석 결과·재발 방지 대책을 지속 공유한다.

## 2. 수행 주체

- **주 수행자**: RA(규제 담당) 리드, 홍보·CS 리드
- **검토자**: 오동석, 법무, DPO(개인정보 유출 시)
- **승인자**: CEO

## 3. 범위

- 식약처 침해행위 신고
- 의료서비스제공자(병원) 통지
- 제품 이용 사용자(환자·의료진) 안내
- 개인정보 유출 신고(개인정보보호위원회) 와 병행 시 연계

## 4. 입력 자료 / 산출물

  - 등급 판정서
  - 영향 범위 데이터
- **Output**

  - 신고 접수번호 기록

## 5. 수행 절차

### 5.1 사전 준비

1. RA 리드가 식약처·MSP·이용자 대상 비상 연락 채널을 상시 유지한다(전화·이메일·FAX·온라인 신고 시스템).
2. 24시간 대응 가능한 비상 Oncall 순번을 운영한다.
3. 신고 양식 표준화: 침해 유형·발생 시간·인지 시점·초기 영향 범위·현재 조치 상황.
4. MSP 리스트(계약 병원) 와 이용자 공지 채널(앱 푸시·메일·홈페이지) 준비.

### 5.2 수행 단계

1. **등급별 신고 대상 확정**
   - 경미: 내부 기록만 유지(MSP 사전 통지 권고).
   - 중대·위기: 식약처·MSP·이용자 전방위 신고 필수.
2. **초기 신고서 작성**
   - [[TMP-IR-002_초기_대외신고서_v0.1]] 의 필수 필드를 작성한다.
   - 오동석 1차 확인, CEO 최종 승인.
3. **신고 전송 (인지 후 24시간 이내)**
   - 식약처: 공식 신고 시스템 또는 지정 이메일·FAX.
   - MSP: 계약서 지정 채널(이메일·파트너 포털).
   - 이용자: 앱 푸시·메일·홈페이지 공지 중 적합 채널.
   - 전송 후 접수 확인(회신 또는 접수번호) 을 티켓에 기록.
4. **개인정보 유출 병행 신고**
   - DPO 가 개인정보보호법 기한 내 개인정보보호위원회·정보주체 통지.
5. **지속 정보 공유**
   - 조치 상황·추가 분석·재발 방지 대책을 주 1회 이상 공유한다.
   - [[TMP-IR-003_지속정보공유_리포트_v0.1]] 작성·배포.
6. **종료 통보**
   - 최종 재발 방지 대책과 영향 해소 확인 시 종료 통보 발송.
   - 종료 통보 이후에도 90일 지속 감시 기간을 운영한다.

### 5.3 완료 조건

- [ ] 인지 후 24시간 이내 식약처·MSP·이용자(해당 등급) 초기 신고가 완료되었다
- [ ] 신고서에 침해 유형·시간·인지·영향·조치 5개 필수 요소가 포함되었다
- [ ] 접수번호·수신 확인 증적이 티켓에 기록되었다
- [ ] 개인정보 유출 병행 시 법정 기한 내 별도 신고되었다
- [ ] 주 1회 이상 지속 정보 공유 리포트가 발송된다
- [ ] 종료 통보 후 90일 지속 감시 기간이 가동된다

## 6. 인터페이스 부서

- **법무**: 신고 문구·책임 범위 검토
- **DPO**: 개인정보 유출 신고 연계
- **홍보/CS**: 이용자 공지 채널 운영
- **계약/파트너**: MSP 지정 채널 관리

## 7. 주의사항 / 예외 처리

### 7.1 영향 범위 미확정 상태 신고

24시간 기한 내 영향 범위가 완전히 확정되지 않은 경우라도 **잠정 평가** 로 1차 신고한다. 이후 정확한 수치·영향으로 보완 신고한다. 기한 초과 사유는 허용되지 않음을 원칙으로 한다.

### 7.2 신고 전 공개 유출 (언론 보도)

침해 사실이 언론에 먼저 보도된 경우에도 규제기관 공식 신고 의무는 유지한다. 홍보팀이 초기 공식 입장(사실 확인·조치 계획) 을 발표하고, 후속 신고·공지는 WI 절차대로 수행.

### 7.3 MSP 다수 연쇄 영향

다수 MSP 가 동시 영향받는 경우 RA 리드가 통합 공지안을 작성하고 채널별 전송을 일괄 수행한다. MSP별 개별 이슈는 포털의 티켓으로 관리하여 혼선 방지.

### 7.4 해외 유통 제품 영향

해외 유통 제품에 영향이 있는 경우 해당 지역 규제기관(FDA·EU NCA 등) 에도 현지 기한·양식으로 신고한다. 해외 신고는 법무·RA 공동 추진.
