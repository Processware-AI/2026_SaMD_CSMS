# VDP 운영 및 외부 보안 연구자 대응 업무지침 (WI-601-02)

> 상위 절차: [[PRO-MDCS-601_취약점_감시_공개_조치_절차_v1.0]] · §5 단계 2 (외부 접수, VDP)

## 1. 업무 목적

취약점 공개 정책(Vulnerability Disclosure Policy, VDP)을 운영하여 외부 보안 연구자가 자사 디지털의료기기에서 발견한 취약점을 안전하고 투명한 채널로 신고할 수 있도록 하고, 접수·확인·조치·보상의 전 과정을 체계적으로 관리함으로써 선제적 취약점 해소와 연구자 신뢰 관계를 확보한다.

## 2. 수행 주체

- **주 수행자**: PSO (VDP 정책 운영, 접수·ACK, 영향 분석 총괄)
- **검토자**: CISO (VDP 정책 승인, 보상 수준 승인), 개발팀 (취약점 기술 재현 및 영향 검토), 법무팀 (연구자 면책 조항 검토)
- **승인자**: CISO (VDP 정책 및 보상 정책 최종 승인)

## 3. 범위

- 자사 홈페이지에 공개된 VDP를 통한 외부 자발적 취약점 신고
- 계약 기반 펜테스터(외주 보안 테스트) 의 취약점 제출 관리
- 접수된 취약점에 대한 임시 계정 발급 및 재현 환경 제공
- VDP 보상 프로그램(금전 보상·명예 게재) 운영
- 아래 사항은 본 WI 범위 제외: 내부 취약점 스캐닝(WI-601-01 담당), 패치 개발(WI-601-05 담당)

## 4. 입력 자료 / 산출물

- **Input**
  - 외부 연구자로부터 수신한 취약점 신고서(이메일·신고 폼)
  - VDP 정책 문서 (홈페이지 게시 버전)
  - 재현 테스트 환경 접근 요청서
  - [[TMP-VDP-020_외부신고_접수_대장_v0.1]] (신고 건 추적)
- **Output**
  - 완료된 [[TMP-VDP-020_외부신고_접수_대장_v0.1]] (접수번호·ACK·처리 상태)
  - [[TMP-VDP-021_연구자_임시계정_발급서_v0.1]] (발급 기기·권한 범위·만료일)
  - [[TMP-VDP-022_보상_지급_기록_v0.1]] (보상 유형·금액·지급일)
  - 외부 연구자 명예 게재 목록 (홈페이지 Hall of Fame)
  - VDP 운영 분기 보고서 (접수 건수·처리 현황·평균 ACK 시간)

## 5. 수행 절차

### 5.1 사전 준비

1. PSO는 VDP 정책 문서를 자사 홈페이지 `security.example.com/vdp` (또는 `security.txt`) 에 게시하고, 정책 버전·최종 업데이트 날짜를 명시한다. VDP 정책에는 다음 항목이 포함되어야 한다.
   - 신고 대상 범위 (제품·도메인·시스템 명시)
   - 허용 활동(접근 가능 범위) 및 금지 활동
   - ACK 기한 및 처리 일정
   - 연구자 면책 조항
   - 보상 정책 개요 (해당하는 경우)
2. 취약점 신고 접수 채널 (`security@[도메인]` 이메일 또는 온라인 신고 폼)이 정상 작동하고 PSO에게 알림이 전달되는지 분기마다 테스트한다.
3. 임시 테스트 계정 발급이 가능한 비운영(Staging/UAT) 환경이 준비되어 있는지 확인하고, 발급 가능 권한 범위를 사전 정의한다.

### 5.2 수행 단계

1. **취약점 신고 접수 및 ACK 발송**
   - 신고가 접수되면 PSO가 즉시 [[TMP-VDP-020_외부신고_접수_대장_v0.1]]에 접수번호(VDP-YYYY-NNN)·신고자(익명 허용)·신고 일시·요약을 등록한다.
   - 3 영업일 이내 연구자에게 접수 확인(ACK) 회신을 발송한다. ACK 메시지에는 접수번호·담당자 연락처·예상 처리 일정을 포함한다.
   - 신고 내용이 명백히 VDP 범위 외(소셜 엔지니어링, DoS, 물리 공격 등)인 경우 즉시 범위 외 통보를 발송한다.

2. **초기 분류 및 우선순위 결정**
   - PSO가 신고 내용을 검토하여 심각도(CVSS 기반) 초안을 산정하고, 관련 개발팀 담당자에게 24시간 이내 할당한다.
   - Critical/High 신고는 영향 분석 착수를 48시간 이내에 시작한다.
   - 동일 취약점이 이미 알려진 CVE이거나 내부적으로 이미 접수된 경우, 연구자에게 중복 접수임을 알리고 조치 진행 상황을 공유한다.

3. **임시 계정 발급 (재현 테스트 지원)**
   - 연구자가 재현 테스트 환경 접근을 요청한 경우, PSO가 필요성을 검토하고 CISO 승인 하에 최대 14일 만료 임시 계정을 발급한다.
   - [[TMP-VDP-021_연구자_임시계정_발급서_v0.1]]에 연구자 신원(실명 또는 연락처)·접근 범위·만료일·IAM 관리자 발급 확인을 기재한다.
   - 임시 계정 권한은 지정된 Staging 환경에만 국한되며, 운영 환경 접근은 절대 허용하지 않는다.
   - 만료 후 계정은 자동 비활성화된다. 연장은 새 신청으로만 가능하며, 재연장 시 CISO 재승인이 필요하다.
   - 계정 사용 로그를 전량 수집하여 지정 범위 외 접근 시도 여부를 모니터링한다.

4. **취약점 영향 분석 및 연구자 소통**
   - 개발팀이 취약점을 재현하고 CVSS 점수를 확정한다. 분석 결과를 PSO가 연구자에게 공유한다. (취약점 공개 전 정보 공유 시 기밀 유지 요청)
   - 연구자의 기술적 의견이 추가로 필요한 경우 PSO가 추가 문의를 조율한다.
   - 이의 제기 시: 연구자가 CVSS 산정에 이의를 제기하면 PSO와 개발팀이 공동으로 재검토하고, 결과를 연구자에게 통보한다.

5. **보상 정책 적용**
   - VDP 보상 정책(PSO·CISO 승인)에 따라 접수 취약점의 심각도·독창성·상세 보고 품질을 평가하여 보상을 결정한다.
   - 금전 보상: 심각도별 보상 범위(PSO·CISO 승인 기준) 내에서 PSO가 확정하고 [[TMP-VDP-022_보상_지급_기록_v0.1]]을 작성한다.
   - 명예 보상: 연구자 동의 하에 자사 홈페이지 `Hall of Fame`에 이름·ID를 게재한다.
   - 패치 완료 전 공개는 취약점이 악용될 위험이 있으므로, 연구자에게 협조 요청 후 조정 공개(Coordinated Disclosure) 일정을 합의한다.

6. **VDP 운영 분기 보고**
   - 분기 말 PSO는 접수 건수·평균 ACK 시간·처리 상태(조치 완료/진행/거절)·보상 지급 현황을 집계하여 CISO에 보고한다.
   - VDP ACK 평균 시간이 3 영업일 초과 시 원인 분석 및 프로세스 개선 계획을 함께 제출한다.

### 5.3 완료 조건

- [ ] 신고 전수에 대해 [[TMP-VDP-020_외부신고_접수_대장_v0.1]]에 접수번호가 등록되었다
- [ ] 3 영업일 이내 ACK 발송이 완료되었다
- [ ] 발급된 임시 계정이 [[TMP-VDP-021_연구자_임시계정_발급서_v0.1]]에 기록되고 CISO 승인이 확인된다
- [ ] 임시 계정이 만료 후 비활성화되었음이 IAM 로그로 확인된다
- [ ] 보상 결정 건이 [[TMP-VDP-022_보상_지급_기록_v0.1]]에 기재되었다
- [ ] 분기 VDP 운영 보고서가 CISO에 제출되었다
- [ ] 패치 완료 전 공개되지 않도록 연구자와 조정 공개 일정이 합의되었다

## 6. 인터페이스 부서

- **개발팀**: 취약점 기술 재현, 영향 분석, 패치 개발 착수
- **IAM 관리자 (인프라팀)**: 연구자 임시 계정 발급·만료 처리 ([WI-401-01] 연계)
- **CISO**: VDP 정책·보상 정책 승인, 임시 계정 발급 승인
- **법무팀**: 면책 조항·보상 조건의 법적 검토
- **홍보팀**: Hall of Fame 홈페이지 게재 지원

## 7. 주의사항 / 예외 처리

### 7.1 무단 침입과의 경계 처리

외부 연구자가 VDP 범위를 초과하여 운영 환경에 접근하거나 데이터를 취득한 경우, PSO는 즉시 CISO에 보고하고 PRO-MDCS-502(침해사고 대응)를 트리거한다. VDP 범위 내 활동이라도 환자 데이터 또는 개인정보가 노출되었다면 DPO를 통해 개인정보 유출 신고 절차를 진행한다.

### 7.2 고위험 취약점 긴급 처리

Critical 취약점 신고가 접수된 경우, 연구자에게 즉시 ACK를 발송하고(1 영업일 이내) PRO-MDCS-601의 Critical SLA(7일 이내 긴급 완화)에 따라 처리를 가속화한다. 연구자에게는 조치 진행 상황을 격일로 업데이트한다.

### 7.3 악의적 신고 또는 협박성 공개 위협

연구자가 조치 전 공개를 위협하거나 금전 요구를 하는 경우:
- PSO가 법무팀·CISO에 즉시 보고한다.
- 외부 연구자와의 커뮤니케이션 전문 창구를 PSO 단일 채널로 집중하고, 비공식 접촉을 차단한다.
- 법무팀이 필요 시 법적 조치 여부를 검토한다.

### 7.4 동일 연구자의 반복 신고

동일 연구자가 6개월 이내 5건 이상 연속으로 낮은 품질의 신고를 제출하는 경우, PSO는 CISO 승인 아래 해당 연구자의 신고를 낮은 우선순위로 처리하거나 VDP 접근을 제한할 수 있다. 해당 조치는 반드시 연구자에게 통보한다.
