# SBOM MSP 제공·계약·포털 운영 업무지침 (WI-302-04)

> 상위 절차: [[PRO-MDCS-302_SBOM_관리_절차_v1.0]] · §5 단계 4

## 1. 업무 목적

SaMD-CSMS 제16조 제3호는 디지털의료기기 제조사가 의료서비스제공자(MSP)에게 SBOM을 제공하고 구매·설치 이전 구성요소 확인을 지원해야 함을 규정한다. 본 지침은 MSP로부터의 SBOM 제공 요청을 접수하여 계약 조건 검증, 제공 채널(포털 또는 SFTP) 운영, 제공 이력 관리까지의 절차를 체계화하여 MSP가 의료 환경에서의 보안 위험을 사전에 평가할 수 있도록 지원한다.

## 2. 수행 주체

- **주 수행자**: SBOM 관리자 (요청 접수·SBOM 추출·제공 이력 관리)
- **검토자**: 법무팀 (계약 조건 확인), PSO (민감 정보 포함 여부 검토)
- **승인자**: CISO (신규 MSP 또는 비계약 채널을 통한 예외 제공 시)
- **협력**: MSP 기술 담당자 (포털 계정·수신 확인), 고객지원팀 (요청 접수 창구)

## 3. 범위

- 당사와 공급 계약·유지보수 계약을 체결한 모든 MSP의 SBOM 요청
- 제품 구매 협상 단계에서 사전 검토 목적으로 요청되는 SBOM (계약 전 제공 포함)
- 릴리스별·컴포넌트별 SBOM 모두 포함
- SBOM 포털 계정 발급·관리 및 SFTP 보조 채널 운영

## 4. 입력 자료 / 산출물

- **Input**
  - MSP의 공식 SBOM 제공 요청서 (이메일·포털 폼·계약 첨부 요청 등)
  - 계약 정보 (MSP 명칭·계약 번호·계약 유효기간·NDA 여부)
  - WI-302-03에 따라 서명·보호된 SBOM 파일
  - 대상 제품명·버전·릴리스 일자
- **Output**
  - MSP 포털 또는 SFTP를 통해 제공된 SBOM 파일
  - SBOM 제공 이력 (요청일·제공일·수신자·채널·제품 버전·서명 검증 결과)
  - MSP 포털 계정 대장 갱신본
  - 계약 전 제공 시 비밀유지 확약서(NDA) 수신 확인

## 5. 수행 절차

### 5.1 사전 준비

1. SBOM 관리자는 MSP 포털의 계정 목록과 계약 데이터베이스를 분기별로 동기화하여 계약이 만료된 MSP 계정이 자동 비활성화되도록 구성한다.
2. 포털에 제공 가능한 최신 릴리스 SBOM 파일이 WI-302-03의 서명·보호 절차를 거쳐 업로드되어 있는지 확인한다.
3. SBOM 요청 SLA(요청 접수 후 5 영업일 이내 제공)를 고객지원팀과 공유하고, SLA 초과 시 자동 에스컬레이션 알림을 설정한다.

### 5.2 수행 단계

1. **요청 접수 및 MSP 자격 확인**
   - 고객지원팀 또는 SBOM 관리자가 MSP의 요청을 접수하고 JIRA 또는 서비스 데스크 티켓으로 등록한다.
   - MSP가 유효한 공급·유지보수 계약을 보유하는지 법무팀 계약 DB에서 확인한다.
   - 계약이 유효하지 않은 경우, 제공을 보류하고 고객지원팀을 통해 계약 체결 또는 갱신을 안내한다.
   - 구매 협상 중인 잠재 MSP가 사전 검토 목적으로 요청하는 경우, NDA 서명 원본을 법무팀이 수신 확인한 후에만 제공한다.

2. **계약 조건 및 SBOM 사용 범위 검토**
   - 법무팀이 계약서 내 SBOM 수신자 의무(기밀 유지·재배포 금지·보안 목적 사용 제한)가 명시되어 있는지 확인한다.
   - 해당 조항이 없는 계약의 경우, SBOM 제공 전 개정 계약서 또는 별도 SBOM 수신 동의서를 MSP로부터 수령한다.

3. **제공 SBOM 특정 및 민감 정보 검토**
   - SBOM 관리자가 요청된 제품·버전에 해당하는 서명된 SBOM 파일을 저장소에서 특정한다.
   - PSO가 SBOM에 내부 개발 환경 경로·미공개 특허 구성요소명 등 외부 노출 제한 정보가 포함되어 있는지 확인한다. 제한 정보가 있는 경우 해당 필드를 마스킹하거나 별도 제한 버전을 생성한다.

4. **포털을 통한 제공**
   - SBOM 관리자가 MSP 포털에서 해당 MSP 계정에 제품·버전별 SBOM 파일을 게시한다.
   - MSP 담당자에게 포털 접속 URL, 파일명, 서명 공개 키 위치를 이메일로 안내한다.
   - MSP가 포털 다운로드 및 서명 검증을 완료하면 수신 확인 이메일을 발송하도록 안내하고, 확인 기록을 제공 이력에 첨부한다.

5. **SFTP 보조 채널 (포털 사용 불가 MSP)**
   - 기술 환경상 포털 접속이 불가능한 MSP에 한해 SFTP(SFTP/TLS 1.2 이상) 채널로 파일을 전송한다.
   - SFTP 계정은 해당 SBOM 제공 완료 후 즉시 비활성화하며, 1회 다운로드 후 파일도 삭제한다.
   - SFTP 채널 사용 시 PSO에 사전 통보하고 제공 이력에 사유를 명기한다.

6. **제공 이력 관리 및 SLA 확인**
   - 모든 제공 건에 대해 제공 이력 대장을 갱신한다: 요청 티켓 번호·MSP명·제품 버전·채널·제공 일시·수신 확인 일시.
   - 요청 접수일로부터 제공 완료까지 5 영업일을 초과한 경우 초과 사유와 조치 내용을 기록하고 PSO에 보고한다.

### 5.3 완료 조건

- [ ] MSP 계약 유효성이 확인되었고, 계약 전 제공의 경우 NDA 수신 확인이 완료되었다
- [ ] 계약서에 SBOM 기밀 유지 조항이 확인되었거나 별도 동의서가 수령되었다
- [ ] PSO의 민감 정보 검토가 완료되었고 필요 시 마스킹 버전이 생성되었다
- [ ] WI-302-03의 서명이 완료된 SBOM 파일이 포털 또는 SFTP를 통해 제공되었다
- [ ] MSP의 수신 확인이 제공 이력 대장에 기록되었다
- [ ] SLA(5 영업일)가 준수되었거나, 초과 시 사유와 조치 내용이 기록되었다
- [ ] SFTP 사용의 경우 계정 비활성화 및 파일 삭제가 확인되었다

## 6. 인터페이스 부서

- **법무팀**: 계약 유효성 확인, NDA·SBOM 조항 검토, 계약 전 제공 동의서 관리
- **고객지원팀**: MSP 요청 접수 창구, SLA 모니터링 대시보드 운영
- **PSO**: 민감 정보 포함 여부 검토, SFTP 사용 사전 통보 수신
- **인프라 팀**: 포털 계정 발급·만료 처리, SFTP 서버 운영
- **CISO**: 계약 외 예외 제공 승인, 신규 채널 도입 승인

## 7. 주의사항 / 예외 처리

### 7.1 규제 기관 긴급 요청

식품의약처 등 규제 기관이 조사 목적으로 SBOM을 요청하는 경우, 계약 요건 검토 없이 법무팀의 요청 진위 확인 후 CISO 승인을 받아 즉시 제공한다. 규제 기관 제공 이력은 별도 규제 대응 로그에 분리 기록하고, 제공 후 24시간 이내 법무팀·CISO에게 보고한다.

### 7.2 구 버전 SBOM 요청 (단종 제품)

지원 종료(EOS) 처리된 제품의 SBOM 요청 시, 해당 제품의 저장소 보관 기간(출시 후 최소 10년) 내에서는 동일한 절차로 제공한다. 보관 기간이 경과한 경우 법무팀의 법적 보관 의무 확인 후 제공 불가 사유를 서면으로 통보한다.

### 7.3 MSP 포털 서비스 장애

포털 서비스 장애로 파일 게시가 불가능한 경우, SLA 시계를 일시 정지하고 인프라 팀에 복구를 요청한다. 복구까지 24시간을 초과하면 PSO 승인 하에 SFTP 보조 채널로 전환한다. 포털 복구 후 동일 파일을 포털에도 재게시하고 이력에 장애 경위를 기록한다.
