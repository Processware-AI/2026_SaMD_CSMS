# 폐기 및 데이터 안전 삭제 업무지침 (WI-303-04)

> 상위 절차: [[PRO-MDCS-303_레거시_의료기기_관리_절차_v1.0]] · §5 단계 7

## 1. 업무 목적

의료기기의 폐기·양도·반품 시 저장매체에 남은 개인정보·환자 데이터·암호키·설정 정보를 복구 불가능 상태로 영구 삭제하여 개인정보보호법 준수와 환자 프라이버시 보호를 달성한다.

## 2. 수행 주체

- **주 수행자**: DPO, 서비스팀
- **검토자**: 법무, PSO
- **승인자**: VP of R&D

## 3. 범위

- 기기 폐기(고장·노후·EOS 후 회수)
- 기기 양도·반품·리퍼브
- 내부 시험·개발 기기의 폐기

## 4. 입력 자료 / 산출물

- **Input**
  - 기기 회수·반품·폐기 요청
  - 기기별 저장매체 사양
- **Output**
  - [[TMP-EOS-002_데이터_안전삭제_지침_v0.1]]
  - 삭제 확인서(Certificate of Sanitization)

## 5. 수행 절차

### 5.1 사전 준비

1. DPO 가 기기 유형별 저장매체 목록(HDD·SSD·내장 플래시·SD 카드) 을 유지.
2. 삭제 기법을 NIST SP 800-88 기준으로 표준화: Clear / Purge / Destroy.
3. 인증 삭제 도구(블랑코 등) 라이선스 관리.

### 5.2 수행 단계

1. **매체 식별**
   - 기기 회수 시 저장매체 유형·용량·암호화 여부 확인.
2. **삭제 기법 선택**
   - 암호화 매체: Crypto-erase(키 폐기).
   - HDD: 다회 덮어쓰기(NIST Purge).
   - SSD: 제조사 Secure Erase 명령.
   - 파손·폐기 매체: 물리 파쇄(Destroy).
3. **삭제 실행**
   - 인증된 도구로 실행하고 로그 수집.
   - 2인 입회(실행자 + DPO 대리인) 로 무결성 보장.
4. **검증**
   - 삭제 후 샘플 섹터 재판독으로 복구 불가 확인.
5. **삭제 확인서 발급**
   - 기기 일련번호·매체 정보·기법·일시·담당자·2인 서명을 포함한 확인서 발급.
   - MSP 에 사본 제공.
6. **MSP 대상 지침 제공**
   - MSP 가 자체 폐기하는 경우 [[TMP-EOS-002_데이터_안전삭제_지침_v0.1]] 배포.
   - 공식 삭제 도구·절차 교육 자료 제공.

### 5.3 완료 조건

- [ ] 기기별 매체 유형이 식별되었다
- [ ] 매체별 적정 삭제 기법(Clear/Purge/Destroy) 이 선택되었다
- [ ] 인증 도구로 삭제가 실행되고 로그가 수집되었다
- [ ] 2인 입회·서명이 기록되었다
- [ ] 샘플 복구 검증이 통과되었다
- [ ] 삭제 확인서가 발급·MSP 에 제공되었다

## 6. 인터페이스 부서

- **법무**: 개인정보보호법 파기 의무 준수 검토
- **서비스팀**: 현장 회수·폐기 실행
- **환경 안전**: 물리 파쇄 시 폐기물 처리
- **재무**: 자산 처분 기록

## 7. 주의사항 / 예외 처리

### 7.1 암호화 매체 키 손상

Crypto-erase 의 대상 키가 손상되어 복호화 키 삭제 확증이 어려운 경우, Purge 방식으로 추가 수행하거나 물리 파쇄로 전환.

### 7.2 현장 폐기 불가

의료 현장에서 즉시 폐기가 불가한 경우 매체만 분리 회수하여 본사로 이관, 본사에서 표준 절차 수행. 이관 중 이중 포장·봉인으로 유출 예방.

### 7.3 MSP 자체 폐기 요구

MSP 가 자체 폐기를 고수하는 경우 제조업자는 지침·도구만 제공하고 삭제 확인서 수령 요구. MSP 서명본 수령 시까지 책임 잔존 인지.

### 7.4 양도·리퍼브

양도 전 완전 초기화 후 MSP 인수인에게 새 계정·키 발급. 이전 사용자 식별 정보가 잔존하지 않도록 이중 검증.
