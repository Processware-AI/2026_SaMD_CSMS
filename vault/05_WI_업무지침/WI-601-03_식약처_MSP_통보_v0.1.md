# 식약처·MSP 취약점 통보 업무지침 (WI-601-03)

> 상위 절차: [[PRO-MDCS-601_취약점_감시_공개_조치_절차_v1.0]] · §5 단계 4

## 1. 업무 목적

디지털의료기기에서 확인된 취약점을 식품의약품안전처와 의료서비스제공자에게 통보할 때 필요한 양식·포함 정보·기한·후속 업데이트 방식을 표준화한다.

## 2. 수행 주체

- **주 수행자**: RA 리드
- **검토자**: PSO, 오동석, 법무
- **승인자**: CEO (위기 등급), 오동석 (일반)

## 3. 범위

- Critical/High 심각도 취약점의 식약처 통보
- MSP(병원) 대상 취약점 통보
- 후속 패치 배포·조치 상황 업데이트

## 4. 입력 자료 / 산출물

- **Input**
  - 영향 분석 보고서 (CVSS·임상 영향)
  - 대응 계획 초안
- **Output**
  - [[TMP-VUL-002_취약점_통보서_v0.1]]
  - 통보 전송 이력·접수번호

## 5. 수행 절차

### 5.1 사전 준비

1. RA 리드가 식약처 취약점 통보 채널(공식 이메일·신고 시스템) 과 MSP 통보 채널(계약 포털·담당자 이메일) 을 상시 유지한다.
2. 통보서 템플릿의 필수 항목(상세 내용·심각도·영향·대응 방안) 을 확정한다.

### 5.2 수행 단계

1. **통보 대상 판정**
   - Critical·High 는 기본 통보 대상.
   - Medium 은 환자 안전·다수 MSP 영향 가능 시 통보.
2. **통보서 작성**
   - [[TMP-VUL-002_취약점_통보서_v0.1]] 작성.
   - 필수 필드: 제품명·버전·VUL ID·CVSS·영향 범위·대응 방안·SLA.
3. **검토·승인**
   - PSO 기술 검토 → 오동석 보안 검토 → 법무 리뷰 → CEO 또는 오동석 승인.
4. **전송**
   - 식약처: 지정 채널로 전송 후 접수 확인.
   - MSP: 영향 제품 설치 병원에 선별 전송.
5. **후속 업데이트**
   - 대응 진행에 따라 주 1회 이상 업데이트.
   - 패치 배포·공개 시 종료 통보.
6. **기록**
   - 통보·회신·후속 이력을 VUL 티켓에 첨부.

### 5.3 완료 조건

- [ ] Critical/High 취약점 통보가 대응 계획 확정 후 기한 내 발송되었다
- [ ] 통보서에 상세·심각도·영향·대응 4대 요소가 포함되었다
- [ ] 식약처 접수번호 또는 수신 확인 증적이 기록되었다
- [ ] MSP 선별 전송 로직이 적용되었다
- [ ] 후속 업데이트가 주 1회 이상 수행된다
- [ ] 최종 종료 통보가 발송·기록되었다

## 6. 인터페이스 부서

- **PSO**: 기술적 정확성 보증
- **법무**: 표현·책임 범위 검토
- **계약·파트너**: MSP 매칭 데이터 제공
- **홍보/CS**: 이용자 공지 병행 시 협업

## 7. 주의사항 / 예외 처리

### 7.1 패치 전 공개 위험

패치 개발 전 통보가 악용 위험을 증가시킬 수 있으므로, 통보 수신자에게 공개 자제 요청을 명시한다. 공개 시점 조정은 [[WI-601-04_홈페이지_공개_및_조정공개_v0.1]] 로 관리.

### 7.2 국내·해외 통보 병행

해외 유통 제품은 현지 규제기관(FDA·EU) 에도 통보한다. 국내외 통보 내용 일관성 유지를 위해 단일 마스터 문서 관리.

### 7.3 MSP 연락 실패

이메일·포털 접속 실패 등으로 MSP 에 전달 불가 시, 계약서 지정 대체 채널(전화·FAX) 로 재시도하고 이력에 기록한다. 2영업일 내 확인 불가 MSP 는 계약 담당에 에스컬레이션.

### 7.4 비공개 요청 (Coordinated Disclosure)

외부 연구자 제보 시 연구자가 비공개·조정 공개를 요청한 경우, 통보 내용에 그 사실을 반영하고 공개 시점을 연구자와 합의.
