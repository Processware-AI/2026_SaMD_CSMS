# 네트워크 보안 설정 업무지침 (WI-402-01)

> 상위 절차: [[PRO-MDCS-402_보안_통신_및_암호화_관리_절차_v1.0]] · §5 단계 1~2

## 1. 업무 목적

디지털의료기기 운영 네트워크를 일반 사무 네트워크·외부 공용 인터넷과 분리하여 공격 표면을 축소하고, 방화벽·IPS·NAC·WPA3 를 조합한 다층 방어를 일관되게 설정·유지한다.

## 2. 수행 주체

- **주 수행자**: 네트워크팀 엔지니어
- **검토자**: 인프라팀장, SecOps
- **승인자**: 오동석

## 3. 범위

- 의료기기 전용 VLAN·DMZ·관리 VLAN 설계
- 방화벽 정책(North-South·East-West)
- 무선 네트워크(WPA3 이상) 구성 및 SSID 분리
- NAC(802.1X) 기반 단말 인증
- IPS·IDS 운영(시그니처·룰 갱신)

## 4. 입력 자료 / 산출물

- **Input**
  - 제품 아키텍처·트래픽 흐름도
  - [[TMP-NET-001_네트워크_변경관리서_v0.1]] 양식
- **Output**
  - 최신 네트워크 도면(VLAN·방화벽 구간)
  - 방화벽 룰셋·IPS 시그니처 설정 기록
  - NAC 등록 단말 목록

## 5. 수행 절차

### 5.1 사전 준비

1. 네트워크팀이 의료기기 운영·관리·사무·게스트 네트워크의 세그먼트 정책을 문서화한다.
2. 모든 방화벽 기본 정책을 deny-by-default 로 유지한다.
3. 네트워크 변경 관리 승인 체계(CAB, Change Advisory Board) 를 운영한다.

### 5.2 수행 단계

1. **세그먼트 설계**
   - 의료기기 운영 VLAN 은 사무 VLAN·게스트 VLAN 과 L3 방화벽으로 분리한다.
   - 외부 연동 서비스는 DMZ 에만 배치하고 내부로 직접 연결하지 않는다.
2. **방화벽 룰 운영**
   - 모든 허용 룰은 변경관리서(CAB 승인) 기반으로만 추가한다.
   - 룰 설명(ticket·기한·담당자) 을 주석으로 남긴다.
   - 분기마다 불용 룰을 식별해 제거한다.
3. **IPS 운영**
   - 시그니처를 주 1회 이상 갱신한다.
   - 의료기기 전용 룰셋(예: DICOM/HL7 이상 트래픽) 을 적용한다.
   - False Positive 는 SecOps 와 협의해 튜닝한다.
4. **무선 보안**
   - WPA3-Enterprise 를 원칙으로 하고 불가피한 경우 WPA2-Enterprise 로 구성한다.
   - 운영 SSID 와 게스트 SSID 를 물리적으로 분리한다.
   - RADIUS 연동으로 802.1X 인증을 적용한다.
5. **NAC(단말 인증)**
   - 신규 단말은 MAC 주소·장치 인증서 등록 후 접속 허용한다.
   - 비인가 단말은 격리 VLAN 으로 전환한다.
6. **로그·모니터링**
   - 방화벽·IPS·NAC 로그를 SIEM 에 전달한다.
   - 분기별 룰셋·시그니처 리뷰 보고서를 작성한다.

### 5.3 완료 조건

- [ ] 의료기기 VLAN 이 사무·게스트 VLAN 과 방화벽으로 분리되었다
- [ ] 방화벽 기본 정책이 deny-by-default 로 유지된다
- [ ] IPS 시그니처가 주 1회 이상 갱신된다
- [ ] 무선 네트워크가 WPA3 또는 WPA2-Enterprise 로 구성되었다
- [ ] NAC 단말 등록 목록이 최신 상태로 유지된다
- [ ] 네트워크 로그가 SIEM 으로 전달된다

## 6. 인터페이스 부서

- **인프라팀**: 서버·스토리지 포트 보안
- **SecOps**: SIEM 룰·IPS 튜닝 협력
- **시설관리팀**: 무선 AP 물리 배치·보안

## 7. 주의사항 / 예외 처리

### 7.1 긴급 장애 우회 룰

서비스 장애 대응 중 임시 허용 룰이 필요한 경우 네트워크팀장 구두 승인으로 24시간 한시 룰을 추가한다. 24시간 내 정식 CAB 승인을 받거나 자동 만료시킨다.

### 7.2 의료기기의 레거시 프로토콜

구형 의료기기가 암호화 미지원 프로토콜(FTP 등) 을 요구할 경우, 운영 VLAN 내부 한정으로 예외 허용하고 보완 통제(네트워크 암호화 터널·보안 게이트웨이) 를 적용한다. 매년 교체 계획 재검토.

### 7.3 방문자·벤더 무선 접속

방문자·벤더는 게스트 SSID 만 사용하며, 내부 VLAN 접근은 VPN 경유로만 허용한다. 방문자 계정은 최대 1일 자동 만료.

### 7.4 IPS False Positive 로 인한 임상 영향

의료기기 정상 트래픽이 IPS 로 차단되어 환자 안전에 영향이 발생한 경우 네트워크팀장이 즉시 해당 룰을 감시(detect-only) 모드로 전환하고, 24시간 이내 튜닝 후 차단 복원한다. 사건은 침해·장애 로그와 별도로 기록하여 [[PRO-MDCS-501_보안_모니터링_및_로그_관리_절차_v1.0]] 에 에스컬레이션한다.
