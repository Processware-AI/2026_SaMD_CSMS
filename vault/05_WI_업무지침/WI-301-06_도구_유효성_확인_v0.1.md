# 도구 유효성 확인 업무지침 (WI-301-06)

> 상위 절차: [[PRO-MDCS-301_보안_SDLC_절차_v1.0]] · §5 단계 10

## 1. 업무 목적

SaMD-CSMS 제14조 제3호에 따라 보안 테스트 도구(SAST·DAST·퍼저·SCA 등)가 **의도한 대로 작동하며 결과를 신뢰할 수 있는 상태**임을 반기(6개월) 1회 이상 공식적으로 확인하고, 도구를 운용하는 담당자의 **교육 이력을 관리**하여 도구 오용·결과 오해석으로 인한 취약점 누락을 방지한다.

## 2. 수행 주체

- **주 수행자**: QA (도구 유효성 확인 수행·보고서 작성)
- **지원**: R&D (기준 입력·결과 비교 지원), PSO (유효성 기준 승인)
- **교육 담당**: PSO 또는 지정 외부 강사
- **검토자**: PSO
- **승인자**: VP of R&D

## 3. 범위

- **대상 도구 범주**:
  - **SAST 도구** (예: SonarQube, Semgrep, Checkmarx, Fortify)
  - **DAST 도구** (예: OWASP ZAP, Burp Suite Professional)
  - **퍼징 도구** (예: AFL++, libFuzzer, Atheris, Jazzer)
  - **SCA 도구** (예: Dependency-Track, OWASP Dependency-Check, Snyk)
  - **펜테스트 도구 세트** (nmap, Metasploit, sqlmap 등 승인 목록)
- **수행 주기**:
  - 반기 1회 정기 유효성 확인 (1월, 7월 기준월)
  - 도구 메이저 버전 업그레이드 후 즉시
  - 새 도구 도입 시 최초 도입 전
  - 중대한 오탐·누락 패턴 발견 시 임시 재확인

## 4. 입력 자료 / 산출물

- **Input**
  - 승인 도구 목록 (PSO 유지 관리)
  - 유효성 확인 기준 입력 세트 (Known-Vulnerability 테스트 케이스): 의도적으로 취약점을 포함한 벤치마크 코드·설정·패키지
  - 이전 주기 유효성 확인 보고서
  - 도구 릴리스 노트 및 변경 이력
  - 교육 이력 대장 (TMP-TRAIN-001)
- **Output**
  - 도구 유효성 확인 보고서 (TMP-TEST-006): 도구별 확인 결과·적합/부적합 판정
  - 교육 이력 업데이트 (TMP-TRAIN-001)
  - 부적합 도구 조치 기록 (해당 시)
  - 승인 도구 목록 갱신 (해당 시)

## 5. 수행 절차

### 5.1 사전 준비

1. QA 는 반기 시작 전 도구 유효성 확인 일정을 수립하고 PSO·R&D 에 공지한다. 정기 주기는 1월 첫째 주, 7월 첫째 주를 기준으로 2주 이내 완료를 목표로 한다.
2. PSO 는 승인 도구 목록을 최신화하고 각 도구의 현재 운용 버전을 확인한다.
3. QA 는 **유효성 확인 기준 입력 세트**를 준비한다. 기준 입력 세트는 다음 기준을 충족해야 한다.
   - 알려진 취약점(CVE 또는 CWE 매핑 가능)을 의도적으로 포함한 코드·패키지·바이너리
   - 해당 도구가 탐지해야 할 취약점 유형을 최소한 다음 수준으로 포함:
     - SAST: SQL Injection, Buffer Overflow, Hard-coded Credentials, Path Traversal 각 1개 이상
     - DAST: XSS, CSRF, Authentication Bypass, Insecure Deserialization 각 1개 이상
     - Fuzzer: NULL 포인터 역참조, 힙 오버플로, 스택 오버플로 각 1개 이상
     - SCA: 알려진 CVE ≥ CVSS 7.0 인 컴포넌트 3개 이상
   - 취약점 없는 정상 입력도 포함하여 위양성(false positive) 비율을 측정한다.
4. 기준 입력 세트의 **기대 결과(expected output)**를 사전에 문서화한다.

### 5.2 수행 단계

#### A. 도구별 유효성 확인 실행

1. **격리 환경 구성**: 유효성 확인은 실제 프로덕션 코드와 분리된 격리 환경에서 수행한다.
2. **도구 실행**: 각 도구를 기준 입력 세트에 적용하고 결과를 수집한다. 도구 설정(룰셋·플러그인·임계값)은 실제 운용 설정과 동일하게 유지한다.
3. **결과 비교**: 수집된 결과를 사전 정의된 기대 결과와 비교한다.
   - **탐지율(Recall)**: 기준 입력의 알려진 취약점 중 도구가 탐지한 비율. 목표: ≥ 90%.
   - **위양성율(False Positive Rate)**: 정상 입력에서 오경보를 발생시킨 비율. 목표: ≤ 10%.
4. **결과 기록**: TMP-TEST-006 에 도구별로 탐지율·위양성율·탐지된 취약점 목록·미탐지 항목을 기재한다.

#### B. 유효성 판정

1. **적합(Qualified)**: 탐지율 ≥ 90% AND 위양성율 ≤ 10%. 해당 도구는 다음 주기까지 승인 상태를 유지한다.
2. **조건부 적합**: 탐지율 80~90% 또는 위양성율 10~20%. PSO 가 원인을 분석하고 룰셋·설정을 조정한 후 재확인을 수행한다. 조정 후 적합 기준 달성 시 승인.
3. **부적합(Disqualified)**: 탐지율 < 80% 또는 위양성율 > 20%, 또는 도구가 실행 자체에 실패. 해당 도구를 승인 목록에서 제외하고 즉시 PSO·QA 에 통보한다. 대체 도구를 지정하거나 도구 교체 계획을 수립한다.

#### C. 부적합 도구 조치

1. 부적합 판정 시 해당 도구가 최근 수행한 스캔 결과의 신뢰성을 재검토한다.
2. 도구 부적합 기간 동안 수행된 스캔 결과가 릴리스 결정에 사용된 경우, PSO 는 위험 영향을 평가하고 필요 시 보완 스캔을 수행한다.
3. 도구 교체 또는 업그레이드 후 최초 도입 절차(§5.2 A~B)를 재수행한다.

#### D. 교육 이력 관리

1. **교육 대상**: 보안 테스트 도구를 직접 운용하는 모든 QA·PSO·R&D 담당자.
2. **교육 주기 및 내용**:
   - **신규 담당자**: 도구 운용 전 초기 교육 이수 필수. 교육 내용: 도구 기본 운용·결과 해석·보안 영향 판단·위양성 처리.
   - **기존 담당자 (연 1회)**: 새 버전 주요 변경 사항·신규 취약점 유형·오탐 사례 검토.
   - **메이저 버전 업그레이드**: 업그레이드 전 신규 기능·변경된 결과 형식 교육.
3. **교육 기록**: TMP-TRAIN-001 에 교육 참가자·일자·내용·수료 확인을 기재한다. 미이수자는 도구 독립 운용 권한을 부여받을 수 없다.
4. **교육 자료 유지**: PSO 는 도구별 운용 매뉴얼·결과 해석 가이드·자주 발생하는 위양성 사례 라이브러리를 최신화하여 공유 저장소에 유지한다.

#### E. 유효성 확인 결과 반영

1. 유효성 확인 결과(TMP-TEST-006)를 PSO 가 검토하고 VP of R&D 가 승인한다.
2. 승인 도구 목록을 갱신하여 현재 적합 상태인 도구 버전을 명시한다.
3. 유효성 확인 보고서를 취약점 평가 실행 시 참고 문서로 첨부하도록 QA 에 공지한다.

### 5.3 완료 조건

- [ ] 승인 도구 목록의 모든 도구에 대해 유효성 확인이 수행되었다
- [ ] 기준 입력 세트가 규정된 취약점 유형을 충족한다
- [ ] 각 도구의 탐지율·위양성율이 측정되고 TMP-TEST-006 에 기재되었다
- [ ] 적합/조건부 적합/부적합 판정이 기준에 따라 내려졌다
- [ ] 부적합 도구가 승인 목록에서 제외되고 조치 계획이 수립되었다
- [ ] 교육 이수 현황이 TMP-TRAIN-001 에 최신화되었다
- [ ] 미이수자에 대한 도구 운용 제한이 적용되었다
- [ ] VP of R&D 가 유효성 확인 보고서를 승인하였다
- [ ] 승인 도구 목록이 갱신되어 팀에 공지되었다

## 6. 인터페이스 부서

- **R&D**: 기준 입력 세트의 알려진 취약점 코드 작성 지원, 부적합 도구 교체 의견 제공
- **PSO**: 유효성 기준 수립·승인 도구 목록 유지, 교육 자료 개발
- **QA**: 유효성 확인 실행, TMP-TEST-006 작성, 교육 이수 현황 관리
- **IT/인프라**: 도구 라이선스 관리, 도구 버전 업그레이드 지원
- **HR/교육**: 외부 교육 프로그램 조율 (해당 시)

## 7. 주의사항 / 예외 처리

### 7.1 기준 입력 세트가 최신 취약점 유형을 반영하지 못하는 경우

보안 환경 변화(신규 공격 기법·새 CWE 유형 공개)로 기존 기준 입력 세트가 최신 위협을 포괄하지 못할 수 있다. PSO 는 연 1회 기준 입력 세트를 검토하고 최신 OWASP Top 10 변경·NVD 신규 CWE 추가 사항을 반영하여 갱신한다. 기준 입력 세트 갱신은 다음 반기 유효성 확인 전에 완료한다.

### 7.2 상용 도구 라이선스 만료 또는 공급사 지원 종료

상용 SAST/DAST 도구의 라이선스가 만료되거나 공급사가 지원을 종료하는 경우, 즉시 PSO 에 통보하고 대체 도구 평가를 시작한다. 대체 도구 적합 판정 전까지 해당 테스트 영역에서 수동 리뷰 또는 대체 오픈소스 도구를 임시 활용하고 임시 사용 사실을 스캔 결과 보고서에 기재한다.

### 7.3 교육 미이수자가 긴급 릴리스 일정으로 도구 운용을 수행해야 하는 경우

교육 미이수자가 불가피하게 도구를 운용해야 하는 경우, PSO 가 긴급 현장 교육(최소 2시간)을 수행하고 교육 기록을 즉시 작성한다. 이후 정식 교육 이수 일정을 부여한다. 긴급 현장 교육만으로 독립 운용 권한을 부여할 경우 PSO 가 결과를 교차 검토한다.

### 7.4 동일 도구의 복수 버전 병행 운용

레거시 제품과 신규 제품이 도구의 서로 다른 버전을 사용해야 하는 경우, 각 버전을 별도 항목으로 유효성 확인을 수행하고 승인 도구 목록에 버전별로 용도를 명시한다. 단일 도구 버전으로 통일이 가능한 경우 점진적 마이그레이션 계획을 수립한다.
