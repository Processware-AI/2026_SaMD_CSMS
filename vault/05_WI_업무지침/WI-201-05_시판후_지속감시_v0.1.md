# 시판 후 지속 감시 업무지침 (WI-201-05)

> 상위 절차: [[PRO-MDCS-201_사이버보안_리스크_관리_절차_v1.0]] · §5 단계 10~11

## 1. 업무 목적

SaMD-CSMS 제13조 제5호·제6호에 따라 제품 출시 이후 **보안 위협 동향·신규 CVE·공격 기법·규제 권고**를 지속 모니터링하고, 중대한 신규 위협이 탐지되었을 때 **14일 이내 긴급 재평가**를 트리거하며, **MSP(의료서비스 제공기관 등 운영 주체) 정보공유 채널**을 운영하여 현장 운용 중 발견된 보안 이슈를 수집·대응한다.

## 2. 수행 주체

- **주 수행자**: PSO (모니터링 총괄, 정보 입수·분류, 재평가 개시)
- **지원**: R&D (기술 분석·패치 검토), SecOps/CSIRT (침해 징후 모니터링)
- **MSP 채널 담당**: PSO, PM (운영 현장 연락 창구)
- **검토자**: PSC
- **승인자**: 오동석

## 3. 범위

- 출시된 모든 SaMD 제품 버전(단종·레거시 지원 기간 내 포함)
- 자사 및 제3자·오픈소스 구성요소(SBOM 등재 항목)
- 모니터링 대상 정보 채널: NVD·CVE·CERT·ICS-CERT·ISAC·공급사 보안 권고·학술·업계 보고서·MSP 현장 제보
- MSP 정보공유: 운영 현장에서 탐지된 이상 행동·침해 의심·패치 요청 등

## 4. 입력 자료 / 산출물

- **Input**
  - 현행 SBOM 및 CVE 추적 대장 (TMP-RISK-007)
  - NVD·CERT·ICS-CERT RSS/API 피드
  - 공급사 보안 공지·패치 릴리스 노트
  - MSP 현장 이슈 제보 (이메일·포털·핫라인)
  - 잔여 위험 수용 기록서 (TMP-RISK-004, 조건부 수용 항목 포함)
- **Output**
  - 월간·분기 시판 후 감시 리포트 (TMP-RISK-005)
  - 신규 위협 영향 분석 메모 (중대 발견 시 즉시 작성)
  - 긴급 재평가 트리거 기록 (해당 시)
  - MSP 정보공유 대응 이력 (REC-RISK-{YYYY}-{###})
  - CVE 추적 대장 업데이트 (TMP-RISK-007)

## 5. 수행 절차

### 5.1 사전 준비

1. PSO 는 모니터링 대상 정보 채널(NVD RSS, CERT 메일링 리스트, 공급사 보안 뉴스레터, ICS-CERT 의료기기 권고)을 구독 설정하고 입수 채널 목록을 유지한다.
2. SBOM 최신본을 분기별로 갱신하여 모니터링 기준 컴포넌트 목록을 최신 상태로 유지한다(PRO-MDCS-302 연계).
3. MSP 정보공유 채널(보안 핫라인 또는 포털)의 연락처와 에스컬레이션 경로를 제품 출하 전 MSP 에게 공지하고 이력을 유지한다.
4. 중대 위협 판단 기준(§5.2 B)을 PSC 사전 승인으로 문서화하여 재평가 트리거 기준을 명확히 한다.

### 5.2 수행 단계

#### A. 상시 모니터링

1. **자동 피드 수집**: NVD·CVE·CERT 피드를 자동으로 수집하고 현재 SBOM 에 등재된 컴포넌트와 매일 자동 비교한다. 신규 CVE 히트 시 즉시 PSO 에게 알림을 발송한다.
2. **수동 정보 수집**: PSO 는 주 1회 이상 ICS-CERT 의료기기 권고·ISAC(의료기기 정보공유 분석 센터)·주요 보안 미디어를 검토한다.
3. **공급사 권고 검토**: 분기별 공급사 보안 업데이트 요약을 검토하고 영향 여부를 판단한다.
4. **MSP 제보 수집**: MSP 로부터 수신된 보안 이슈 제보를 영업일 기준 2일 이내에 접수 확인하고 초기 영향 분류를 수행한다.

#### B. 신규 위협 분류 및 재평가 트리거 결정

1. 입수된 신규 위협·취약점을 다음 기준으로 분류한다.
   - **중대(Critical) 트리거 기준** (14일 이내 긴급 재평가):
     - CVSS Base Score ≥ 7.0 이고 자사 SBOM 컴포넌트에 직접 영향
     - 환자 안전에 직접 위해를 줄 수 있는 공격 시나리오가 공개
     - 자사 제품 또는 동종 의료기기를 표적으로 한 실제 공격 사례 보고
     - 규제 기관의 긴급 권고 또는 리콜 가이드라인 발표
   - **일반 트리거 기준** (연 1회 정기 재평가 통합):
     - CVSS < 7.0 이고 임상 영향 없음
     - 이미 적용된 통제로 영향이 충분히 완화됨
2. 중대 트리거 발생 시, PSO 는 24시간 이내에 PSC 에 긴급 재평가 개시를 보고하고 재평가 일정을 확정한다.

#### C. 긴급 재평가 수행 (중대 트리거 발생 시)

1. **영향 범위 분석 (D+1~3)**: PSO 는 R&D 와 협력하여 신규 위협이 자사 제품의 어떤 기능·컴포넌트·데이터에 영향을 미치는지 분석한다.
2. **위험 재산정 (D+4~7)**: WI-201-03(위험 평가 보고서 작성) 절차를 준용하여 신규 위협의 CVSS × 임상 영향 기반 위험 점수를 산정한다.
3. **통제 조치 검토 (D+7~10)**: 기존 통제의 충분성을 재검토하고 추가 통제가 필요한 경우 R&D 에 긴급 조치를 의뢰한다.
4. **잔여 위험 재수용 (D+10~14)**: 통제 후 잔여 위험을 WI-201-04 절차에 따라 PSC/CEO 에게 재수용 결정을 받는다.
5. **MSP 통보 (해당 시)**: 운영 중인 MSP 에 영향 및 권고 조치를 서면 통보한다.

#### D. 정기 재평가 (연 1회)

1. PSO 는 매년 4분기에 전 제품 버전을 대상으로 정기 재평가를 수행한다.
2. 당해 연도의 신규 위협·CVE·공격 기법을 종합하여 위험 평가를 갱신한다.
3. 재평가 결과를 종합 위험 보고서에 반영하고 PSC 에 보고한다.

#### E. MSP 정보공유 채널 운영

1. **제보 접수**: MSP 는 제품 내 이상 행동·보안 의심 사례를 핫라인(이메일 또는 포털) 으로 신고한다.
2. **초기 분류**: PSO 는 영업일 2일 이내에 제보를 분류하고 추가 정보 요청 여부를 MSP 에 회신한다.
3. **조사 및 대응**: 실제 보안 이슈로 확인 시 WI-201-02(취약점 평가) 또는 CSIRT 침해대응 절차로 연계한다.
4. **피드백 제공**: 조사 완료 후 MSP 에 결과 및 권고 조치를 통보하고 이력을 기록한다.
5. **익명화 집계**: 분기 감시 리포트에 MSP 제보 건수 및 유형을 익명화하여 포함한다.

### 5.3 완료 조건

- [ ] 월간 모니터링 활동이 수행되고 TMP-RISK-005 에 기록되었다
- [ ] 신규 CVE 가 SBOM 과 비교·분류되었고 TMP-RISK-007 이 갱신되었다
- [ ] 중대 트리거 발생 시 14일 이내 긴급 재평가가 완료되었다
- [ ] 긴급 재평가 결과에 따른 통제 조치 또는 잔여 위험 재수용이 기록되었다
- [ ] MSP 제보가 영업일 2일 이내 접수 확인되었고 이력이 관리되었다
- [ ] 연 1회 정기 재평가 보고서가 PSC 에 제출되었다
- [ ] 분기 감시 리포트가 PSC 에 배포되었다

## 6. 인터페이스 부서

- **R&D**: 영향 범위 분석·패치 개발·컴포넌트 업데이트
- **CSIRT**: 침해 의심 사례의 침해대응 절차 연계
- **Safety/QA**: 임상 영향 재평가 지원
- **PM**: MSP 연락처 유지, 제품 정보 제공
- **PSC**: 긴급 재평가 개시 승인, 분기 리포트 검토
- **규제/법무**: 규제 기관 권고 대응 전략 수립 지원

## 7. 주의사항 / 예외 처리

### 7.1 중대 트리거 발생 시 14일 내 재평가 완료 불가 상황

기술적 복잡성 또는 외부 전문가 부재로 14일 내 완료가 어려운 경우, PSO 는 즉시 PSC 에 보고하고 임시 완화 조치(서비스 격리·기능 비활성화 등)를 적용하면서 재평가 일정을 연장 요청한다. 연장 사유와 임시 조치 내역을 문서화하고 규제 기관 보고 의무(SaMD-CSMS 제18조) 해당 여부를 확인한다.

### 7.2 MSP 제보가 경쟁사 정보를 포함하거나 허위 신고일 가능성

MSP 제보가 자사 제품과 무관하거나 허위로 의심되는 경우, PSO 는 기술적 검증을 수행하여 근거 없음을 기록하고 MSP 에 회신한다. 악의적 허위 신고 패턴이 감지되면 법무팀에 통보한다. 진위 확인 전 공개 대응을 하지 않는다.

### 7.3 단종 또는 지원 종료 제품에 대한 신규 취약점 발견

지원 기간 종료(EOL) 제품에 대해 Critical 취약점이 발견된 경우, PSO 는 MSP 에 즉시 통보하고 마이그레이션을 권고한다. 자사가 통제할 수 없는 레거시 컴포넌트 취약점은 WI-201-04(잔여 위험 수용) 경로를 통해 CEO 수용 결정을 받는다. 필요 시 규제 기관에 보고한다.
