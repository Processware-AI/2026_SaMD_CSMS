# AI 데이터·모델 위협평가 업무지침 (WI-301-06)

> 상위 절차: [[PRO-MDCS-201_사이버보안_리스크_관리_절차_v1.0]] · §5 단계 3

## 1. 업무 목적

AI 적용 디지털의료기기의 데이터 중독·적대적 회피·모델 추출·반복 질의 조회 등 AI 특화 위협을 STRIDE 와 별도로 전개하여 식별·평가하고, Fail-safe·대안 운영 모드 등 AI 특화 통제를 검증한다.

## 2. 수행 주체

- **주 수행자**: PSO, R&D 리드
- **검토자**: AI Safety 리드, 임상 담당자, DPO
- **승인자**: CISO

## 3. 범위

- 학습 데이터(수집·저장·사용)
- 모델(학습·검증·배포)
- 추론 API·엣지 추론 엔진
- 재학습·모델 업데이트 파이프라인

## 4. 입력 자료 / 산출물

- **Input**
  - 모델 카드·데이터 시트
  - AI 아키텍처 도면·재학습 주기
- **Output**
  - [[TMP-RISK-003_AI_위협평가_워크시트_v0.1]]
  - Fail-safe 및 대안 운영 모드 설계 검증 결과

## 5. 수행 절차

### 5.1 사전 준비

1. PSO 가 AI 위협 카탈로그를 유지: 데이터 중독·회피·모델 추출·멤버십 추론·Prompt Injection(LLM).
2. AI Safety 리드가 모델의 임상 용도·실패 시 영향을 정리.
3. 데이터 보안 기준(수집·저장·전송) 을 [[WI-403-05_개인정보_비식별화_v0.1]] 및 [[WI-402-03_통신_암호화_v0.1]] 와 정합.

### 5.2 수행 단계

1. **데이터 공격 방어**
   - 수집 경로의 무결성 검증·필터링·정제·검증 프로세스.
   - 적대적 학습(Adversarial Training)·이상 탐지 적용.
2. **학습 데이터 보안**
   - 보안 채널 전송·소스 신뢰성 확인·해시 무결성·암호화·접근 권한·백업/복구.
3. **모델 추출·회피 방어**
   - 반복 질의 조회 차단, API 호출 제한, 적대적 공격 탐지 모델.
   - 모델 워터마킹·서명.
4. **Fail-safe 설계**
   - 문제 발생 시 피드백 화면 전환·이용자 안내 메시지·안전 중단·대안 운영 모드.
   - 임상 안전 모드(대체 알고리즘·수동 모드) 검증.
5. **프라이버시 공격 방어**
   - 멤버십 추론·속성 추론에 대한 DP-SGD 등 차등 프라이버시 평가.
6. **평가·기록**
   - 각 위협별 통제·잔여 위험을 [[TMP-RISK-003_AI_위협평가_워크시트_v0.1]] 에 기록.
   - [[WI-301-03_위험_평가_보고서_작성_v0.1]] 에 통합.

### 5.3 완료 조건

- [ ] 데이터 수집·저장·전송의 보안 통제가 정의·구현되었다
- [ ] 적대적 학습·이상 탐지가 평가되었다
- [ ] 모델 추출·회피 방어(반복 질의 차단·API 호출 제한) 가 적용되었다
- [ ] Fail-safe 화면 전환·대안 운영 모드가 검증되었다
- [ ] 멤버십 추론 등 프라이버시 공격 방어가 평가되었다
- [ ] AI 위협·통제·잔여 위험이 종합 보고서에 통합되었다

## 6. 인터페이스 부서

- **AI Safety**: 임상 용도·실패 영향 평가
- **임상 담당자**: 안전 모드 검증
- **DPO**: 데이터·프라이버시 통제
- **R&D**: 학습·배포 파이프라인 구현

## 7. 주의사항 / 예외 처리

### 7.1 데이터 중독 의심 탐지

학습 데이터에 중독 의심 샘플이 발견되면 해당 배치 재학습에서 제외하고, 소스 신뢰성 재평가. 필요 시 모델 롤백.

### 7.2 회피 공격 실시간 탐지

운영 중 적대적 입력 탐지 시 해당 요청 거부 + SecOps 알림. 누적 시도 초과 시 해당 IP·계정 제한.

### 7.3 재학습 주기 단축 압박

임상 성능 개선 요구로 재학습 주기를 단축하는 경우 각 사이클의 보안 평가가 가벼워질 위험. 재학습마다 최소 체크리스트 적용을 보장하고 분기 1회 전면 평가.

### 7.4 LLM 기반 기기의 Prompt Injection

LLM 기반 기기의 경우 Prompt Injection·탈옥 공격 방어를 별도 세션으로 평가. 입·출력 가드레일·콘텐츠 필터·시스템 프롬프트 보호 검증.
