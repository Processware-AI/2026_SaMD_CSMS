# SBOM 보호·보안 통신·저장 업무지침 (WI-302-03)

> 상위 절차: [[PRO-MDCS-302_SBOM_관리_절차_v1.0]] · §5 단계 3

## 1. 업무 목적

SBOM 파일은 디지털의료기기의 소프트웨어 구성을 완전히 노출하는 민감 자산이다. 본 지침은 SBOM의 **생성 후 디지털 서명, 전송 시 TLS 암호화, 안전 저장소 보관, 저장소 접근통제, 위변조 탐지·대응** 절차를 규정하여 SBOM 파일이 권한 없이 열람·수정·삭제되거나 전송 중 조작되지 않도록 보장한다.

## 2. 수행 주체

- **주 수행자**: SBOM 관리자
- **검토자**: Product Security Officer (PSO)
- **승인자**: CISO (접근통제 정책 변경·예외 승인)
- **협력**: 빌드 파이프라인 엔지니어 (서명 도구 통합), 인프라 팀 (저장소·TLS 구성)

## 3. 범위

- 빌드 파이프라인에서 생성된 모든 SPDX·CycloneDX 형식 SBOM 파일
- 내부 저장소(저장소 서버·오브젝트 스토리지·버전 관리 시스템)에 보관 중인 SBOM
- SBOM을 외부(MSP 포털, 이메일, SFTP 등)로 전송하는 모든 채널
- 침해 대응 또는 감사 목적으로 SBOM을 추출·조회하는 절차

## 4. 입력 자료 / 산출물

- **Input**
  - 빌드 파이프라인이 생성한 원본 SBOM 파일 (SPDX JSON 또는 CycloneDX XML/JSON)
  - 코드 서명용 비공개 키 (HSM 또는 KMS 관리)
  - 저장소 접근통제 매트릭스 (역할별 읽기·쓰기·삭제 권한)
- **Output**
  - 디지털 서명 및 해시(SHA-256)가 첨부된 SBOM 파일
  - 서명 로그 (서명 일시·서명자 식별자·서명 알고리즘)
  - 저장소 접근 로그 (접근자·조작 유형·일시)
  - 전송 이력 (수신자·채널·암호화 방식·전송 일시)

## 5. 수행 절차

### 5.1 사전 준비

1. SBOM 관리자는 서명에 사용할 비공개 키가 HSM(Hardware Security Module) 또는 클라우드 KMS에 안전하게 보관되어 있는지 확인한다. 키 파일을 로컬 디스크에 평문으로 저장하는 것은 금지한다.
2. 빌드 파이프라인 엔지니어와 함께 서명 단계(cosign, Sigstore, GPG 등)가 SBOM 생성 단계 직후에 자동 실행되도록 CI/CD 파이프라인을 구성한다.
3. SBOM 저장소 서버의 TLS 인증서 유효기간을 확인하고, 만료 30일 전 자동 갱신이 설정되어 있는지 검증한다.
4. 저장소 접근통제 매트릭스를 최신 상태로 유지한다: SBOM 관리자(읽기·쓰기), PSO(읽기), 감사자(읽기), 외부 제공 전용 서비스 계정(읽기). 삭제 권한은 SBOM 관리자 + CISO 이중 승인 방식으로 구성한다.

### 5.2 수행 단계

1. **디지털 서명 및 무결성 해시 생성**
   - 빌드 파이프라인이 SBOM 파일 생성을 완료하는 즉시, 서명 도구가 해당 파일의 SHA-256 해시를 계산하고 비공개 키로 서명한다.
   - 서명 결과물(서명 파일 또는 서명 내장 SBOM)과 원본 SBOM 파일을 함께 저장소에 업로드한다.
   - 서명 일시, 서명자 식별자(키 ID 또는 인증서 지문), 사용 알고리즘(RSA-PSS 4096 이상 또는 ECDSA P-256 이상)을 서명 로그에 기록한다.

2. **안전 저장소 보관**
   - 서명된 SBOM 및 서명 파일을 전용 SBOM 저장소(내부 오브젝트 스토리지 또는 아티팩트 레지스트리)에 저장한다.
   - 저장소는 저장 데이터 암호화(AES-256)를 적용하고, 버전 관리(덮어쓰기 금지·삭제 보호)를 활성화한다.
   - SBOM 파일명 규칙을 준수한다: `{제품명}_{버전}_{릴리스일자}_sbom.{확장자}`.

3. **저장소 접근통제 적용 및 검증**
   - 저장소 접근은 서비스 계정 토큰 또는 OIDC 기반 인증을 사용하며, 비밀번호 단독 접근은 허용하지 않는다.
   - 매월 1회 SBOM 관리자가 접근 로그를 검토하여 미인가 접근 또는 삭제 시도가 없었는지 확인한다.
   - 접근 로그는 최소 1년 보관하고, 감사 시 제출 가능한 형태로 유지한다.

4. **전송 시 TLS 암호화 적용**
   - 내부 시스템 간 SBOM 전송: TLS 1.2 이상 강제 적용. TLS 1.0/1.1은 저장소·API 서버 설정에서 비활성화한다.
   - 외부(MSP) 전송: WI-302-04에서 정의하는 포털 또는 SFTP 채널을 사용한다. 이메일 첨부 전송은 원칙적으로 금지하며, 불가피한 경우 AES-256 암호화 압축 파일로 만들고 비밀번호를 별도 채널로 전달한다.
   - 전송 완료 후 수신자의 서명 검증 결과(성공/실패)를 전송 이력에 기록한다.

5. **무결성 정기 검증**
   - 매월 1회 SBOM 관리자가 저장소 내 모든 SBOM 파일의 SHA-256 해시를 재계산하고 서명 시점의 해시와 비교한다.
   - 해시 불일치 파일이 발견되면 즉시 §7.1 위변조 탐지 대응 절차를 실행한다.

6. **키 수명 관리**
   - 서명 키는 24개월 주기로 교체한다. 교체 전 구 키로 서명된 SBOM에 대한 재서명 계획을 PSO와 협의한다.
   - 키 교체 이력(교체 일시·사유·신규 키 ID)을 별도 키 관리 대장에 기록한다.

### 5.3 완료 조건

- [ ] 모든 신규 릴리스 SBOM에 디지털 서명과 SHA-256 해시가 첨부되어 있다
- [ ] 서명 로그가 생성 일시·서명자·알고리즘을 포함하여 저장소에 보관된다
- [ ] 저장소가 AES-256 저장 암호화·덮어쓰기 금지·삭제 보호를 적용하고 있다
- [ ] 저장소 접근 로그가 1년 이상 보관되고 있으며, 최근 1개월 내 미인가 접근이 없음이 확인된다
- [ ] 모든 전송 채널이 TLS 1.2 이상을 사용하고 있으며 TLS 1.0/1.1은 비활성화되어 있다
- [ ] 월간 무결성 검증이 실행되고 결과 로그가 보관된다
- [ ] 이메일 직접 첨부 전송 이력이 없거나, 예외 발생 시 CISO 승인 기록이 있다

## 6. 인터페이스 부서

- **빌드 파이프라인 엔지니어**: CI/CD에 서명 단계 통합 및 유지
- **인프라 팀**: 저장소 서버 TLS 설정, 저장 암호화 구성, 접근 로그 수집 인프라
- **PSO**: 월간 접근 로그 검토 결과 보고 수신, 위변조 탐지 시 조사 지원
- **CISO**: 키 교체 계획 승인, 예외 전송 방식 승인
- **감사팀**: 접근 로그·서명 로그 감사 시 제출

## 7. 주의사항 / 예외 처리

### 7.1 위변조 탐지 대응

무결성 정기 검증 또는 실시간 알림을 통해 저장소 내 SBOM 파일의 해시 불일치가 탐지된 경우, SBOM 관리자는 즉시 해당 파일에 대한 외부 제공을 일시 차단하고 PSO에 보고한다. PSO는 접근 로그를 분석하여 변조 원인(내부 오조작 또는 보안 침해)을 판별한다. 보안 침해로 의심되는 경우 [[PRO-MDCS-502_침해사고_대응_절차]] 를 즉시 개시한다. 내부 오조작인 경우 마지막으로 확인된 정상 서명본을 복원하고 사고 원인을 시정 조치 보고서에 기록한다.

### 7.2 저장소 서비스 장애 시 긴급 제공

저장소 서비스 장애로 SBOM 파일 접근이 불가능한 상태에서 규제 기관 또는 MSP의 긴급 제공 요청이 발생하는 경우, SBOM 관리자는 마지막 정기 백업본에서 파일을 추출하여 암호화 압축 파일로 만들고 CISO 승인 후 제공한다. 긴급 제공 이력(수신자·일시·백업 버전·승인자)을 별도 로그에 기록하고, 저장소 복구 후 정상 채널 제공으로 전환한다.

### 7.3 레거시 서명 알고리즘 호환

외부 수신자 시스템의 기술적 제약으로 RSA-PSS 4096 또는 ECDSA P-256 검증이 불가능한 경우, PSO가 기술적 불가 사유를 서면으로 확인하고 CISO가 하위 알고리즘 사용을 예외 승인할 수 있다. 예외는 최대 6개월로 제한하며, 수신자 시스템 업그레이드 일정을 계약서에 명시한다.

## 8. 연계 템플릿 / 기록

- 기록 폴더: `08_REC_기록/` (REC-SBOM-{YYYY}-{###})
- 서명 로그: SBOM 저장소 내 `signature-log/` 디렉터리
- 접근 로그: 인프라 팀 중앙 로그 저장소
- 전송 이력: SBOM 관리자 대장 (스프레드시트 또는 JIRA 티켓)
- 연계 지침: [[WI-302-04_SBOM_MSP_제공_계약_포털_v0.1]] (외부 전송 채널 상세)
