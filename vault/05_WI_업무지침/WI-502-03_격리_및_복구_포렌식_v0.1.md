# 격리·복구·포렌식 업무지침 (WI-502-03)

> 상위 절차: [[PRO-MDCS-502_침해사고_대응_및_신고_절차_v1.0]] · §5 단계 4, 8, 9

## 1. 업무 목적

침해 확산을 조기 봉쇄하고 포렌식 증거의 법적 효력을 유지한 채 확보하며, 클린 상태로 복구하여 제품 사용 연속성과 사후 분석·재발 방지의 근거를 확보한다.

## 2. 수행 주체

- **주 수행자**: CSIRT 포렌식 분석관, 인프라팀
- **검토자**: CISO
- **승인자**: CEO (위기 등급 격리 결정)

## 3. 범위

- 영향 자산 네트워크 격리
- 악성코드 제거 및 취약점 패치
- 클린 이미지 기반 재구성
- 포렌식 증거 체인 오브 커스터디(Chain of Custody)

## 4. 입력 자료 / 산출물

- **Input**
  - 초기 분석 보고서·등급 판정서
  - 침해 티켓·증거 보전 자료
- **Output**
  - 격리 기록(방화벽·EDR·NAC 조치)
  - [[TMP-IR-004_포렌식_분석보고서_v0.1]]
  - 복구 완료 확인서·무결성 검증 결과

## 5. 수행 절차

### 5.1 사전 준비

1. CSIRT 리더가 격리 결정 권한 매트릭스를 유지한다(경미: CSIRT 리더, 중대 이상: CISO).
2. 체인 오브 커스터디 양식·봉인 장비를 준비한다.
3. 클린 이미지 저장소와 복구 자동화 스크립트를 유지한다.

### 5.2 수행 단계

1. **격리 결정·실행**
   - 네트워크 격리: 방화벽 차단, NAC 격리 VLAN 전환, EDR 엔드포인트 격리.
   - 계정 격리: 탈취 의심 계정 강제 로그아웃·비활성화.
   - 프로세스 격리: 악성 프로세스 종료·격리.
2. **포렌식 확보**
   - 디스크 이미지(비트 단위) 와 메모리 덤프를 해시와 함께 보전.
   - 네트워크 흐름(Netflow·PCAP) 확보.
   - 체인 오브 커스터디 서명·봉인.
3. **분석**
   - 악성코드 분석, 공격 경로 재구성(MITRE ATT&CK 매핑).
   - SBOM 기반 영향 컴포넌트 식별 ([[WI-302-05_SBOM_침해대응_연계_v0.1]]).
4. **클린 복구**
   - 감염 자산을 클린 이미지로 재구성.
   - 인증 정보·키·토큰 교체(키 로테이션 [[WI-402-04_암호키_수명주기_v0.1]] 연계).
   - 패치 적용 후 취약점 재스캔.
5. **무결성 재검증**
   - FIM 해시·서명 검증.
   - 복구 후 72시간 집중 모니터링.
6. **해제·복귀**
   - 격리 해제는 CISO 승인 후 실시.
   - 서비스 복귀 전 사용자·MSP 통지.

### 5.3 완료 조건

- [ ] 등급에 맞는 격리 결정권자 승인 아래 격리가 실행되었다
- [ ] 네트워크·계정·프로세스 격리 조치가 모두 완료되었다
- [ ] 포렌식 증거(디스크·메모리·네트워크) 가 해시·봉인과 함께 보전되었다
- [ ] SBOM 기반 영향 컴포넌트 식별이 완료되었다
- [ ] 클린 이미지 재구성·키/토큰 교체·재스캔이 완료되었다
- [ ] FIM 재검증과 72시간 집중 모니터링이 진행된다
- [ ] 격리 해제·서비스 복귀 전 CISO 승인과 통지가 수행되었다

## 6. 인터페이스 부서

- **인프라팀**: 네트워크 격리·복구 실행
- **SecOps**: EDR·SIEM 연동·모니터링
- **법무**: 증거 처리 법적 자문
- **SBOM 관리자**: 영향 컴포넌트 식별 지원
- **홍보/CS**: 복귀 통지 공지

## 7. 주의사항 / 예외 처리

### 7.1 격리가 환자 안전에 영향

의료기기의 즉시 격리가 치료 중단으로 환자 안전에 영향을 주는 경우, Safety 리드·임상 담당자가 참여하여 격리 범위를 세분화(서비스별 격리) 하고 환자 이관 계획을 병행 수립한다.

### 7.2 포렌식 법적 대응 (민·형사)

수사·소송으로 이어질 가능성 있는 사건은 법무가 외부 포렌식 업체(법정 공인) 를 지정하여 체인 오브 커스터디를 강화한다. 증거 원본은 봉인 후 외부 업체로 이관.

### 7.3 복구 지연(대규모 감염)

대규모 랜섬웨어 등으로 복구가 RTO 를 초과할 것이 예상되면 BCP·Fail-safe ([[WI-502-04_BCP_및_Fail-safe_v0.1]]) 를 발동하고, 우선순위 서비스부터 순차 복구.

### 7.4 재감염 방지

복구 완료 후 72시간 내 재감염 탐지 시 해당 복구 절차의 결함(잔여 악성코드·미패치 취약점) 을 분석하고 전체 복구 재수행한다. 근본 원인은 [[WI-502-06_재발방지_및_사후분석_v0.1]] 로 이관.
