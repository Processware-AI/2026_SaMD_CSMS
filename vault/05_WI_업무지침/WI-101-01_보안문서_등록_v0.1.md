# 보안문서 등록 업무지침 (WI-101-01)

> 상위 절차: [[PRO-MDCS-101_보안_문서화_관리_절차_v1.0]] · §5 단계 3~6
> 연계: [[GS-PRO-QMS-102_문서_개정_관리_절차]]

## 1. 업무 목적

SaMD-CSMS 제2조-2 문서 목록에 근거한 신규 보안 문서(POL·PRO·WI·TMP·EX)가 표준 번호체계·템플릿·승인 경로를 준수하여 등록되도록 하고, MAT-001 문서관리대장에 일관된 메타데이터로 반영한다.

## 2. 수행 주체

- **주 수행자**: 해당 문서의 Doc Owner (개발·SecOps·CSIRT 등 부서의 작성 담당자)
- **검토자**: Product Security Officer (PSO), QMR
- **승인자**: POL/PRO 는 오동석, WI/TMP 는 PSO (경미) 또는 PSC (중대)

## 3. 범위

본 지침은 PRO-MDCS-101 §5 의 **단계 3(초안 작성) ~ 단계 6(배포·등록)** 에 적용한다. 제2조-2 문서 목록에 이미 등록된 기존 문서의 개정은 [[WI-101-02_문서_갱신_및_폐기_v0.1]] 를 따르고, 증적 수집은 [[WI-101-03_증적_수집_기준_v0.1]] 를 참조한다.

## 4. 입력 자료 / 산출물

- **Input**
  - SaMD-CSMS 제2조-2 문서 목록 (부속서)
  - PSO 가 작성한 문서 Gap 리스트 (PRO-MDCS-101 §5 단계 1 산출)
  - 표준 번호체계 `vault/00_공통관리/02_문서번호체계.md` (MDCS 영역)
  - 템플릿 `vault/99_템플릿/T01~T07`
- **Output**
  - 승인된 신규 문서 (POL/PRO/WI/TMP/EX) 한 편
  - [[TMP-DOC-001_보안문서_등록신청서_v0.1]] 제출본
  - MAT-001 신규 Row
  - 배포 공지 기록

## 5. 수행 절차

### 5.1 사전 준비

1. 작성자가 제2조-2 문서 목록에서 자신이 담당하는 항목의 doc_id 와 제목을 확정한다.
2. PSO 에게 문서 유형(POL/PRO/WI/TMP/EX) 및 번호 예약을 요청한다. PSO 는 중복을 확인한 뒤 번호를 회신한다.
3. 작성자가 해당 유형의 템플릿(T03/T04/T05/T06/T07)을 복사하여 초안 파일을 생성한다.
4. 상위 문서 링크(parent_policy 또는 parent_pro) 를 frontmatter 에 기입한다.

### 5.2 수행 단계

1. **초안 작성**
   - 템플릿의 모든 섹션을 채운다. 공란은 `<미정>` 으로 표기하지 않고, 미정 사유를 본문에 기술한다.
   - source_citation 블록에 `_inputs/` 경로와 로케이터를 YAML 로 명시한다.
   - 표준 원문의 20단어 이상 연속 복사를 하지 않는다. paraphrase 로 작성한다.
2. **등록 신청서 제출**
   - [[TMP-DOC-001_보안문서_등록신청서_v0.1]] 를 작성한다.
   - 첨부: 초안 파일 경로, 출처 목록, 영향 받는 기존 문서 리스트.
3. **QMR 형식 검토**
   - QMR 이 frontmatter 필수 필드, 템플릿 준수, 번호체계 정합성을 2 영업일 이내 검토한다.
   - 반려 시 사유를 등록 신청서에 기입하여 작성자에게 반환한다.
4. **PSO 기술 검토**
   - PSO 가 보안 요구사항 커버리지, Req-ID 매핑(§8 표준 매핑), 상·하위 문서 연계를 검토한다.
   - 중대 문서(POL·영역 횡단 PRO) 는 PSC 심의로 에스컬레이션한다.
5. **승인 및 배포**
   - 승인자가 전자결재 또는 PCB 회의록에 승인을 기록한다.
   - 작성자가 파일을 최종 저장하고 Git 커밋 메시지를 `docs: register {doc_id} v{version}` 형식으로 작성한다.
6. **MAT-001 등록**
   - 작성자가 MAT-001 에 `문서번호 / 유형 / 제목 / 버전 / Owner / 상태 / 최종 갱신 / 보관기간` 컬럼을 채워 Row 를 추가한다.
   - 배포 공지문을 사내 채널에 게시한다.

### 5.3 완료 조건

- [ ] frontmatter 의 doc_id·type·version·owner·parent 필드가 모두 채워졌다
- [ ] source_citation 블록에 최소 1건의 `_inputs/` 근거가 포함되었다
- [ ] 표준 매핑 섹션에 최소 1건의 Req-ID 가 연결되었다
- [ ] QMR 형식 검토 및 승인자 승인 기록이 확보되었다
- [ ] MAT-001 에 신규 Row 가 추가되었다
- [ ] 배포 공지가 사내 채널에 게시되었다

## 6. 인터페이스 부서

- **QMR**: 문서 형식·버전·번호 일관성 검토
- **IT팀**: Git/EDMS 저장소 운영 및 접근권한 부여
- **커뮤니케이션팀**: 사내 공지 채널 운영
- **RA(규제 담당)**: 식약처 요구 대응 시 문서 외부 제공 협의

## 7. 주의사항 / 예외 처리

### 7.1 긴급 등록 (Hot-registration)

식약처 현장 심사 또는 침해사고 대응 중 신규 문서 즉시 등록이 요구되는 경우, 작성자는 PSO 구두 승인 아래 초안 상태로 배포한 뒤 48시간 이내 정식 절차를 소급 보완한다. MAT-001 에는 `status: draft-emergency` 로 등록하고 정식 승인 완료 시 상태를 갱신한다.

### 7.2 번호 충돌

동일 유형·상위 PRO 하위에서 작성자가 동시 등록을 시도해 번호가 겹치는 경우, PSO 가 타임스탬프 순서로 선행 작성자에게 예약 번호를 우선 부여하고 후행 작성자는 다음 번호로 이동한다.

### 7.3 `_inputs/` 근거 부재

표준 원문이 아직 `_inputs/01_표준원문/` 에 배치되지 않은 항목(예: 해설서 의존 WI)은 source_citation 에 `[_inputs 미제공 — LLM 추정]` 꼬리표를 붙이고 등록한다. 근거 배치 완료 시 WI-101-02(갱신) 로 보완한다.

### 7.4 템플릿 부적합

기존 T03~T07 로 표현이 어려운 AI 특화·규제 특화 문서는 작성자가 템플릿 변경 요청을 `vault/99_템플릿/_issues/` 에 등록하고, QMR 승인 후 임시 템플릿으로 등록한다. 분기별 템플릿 리뷰에서 정식화 여부를 결정한다.
