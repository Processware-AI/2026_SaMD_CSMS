# MFA 및 패스워드 정책 업무지침 (WI-401-03)

> 상위 절차: [[PRO-MDCS-401_접근통제_및_인증_관리_절차_v1.0]] · §5 단계 4

## 1. 업무 목적

디지털의료기기 관리자 콘솔·개발자 저장소·운영 인프라에 대한 계정 탈취를 예방하기 위해, 다단계 인증(MFA) 의 적용 대상·수단 우선순위, 강력한 패스워드 정책, 로그인 실패 잠금 조건을 조직 단일 기준으로 규정한다.

## 2. 수행 주체

- **주 수행자**: IAM 관리자
- **검토자**: 오동석, 시스템 소유자
- **승인자**: 오동석

## 3. 범위

- 특권·관리자·원격 계정 전체 (MFA 필수)
- 일반 임직원 계정 (패스워드 정책 필수, MFA 권장)
- 의료기기 장치 인증(인증서 기반 MFA 보완)
- MSP·외주 협력사 계정

## 4. 입력 자료 / 산출물

- **Input**
  - 계정 인벤토리 ([[WI-401-01_계정_생명주기_v0.1]] 산출)
  - [[TMP-ACC-003_MFA_등록대장_v0.1]] 템플릿
- **Output**
  - MFA 적용 계정 목록·등록 대장
  - 패스워드 정책 설정 기록(IAM 시스템·AD·IDP)
  - 잠금·해제 로그

## 5. 수행 절차

### 5.1 사전 준비

1. IAM 관리자가 조직의 MFA 수단 우선순위를 확정한다. 1순위는 FIDO2/WebAuthn 보안키, 2순위는 OTP 앱(TOTP), 3순위는 SMS(긴급·보조만).
2. 패스워드 정책 기준을 확정한다(§5.2 의 규격 참조).
3. 잠금 정책 기준(실패 5회 → 15분 잠금, 15회 초과 → 관리자 수동 해제) 을 확정한다.

### 5.2 수행 단계

1. **MFA 대상 선정**
   - 특권·관리자·원격·클라우드 콘솔·SaaS 관리 계정을 MFA 필수 대상으로 지정한다.
   - 일반 임직원 계정은 최소 권장 대상, 의료정보 접근 계정은 필수로 격상한다.
2. **MFA 등록**
   - 사용자가 IAM 포털에서 1순위 수단 등록을 시도한다. 기기 미보유 시 OTP 앱을 등록한다.
   - 백업 수단(백업 코드 10개) 을 생성하고 사용자가 안전 저장한다.
   - IAM 관리자가 [[TMP-ACC-003_MFA_등록대장_v0.1]] 에 등록 상태를 기록한다.
3. **패스워드 정책**
   - 최소 길이: 12자 (특권 계정 16자).
   - 조합: 영문 대소·숫자·특수문자 3종 이상.
   - 금지: 사용자 식별자 포함, 최근 5개 재사용, 유출 사전(HIBP) 매칭.
   - 만료: 90일 (특권 60일). 단 MFA 와 결합 시 만료 주기를 180일까지 연장 가능.
4. **로그인 실패 잠금**
   - 연속 5회 실패 → 15분 자동 잠금.
   - 24시간 내 누적 15회 초과 → 계정 관리자 수동 해제 + 사용자 확인 절차.
   - 잠금 이벤트는 SIEM 에 전달하여 계정 탈취 시도 탐지에 활용한다.
5. **MFA 우회·면제**
   - 특수 장비(레거시 시스템) 우회는 오동석 승인 + 보완 통제(IP 화이트리스트·추가 로그) 필수.
   - 면제 계정 목록은 분기마다 오동석 재검토.
6. **교육·공지**
   - 신규 입사자 온보딩 1일차에 MFA 등록을 완료하도록 안내한다.
   - 피싱 내성 강화를 위해 분기마다 MFA 모의 훈련을 시행한다.

### 5.3 완료 조건

- [ ] 특권·관리자·원격 계정의 MFA 적용률이 100% 이다
- [ ] MFA 등록 대장이 최신 상태로 유지된다
- [ ] 백업 코드가 사용자에게 발급되었고 재발급 이력이 기록된다
- [ ] 패스워드 복잡도·만료 정책이 IAM 시스템에 설정·검증되었다
- [ ] 로그인 실패 잠금·해제 로그가 SIEM 으로 전달된다
- [ ] MFA 우회·면제 계정 목록이 오동석 승인과 일치한다

## 6. 인터페이스 부서

- **오동석**: MFA 면제·우회 승인
- **시스템 소유자**: 각 시스템의 MFA 연동 구현
- **SecOps**: 잠금·우회 시도 SIEM 분석
- **HR**: 온보딩·오프보딩 시 MFA 등록·해제 연동

## 7. 주의사항 / 예외 처리

### 7.1 MFA 기기 분실

사용자가 MFA 기기를 분실한 경우 백업 코드로 재등록을 수행한다. 백업 코드도 분실한 경우, 사용자가 대면 또는 영상 ID 확인을 거쳐 IAM 관리자가 임시 코드를 발급하고 72시간 이내 정식 재등록을 완료한다.

### 7.2 SMS 의존 금지 계정

금융·의료정보 접근 계정 또는 특권 계정은 SIM 스와핑 리스크로 SMS 를 주 수단으로 사용하지 않는다. 부득이한 경우 오동석 승인 필수.

### 7.3 긴급 인증 장애 (IDP 장애)

IDP 장애로 MFA 인증이 불가한 경우, 사전 지정된 비상 관리자 계정(최소 2인 승인 필요)으로 로컬 직접 접근을 수행하며, 모든 작업은 세션 녹화한다. 복구 후 세션 녹화본을 오동석 가 감사한다.

### 7.4 패스워드 유출 탐지

HIBP 또는 다크웹 모니터링에서 임직원 이메일·패스워드 조합이 유출 확인된 경우, IAM 관리자가 해당 계정 패스워드를 강제 만료하고 사용자에게 24시간 이내 재설정을 요구한다.
