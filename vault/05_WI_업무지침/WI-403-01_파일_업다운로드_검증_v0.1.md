# 파일 업·다운로드 검증 업무지침 (WI-403-01)

> 상위 절차: [[PRO-MDCS-403_데이터_및_파일_보안_관리_절차_v1.0]] · §5 단계 2

## 1. 업무 목적

디지털의료기기가 처리하는 파일의 업로드·다운로드 구간에서 악성코드 유입·경로 조작·위·변조된 파일 전달을 예방하도록, 확장자 화이트리스트·매직넘버 검증·크기 제한·안티바이러스 스캔·해시 검증의 최소 통제 수준을 정의한다.

## 2. 수행 주체

- **주 수행자**: 개발팀 개발자
- **검토자**: QA 리드, PSO
- **승인자**: 오동석 (릴리스 보안 게이트에서)

## 3. 범위

- 사용자·MSP 가 업로드하는 의료 영상·보고서·설정 파일
- 기기가 다운로드 받는 업데이트 파일·참조 데이터
- API 기반 파일 송수신

## 4. 입력 자료 / 산출물

- **Input**
  - 파일 처리 요구사항(기능 명세)
  - 허용 확장자·매직넘버 목록
  - AV 엔진·해시 라이브러리
- **Output**
  - 파일 검증 로직 구현 코드
  - [[TMP-SDLC-001_파일검증_설계서_v0.1]] 작성본
  - 단위·통합 테스트 케이스(악성 파일 샘플 대응)

## 5. 수행 절차

### 5.1 사전 준비

1. 개발팀이 제품에서 처리 가능한 파일 확장자 화이트리스트(예: `.dcm`, `.pdf`, `.png`, `.jpg`, `.hl7`) 를 문서화한다.
2. 매직넘버(파일 시그니처) 매핑 테이블을 유지한다(예: PDF=25 50 44 46).
3. 크기 제한을 유스케이스별로 정의한다(예: DICOM 단일 파일 1GB, 보고서 50MB).
4. AV 엔진 라이선스·정의 파일 자동 갱신 설정을 확인한다.

### 5.2 수행 단계

1. **업로드 전 클라이언트 검증**
   - 클라이언트에서 확장자·크기 사전 확인(사용자 편의)을 수행하되, 이를 보안 신뢰 기준으로 삼지 않는다.
2. **서버 검증(필수)**
   - 확장자 화이트리스트 검사.
   - 매직넘버 검증(파일 상단 바이트 비교). 확장자와 매직넘버 불일치 시 거부.
   - 크기 제한 검사.
   - 파일명 경로 조작 방지(`../`, 절대 경로, 유니코드 제어 문자 차단).
3. **AV 스캔**
   - 업로드 파일을 임시 격리 저장소에 저장 후 AV 엔진으로 스캔한다.
   - 스캔 결과 악성 확인 시 즉시 삭제·로그·SIEM 이벤트 발행.
4. **정상 처리**
   - 검증 통과 파일만 정식 저장소로 이동한다.
   - 저장 시 파일명을 해시·UUID 기반으로 재명명한다(원본명은 메타데이터로 보관).
5. **다운로드 검증**
   - 배포되는 파일에 대해 SHA-256 해시를 함께 제공한다.
   - 클라이언트가 다운로드 후 해시 재계산·비교를 수행한다.
   - 업데이트 파일의 경우 디지털 서명 검증을 병행한다([[WI-601-05_패치_및_업데이트_배포_v0.1]] 연계).
6. **보안 테스트**
   - QA 는 악성 파일 샘플(EICAR·매직넘버 조작) 로 바이패스 시도 테스트를 수행한다.

### 5.3 완료 조건

- [ ] 확장자 화이트리스트·매직넘버·크기 검증이 서버 측에 구현되었다
- [ ] 경로 조작 방지 로직이 구현·테스트되었다
- [ ] AV 스캔이 업로드 파이프라인에 통합되었다
- [ ] 다운로드 파일에 SHA-256 해시 및 필요 시 서명이 제공된다
- [ ] QA 악성 파일 바이패스 테스트가 모두 통과되었다
- [ ] 검증 로그(성공·실패) 가 SIEM 으로 전달된다

## 6. 인터페이스 부서

- **QA**: 악성 파일 테스트 케이스 설계·실행
- **SecOps**: AV 엔진 운영, SIEM 연동
- **PSO**: 릴리스 게이트에서 검증 로직 리뷰

## 7. 주의사항 / 예외 처리

### 7.1 대용량 의료 영상 (DICOM 시리즈)

수 GB 단위 DICOM 시리즈 업로드는 청크 분할 업로드 사용 시 각 청크·전체에 대한 해시를 모두 검증한다. AV 스캔은 재조립 후 수행.

### 7.2 압축 파일 (ZIP·TAR)

압축 파일은 해제 후 내부 파일 각각을 재귀적으로 검증한다. ZIP 폭탄(Zip Bomb) 방지를 위해 해제 후 총 크기 한도를 설정(예: 5GB) 하고 초과 시 거부한다.

### 7.3 AV 엔진 장애

AV 엔진 장애로 스캔이 불가한 경우, 업로드 기능 전체를 일시 차단하고 대체 엔진으로 전환하거나 격리 큐에 적재한 뒤 복구 후 일괄 스캔한다. 환자 안전 영향 가능 시 오동석 가 임시 정책을 결정.

### 7.4 암호화된 파일

사용자가 암호로 보호된 ZIP 을 업로드한 경우, AV 스캔이 불가하므로 정책에 따라 업로드를 거부하거나 별도 격리 구역으로 이동 후 사용자에게 암호 제공 요청 프로세스를 안내한다.
