# 보안 코딩 표준 업무지침 (WI-201-02)

> 상위 절차: [[PRO-MDCS-301_보안_SDLC_절차_v1.0]] · §5 단계 5

## 1. 업무 목적

SaMD 개발에 사용되는 언어·프레임워크별 보안 코딩 규칙을 명확화하여 OWASP Top 10/CWE Top 25 취약점의 소스 단계 삽입을 예방하고, 재사용 가능한 안전 라이브러리 사용을 정착시킨다.

## 2. 수행 주체

- **주 수행자**: 개발자 전원
- **검토자**: R&D 리드, PSO, 보안 Champion
- **승인자**: VP of R&D

## 3. 범위

- 사용 언어: C/C++, Java, Kotlin, Python, JavaScript/TypeScript, Go, Rust 등
- 프레임워크: Spring, Django, FastAPI, React/Next.js, Angular 등
- 공통 라이브러리: 암호화·입력검증·인증 라이브러리

## 4. 입력 자료 / 산출물

- **Input**
  - OWASP Top 10·CWE Top 25·CERT 표준
  - 내부 라이브러리 카탈로그
- **Output**
  - 언어·프레임워크별 보안 코딩 규칙 문서
  - 금지·권장 함수 목록

## 5. 수행 절차

### 5.1 사전 준비

1. R&D 리드가 언어별 Champion 을 지정한다.
2. 내부 위키에 보안 코딩 규칙 섹션을 설치한다.

### 5.2 수행 단계

1. **공통 규칙**
   - 입력 검증은 화이트리스트 우선.
   - SQL: Prepared Statement/ORM 파라미터 바인딩 필수.
   - 비밀 정보(키·패스워드) 하드코딩 금지. Vault/KMS 사용.
   - 에러 메시지에 시스템 내부 정보 노출 금지.
   - 모든 출력은 컨텍스트 인코딩(HTML·URL·JS).
2. **메모리 안전**
   - C/C++ 신규 코드는 `unique_ptr`·`shared_ptr`·`std::array` 등 현대 C++ 패턴 사용.
   - 경계 검사 생략 금지. `strcpy`·`sprintf`·`gets` 등 금지.
   - 가능하면 Rust·Go 등 메모리 안전 언어로 구현.
3. **암호화**
   - 자체 암호 구현 금지. 검증된 라이브러리(OpenSSL·libsodium·BouncyCastle) 사용.
   - 승인된 알고리즘([[WI-402-04_암호키_수명주기_v0.1]]) 만 사용.
4. **의존성 관리**
   - 새 의존성 도입 시 보안·라이선스 검토 필수.
   - Dependabot·Renovate 로 자동 갱신.
5. **코드 리뷰**
   - 모든 PR 에 보안 체크리스트를 포함한 리뷰 필수.
   - Champion 또는 PSO 가 High-risk 변경을 2차 리뷰.
6. **교육·인증**
   - 분기별 보안 코딩 교육. 신입 개발자 필수 이수.

### 5.3 완료 조건

- [ ] 언어·프레임워크별 보안 코딩 규칙이 문서화되었다
- [ ] 금지 함수·권장 라이브러리 목록이 유지된다
- [ ] 모든 PR 에 보안 체크리스트 리뷰가 적용된다
- [ ] High-risk 변경은 Champion/PSO 2차 리뷰를 거친다
- [ ] 의존성 자동 갱신이 작동한다
- [ ] 분기 보안 코딩 교육이 실시된다

## 6. 인터페이스 부서

- **QA**: SAST 룰과 정합성 유지
- **PSO**: High-risk 리뷰
- **인사**: 교육 이수 관리

## 7. 주의사항 / 예외 처리

### 7.1 레거시 코드

규칙 소급 적용 시 대규모 리팩터링 필요한 경우, 신규 코드 우선 적용·베이스라인 설정. 12개월 로드맵으로 점진 개선.

### 7.2 성능 제약

안전 라이브러리 사용이 성능 요구를 충족 못하는 경우 PSO 예외 승인 하에 대안 구현하되, 추가 테스트·감사 필수.

### 7.3 금지 함수 불가피 사용

외부 시스템 연동 등으로 금지 함수 사용이 불가피한 경우 예외 주석(`// nosec: reason`) 과 PSO 승인 기록 필수. 분기 리뷰에서 재평가.

### 7.4 규칙 충돌 (팀 간 표준 차이)

팀 간 규칙 충돌 발생 시 R&D 리드가 통합 원칙을 정하고 위키에 명시. Champion 협의체 분기 1회 운영.
