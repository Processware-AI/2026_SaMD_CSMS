# 홈페이지 공개 및 조정 공개 업무지침 (WI-601-04)

> 상위 절차: [[PRO-MDCS-601_취약점_감시_공개_조치_절차_v1.0]] · §5 단계 9~10

## 1. 업무 목적

확인·대응된 취약점을 공개 채널을 통해 투명하게 공지하여 MSP·이용자·보안 커뮤니티가 동일한 정보를 갖도록 하며, 해결 지연·악용 위험 상황에서는 식약처와 협의하여 공개 시점을 조정한다.

## 2. 수행 주체

- **주 수행자**: PSO
- **검토자**: RA 리드, 법무, 홍보/CS
- **승인자**: 오동석 (일반), CEO (위기)

## 3. 범위

- 홈페이지 보안 공지 페이지 운영
- 고급 공지(Advisory·CVE 발행 요청)
- 공개 시점 조정 협의(식약처)

## 4. 입력 자료 / 산출물

- **Input**
  - 취약점 분석 결과·패치 배포 상태
  - 식약처 통보 접수번호
- **Output**
  - [[TMP-VUL-003_취약점_공개자료_v0.1]]
  - 공개 URL·CVE ID(해당 시)

## 5. 수행 절차

### 5.1 사전 준비

1. 홈페이지 보안 공지 페이지(영문 병기) 를 운영하고 RSS·이메일 구독 채널을 제공한다.
2. CVE Numbering Authority(CNA) 또는 MITRE 협력 경로를 확보한다.
3. 공개 자료 양식을 [[TMP-VUL-003_취약점_공개자료_v0.1]] 로 표준화.

### 5.2 수행 단계

1. **공개 자료 작성**
   - 제목·영향 제품·취약점 요약·심각도(CVSS)·권고 조치·패치 제공·문의 채널 포함.
   - 기술적 세부(exploit 경로) 는 악용 위험 고려하여 축약.
2. **식약처 통보 연계**
   - [[WI-601-03_식약처_MSP_통보_v0.1]] 접수번호를 공개 자료에 참조 표기.
3. **시점 결정**
   - 원칙: 패치 제공일 이후 공개.
   - 조정 필요 시 식약처와 협의(해결 지연·심각도 고려). 협의 결과를 기록.
4. **CVE 발급(해당 시)**
   - CNA 프로세스로 CVE ID 발급 요청.
5. **게시·전파**
   - 홈페이지 게시 → RSS/메일 구독자 전파.
   - 소셜·파트너 포털 병행 공지.
6. **후속 관리**
   - 신규 정보(패치 업데이트·추가 영향) 발견 시 자료 개정 및 개정 이력 표기.

### 5.3 완료 조건

- [ ] 공개 자료에 상세·영향·대응 3대 요소가 포함되었다
- [ ] 기술 세부가 악용 위험을 고려해 적정 수준으로 요약되었다
- [ ] 식약처 접수번호가 참조 표기되었다
- [ ] 조정 공개 시점을 식약처와 협의한 기록이 남았다
- [ ] CVE ID(해당 시) 가 발급·기재되었다
- [ ] 홈페이지 게시 및 구독 채널 전파가 수행되었다

## 6. 인터페이스 부서

- **RA**: 규제기관 협의 주관
- **법무**: 공개 문구 법적 리스크 검토
- **홍보/CS**: 사용자 문의 1차 응대 준비
- **CNA/MITRE**: CVE 등록

## 7. 주의사항 / 예외 처리

### 7.1 패치 미확보 상태 공개

패치 미확보로 조정 공개가 필요할 때, 임시 완화(구성 변경·네트워크 차단) 지침을 공개 자료에 포함한다. 식약처 협의 아래 공개 시점을 결정.

### 7.2 외부 연구자 공개 마감일

연구자가 90일 공개 마감일을 설정한 경우, 그 기한 내 패치·공개 가능하도록 역산 계획을 수립한다. 불가 시 연구자와 연장 협의.

### 7.3 허위·오보 유통

공개 자료 오류·허위 유통 발견 시 즉시 정정 공지를 상단에 표시하고 개정 이력에 상세 기록. 기존 URL 은 유지해 링크 무효화 방지.

### 7.4 공개 후 악용 급증

공개 후 악용 시도가 급증하면 [[PRO-MDCS-502_침해사고_대응_및_신고_절차_v1.0]] IR 루틴과 병행 대응하고, 공개 자료에 임시 완화 강화 조치를 추가 반영.
