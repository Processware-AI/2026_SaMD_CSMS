# 계정 생명주기 관리 업무지침 (WI-401-01)

> 상위 절차: [[PRO-MDCS-401_접근통제_및_인증_관리_절차_v1.0]] · §5 단계 1~3, 7

## 1. 업무 목적

사용자·서비스 계정의 생성·변경·삭제·주기적 권한 검토를 RBAC 와 최소권한 원칙에 따라 일관되게 운영함으로써 비인가 접근·불용 계정·과다 권한으로 인한 보안 사고를 예방한다.

## 2. 수행 주체

- **주 수행자**: IAM 관리자
- **검토자**: 라인매니저(업무 필요성), 시스템 소유자(기술 영향)
- **승인자**: 특권/관리자 계정은 CISO, 일반 계정은 라인매니저

## 3. 범위

- 임직원·외주·협력사·의료서비스제공자(MSP) 운영 인력 계정
- 서비스 계정, API 키, 장치 인증서 등 비인적 계정
- 생성·권한 변경·직무 이동·퇴사·외주 계약 종료 시 처리
- 반기 권한 검토 및 불용 계정 식별

## 4. 입력 자료 / 산출물

- **Input**
  - [[TMP-ACC-001_권한_신청서_v0.1]] (사유·직무·요청 권한·기간)
  - 인사 시스템의 입퇴사·직무 이동 데이터
  - 시스템·역할(RBAC) 매트릭스
- **Output**
  - 계정 생성·변경·삭제 기록
  - [[TMP-ACC-002_권한_검토_대장_v0.1]] 반기 산출물
  - 불용 계정 비활성화 로그

## 5. 수행 절차

### 5.1 사전 준비

1. IAM 관리자가 RBAC 매트릭스(역할 ↔ 권한) 와 직무별 표준 권한 세트를 최신 상태로 유지한다.
2. 인사 시스템 연동이 가능한 경우, 입·퇴사·직무 변경 이벤트가 IAM 시스템에 24시간 이내 자동 반영되도록 구성한다.
3. 분기 점검에서 불용 계정 임계값(기본 90일 미사용) 을 확정한다.

### 5.2 수행 단계

1. **계정 생성**
   - 요청자가 [[TMP-ACC-001_권한_신청서_v0.1]] 를 작성한다. 필수 항목: 성명·소속·직무·요청 역할·사유·기간.
   - 라인매니저가 업무 필요성을 검토하여 승인한다.
   - IAM 관리자가 RBAC 매트릭스의 표준 역할을 배정한다. 특권 권한은 별도 신청 분리.
   - 생성 직후 초기 비밀번호는 강제 변경 플래그를 설정한다.
2. **권한 변경 (직무 이동·추가 권한 요청)**
   - 요청자가 사유와 추가 권한을 기재한 새 신청서를 제출한다.
   - 기존 권한의 유지·회수 여부를 라인매니저가 명시적으로 결정한다(유지·회수 디폴트 없음).
   - IAM 관리자가 변경 적용 후 변경 로그를 남긴다.
3. **계정 삭제 (퇴사·계약 종료)**
   - 인사 이벤트 수신 즉시 IAM 관리자가 대상 계정을 비활성화한다.
   - 30일간 보존 후 완전 삭제한다. 조사·법무 홀드 대상은 CISO 승인 아래 유지한다.
4. **서비스 계정·API 키 관리**
   - 소유자 시스템·용도·만료일을 대장에 기록한다.
   - 만료 30일 전 자동 알림을 발행한다.
5. **반기 권한 검토**
   - IAM 관리자가 전 계정 목록을 [[TMP-ACC-002_권한_검토_대장_v0.1]] 에 추출한다.
   - 라인매니저가 계정별 유지·회수를 기재한다.
   - 회수 결과는 2주 이내 적용한다.
6. **불용 계정 처리**
   - 90일 미사용 계정을 매월 탐지하고, 14일 경고 후 비활성화한다.

### 5.3 완료 조건

- [ ] 모든 신규 계정이 [[TMP-ACC-001_권한_신청서_v0.1]] 에 근거하여 생성되었다
- [ ] 특권 계정 생성·변경 이력이 CISO 승인 기록과 일치한다
- [ ] 퇴사·계약 종료 이벤트 후 24시간 이내 비활성화되었다
- [ ] 반기 권한 검토가 완료되고 회수 결과가 적용되었다
- [ ] 90일 미사용 불용 계정이 비활성화되었거나 사유 기록이 남았다
- [ ] 서비스 계정·API 키의 소유자·만료일이 대장에 최신 상태로 유지된다

## 6. 인터페이스 부서

- **인사팀**: 입·퇴사·직무 이동 이벤트 데이터 제공
- **라인매니저**: 권한 승인·회수 결정
- **시스템 소유자**: 역할 매트릭스 정의, 기술 영향 검토
- **CISO**: 특권 계정 승인, 예외 승인
- **SecOps**: 권한 이상 활동 모니터링 연계

## 7. 주의사항 / 예외 처리

### 7.1 긴급 권한 요청

침해사고 대응 중 CSIRT 구성원에게 즉시 특권 권한이 필요한 경우, CISO 구두 승인만으로 IAM 관리자가 임시 권한을 부여한다. 부여 후 24시간 이내 정식 신청서를 제출하고, 사후 PSC 회의에서 승인 정당성을 검증한다.

### 7.2 장기 휴직·파견

3개월 이상 휴직·외부 파견 시 계정을 비활성화 대신 `suspended` 상태로 전환한다. 복귀 1주일 전 라인매니저가 재활성화를 신청해야 하며, 1년 이상 휴직 시 완전 삭제한다.

### 7.3 공유 계정 금지

개인이 식별되지 않는 공유 계정은 원칙적으로 생성하지 않는다. 장비 제한 등 기술적 불가피 사유로 공유 계정이 불가피한 경우, 사유와 대체 통제(세션 녹화·IP 제한·시간 제한) 를 신청서에 명시하고 CISO 예외 승인을 받는다.

### 7.4 외부 연구자·펜테스터 임시 계정

[[WI-601-02_VDP_운영_외부연구자_v0.1]] 의 VDP 또는 계약 기반 펜테스트에 한해 최대 14일 만료 계정을 발급한다. 만료 후 자동 삭제되며 연장은 새 신청으로만 가능하다.
