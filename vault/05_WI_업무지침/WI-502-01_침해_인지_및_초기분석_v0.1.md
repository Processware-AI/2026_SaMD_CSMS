# 침해 인지 및 초기 분석 업무지침 (WI-502-01)

> 상위 절차: [[PRO-MDCS-502_침해사고_대응_및_신고_절차_v1.0]] · §5 단계 1~3

## 1. 업무 목적

전자적 침해행위의 조기 인지 기준을 표준화하고, 인지 즉시 2시간 이내에 초기 분석과 증거 보전을 완료하여 격리·신고·복구 의사결정의 근거를 확보한다.

## 2. 수행 주체

- **주 수행자**: CSIRT 분석관, SecOps Oncall
- **검토자**: CSIRT 리더
- **승인자**: CISO (등급 판정), CEO (위기 등급)

## 3. 범위

- SIEM·FIM·AV·DLP 탐지로 자동 인지
- 사용자·MSP 제보로 수동 인지
- 외부 기관(식약처·KISA) 통보 기반 인지
- 위협 인텔 공개로 추정 인지

## 4. 입력 자료 / 산출물

- **Input**
  - 탐지 알림·제보 내용
  - 자산 인벤토리·로그
  - 위협 인텔·SBOM 연동 정보
- **Output**
  - 침해 티켓 (IR ID 부여)
  - [[TMP-IR-001_침해_초기분석_보고서_v0.1]]
  - 등급 판정서

## 5. 수행 절차

### 5.1 사전 준비

1. CSIRT 리더가 인지 기준(비정상 접근·데이터 유출 징후·시스템 마비·악성코드 감염·환자 안전 이벤트) 을 문서화한다.
2. 24시간 Oncall 순번표와 비상 연락망을 유지한다.
3. 초기 분석 툴킷(디스크 이미징·메모리 덤프·패킷 캡처) 과 증거 보전 저장소를 준비한다.

### 5.2 수행 단계

1. **인지 접수**
   - Oncall 이 알림·제보를 받은 시점을 IR 티켓에 기록한다(타임스탬프·인지 채널·제보자).
   - IR ID 를 부여한다: `IR-{YYYY}-{###}`.
2. **초기 분류 (15분 이내)**
   - 명백한 False Positive 제외 후 침해 가능성 여부 1차 판단.
   - CSIRT 리더·CISO 에 즉시 통보.
3. **증거 보전**
   - 영향 자산의 디스크·메모리·네트워크 패킷을 해시와 함께 보전.
   - 로그를 Cold 저장소에 별도 격리.
   - 의료기기 장치는 장치 상태를 사진·시리얼·환경 기록.
4. **초기 분석 (2시간 이내)**
   - 침해 유형·발생 추정 시각·인지 시점·초기 영향 범위·현재 조치 상황 기록.
   - [[TMP-IR-001_침해_초기분석_보고서_v0.1]] 의 필수 필드를 모두 채운다.
5. **등급 판정**
   - 경미(Low)/중대(High)/위기(Critical) 기준에 따라 판정.
   - Critical 시 CEO 에 즉시 보고, CSIRT 발동.
6. **후속 단계 연계**
   - 격리·포렌식은 [[WI-502-03_격리_및_복구_포렌식_v0.1]].
   - 신고는 [[WI-502-02_대외신고_식약처_MSP_이용자_v0.1]].

### 5.3 완료 조건

- [ ] IR ID 가 부여되고 인지 채널·타임스탬프가 기록되었다
- [ ] 인지→초기 판단이 15분 이내 수행되었다
- [ ] 디스크·메모리·네트워크·로그 증거가 해시와 함께 보전되었다
- [ ] [[TMP-IR-001_침해_초기분석_보고서_v0.1]] 필수 필드가 2시간 이내 채워졌다
- [ ] 경미/중대/위기 등급 판정이 완료되었다
- [ ] 격리·신고 후속 WI 가 즉시 가동되었다

## 6. 인터페이스 부서

- **SecOps**: SIEM·FIM 알림 1차 수신 및 증거 수집 지원
- **CISO**: 등급 판정 최종 권한
- **개발·인프라팀**: 영향 자산 컨텍스트 제공
- **CS/홍보**: 사용자 제보 채널 운영

## 7. 주의사항 / 예외 처리

### 7.1 환자 안전 직접 영향

초기 분석 중 환자 안전에 직접 영향(예: 기기 오작동) 이 확인되면 2시간 기준을 기다리지 않고 즉시 위기(Critical) 로 격상하고 CEO·Safety 리드·RA 를 병렬 호출한다. 병렬로 BCP 발동([[WI-502-04_BCP_및_Fail-safe_v0.1]]).

### 7.2 False Positive 판정 후 재발

False Positive 로 종료한 케이스가 48시간 이내 유사 탐지로 재발하면 Hard 상향하여 재오픈한다. 이력을 SIEM 룰 튜닝에 반영.

### 7.3 증거 보전 불가(운영 중단 불가 자산)

환자 치료 중 의료기기의 즉시 이미지 확보가 불가한 경우, 가능한 범위의 원격 로그·메모리 덤프만 수집하고 환경 변수(상태·버전·트래픽) 를 상세 기록한다. 사후 점검 시점에 재수집 시도.

### 7.4 다중 동시 인지

복수 사건 동시 발생 시 CSIRT 리더가 우선순위를 지정한다. 동일 공격자 의심 시 단일 IR 티켓으로 통합, 별개 공격 시 다중 티켓 병렬 운영.
