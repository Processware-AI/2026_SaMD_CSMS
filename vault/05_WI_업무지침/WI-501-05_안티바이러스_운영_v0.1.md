# 안티바이러스 운영 업무지침 (WI-501-05)

> 상위 절차: [[PRO-MDCS-501_보안_모니터링_및_로그_관리_절차_v1.0]] · §5 단계 2 (AV 설치·운영)

## 1. 업무 목적

백신 소프트웨어(Anti-Virus, AV)의 체계적 배포·자동 업데이트·정기 스캔 실행을 통해 악성코드(바이러스·랜섬웨어·스파이웨어 등) 감염을 예방하고, 탐지 발생 시 즉각 대응 절차를 이행하여 운영 환경 및 디지털의료기기의 가용성·무결성을 보호한다.

## 2. 수행 주체

- **주 수행자**: 인프라팀 (AV 배포·업데이트·스캔 스케줄 관리)
- **검토자**: SecOps 팀장 (탐지 이벤트 분석, SIEM 연동 상태 점검)
- **승인자**: CISO (AV 정책 승인, 탐지 대응 예외 승인)

## 3. 범위

- 운영 환경 서버 (Windows Server, Linux 서버)
- 업무용 단말기(데스크톱·노트북)
- 의료기기 온보드 시스템 (AV 설치 가능한 범위)
- 가상화 환경(VMware, Hyper-V 등) 및 클라우드 인스턴스
- 배제 대상: AV 설치가 기술적으로 불가하거나 제조사가 금지한 임베디드 장치 (대체 통제 적용)

## 4. 입력 자료 / 산출물

- **Input**
  - 자산 목록 (AV 설치 대상·OS 유형·버전)
  - AV 중앙 관리 서버 콘솔 데이터 (패턴 버전·스캔 상태)
  - AV 벤더 패턴 업데이트 릴리스 노트
  - [[TMP-OPS-030_AV_운영_현황_대장_v0.1]] (설치 현황·패턴 버전 추적)
- **Output**
  - 완료된 [[TMP-OPS-030_AV_운영_현황_대장_v0.1]] (월간 현황 포함)
  - [[TMP-OPS-031_AV_탐지_대응_기록_v0.1]] (탐지→격리→대응 이력)
  - 월간 AV 운영 보고서 (패턴 적용률·스캔 완료율·탐지 건수)
  - 격리·제거 완료 증적 로그

## 5. 수행 절차

### 5.1 사전 준비

1. 인프라팀은 AV 중앙 관리 서버가 정상 운영 중인지 확인하고, 관리 콘솔 접근 계정을 최소 권한 원칙에 따라 유지한다.
2. [[TMP-OPS-030_AV_운영_현황_대장_v0.1]]에 모든 대상 자산이 등록되어 있는지 확인하고, 신규 자산 추가 시 즉시 업데이트한다.
3. AV 패턴 업데이트 서버(내부 배포 서버 또는 벤더 클라우드)와의 연결 상태를 매일 확인한다.
4. 인터넷 비연결 환경(Air-gap 시스템)을 위한 오프라인 패턴 업데이트 절차를 별도로 문서화하고, 매주 오프라인 업데이트 패키지를 준비한다.

### 5.2 수행 단계

1. **AV 배포 및 초기 설치**
   - 신규 자산에 AV 에이전트를 배포 전, AV 중앙 관리 서버에 자산을 등록한다.
   - 배포 후 AV 에이전트가 중앙 관리 서버와 통신하고 최신 패턴을 수신하였는지 확인한다.
   - 초기 전체 스캔(Full Scan)을 실행하고 결과를 [[TMP-OPS-030_AV_운영_현황_대장_v0.1]]에 기록한다.
   - 의료기기에 AV를 설치하는 경우, 개발팀 및 제조사의 설치 가이드를 준수하고 임상 기능 영향 없음을 확인한 후 배포한다.

2. **패턴 업데이트 관리**
   - AV 패턴 파일은 벤더 릴리스 후 24시간 이내 모든 대상 자산에 자동 배포되도록 중앙 관리 서버 정책을 구성한다.
   - 자동 업데이트 성공률을 매일 모니터링하고, 실패한 자산은 수동으로 업데이트하거나 원인을 파악하여 조치한다.
   - 패턴 적용률이 99% 미만인 경우 인프라팀장이 즉시 원인을 파악하고 CISO에 보고한다.
   - 오프라인(Air-gap) 환경은 주 1회 이상 오프라인 패턴 업데이트 패키지를 적용한다.

3. **정기 스캔 스케줄 운영**
   - 전체 스캔(Full Scan): 월 1회 이상, 업무 시간 외(야간·주말)에 실행
   - 빠른 스캔(Quick Scan): 주 1회 이상, 메모리·시작 프로그램·임시 파일 대상
   - 이메일·다운로드 파일 실시간 스캔: 상시 활성화
   - 이동식 저장매체(USB 등) 연결 시 자동 스캔: 반드시 활성화
   - 스캔 일정은 AV 중앙 관리 서버에서 중앙 통제하고, 개별 단말에서 임의 비활성화를 기술적으로 차단한다.

4. **탐지 이벤트 처리**
   - AV 탐지 이벤트 발생 시 인프라팀은 30분 이내 [[TMP-OPS-031_AV_탐지_대응_기록_v0.1]]을 생성한다.
   - 탐지된 악성코드의 유형·위치·관련 프로세스를 확인한다.
   - AV가 자동 격리·제거한 경우: 격리 파일을 확인하고 오탐 여부를 검토한다.
   - AV가 자동 처리하지 못한 경우(치료 실패): 해당 자산을 네트워크에서 수동 격리하고 SecOps에 즉시 에스컬레이션한다.
   - 랜섬웨어·심층 침해 의심 탐지: PRO-MDCS-502(침해사고 대응)를 즉시 트리거한다.

5. **격리 파일 검토 및 처리**
   - 격리된 파일은 7일 이내 오탐 여부를 최종 검토한다.
   - 오탐으로 확인된 경우: AV 제외 목록(Exclusion)에 추가하고 벤더에 오탐 리포트를 제출한다.
   - 실제 악성코드로 확인된 경우: 영구 삭제하고 감염 경로를 분석하여 재발 방지 조치를 수립한다.

6. **월간 운영 현황 집계 및 보고**
   - 매월 말 인프라팀장은 패턴 적용률·스캔 완료율·탐지 건수·격리 건수를 집계하여 AV 운영 보고서를 작성한다.
   - AV 미설치 자산 또는 패턴 미업데이트 자산을 목록화하여 CISO에 보고한다.

### 5.3 완료 조건

- [ ] 모든 대상 자산이 [[TMP-OPS-030_AV_운영_현황_대장_v0.1]]에 등록되고 AV 에이전트가 활성화되어 있다
- [ ] AV 패턴 적용률이 99% 이상 유지되고 있음이 월간 보고서로 확인된다
- [ ] 정기 스캔 스케줄(전체 스캔 월 1회, 빠른 스캔 주 1회)이 실행되고 결과 로그가 존재한다
- [ ] 탐지 이벤트 전수에 대해 [[TMP-OPS-031_AV_탐지_대응_기록_v0.1]]이 작성되었다
- [ ] 자동 처리 불가 탐지 건이 4시간 이내 SecOps로 에스컬레이션되었다
- [ ] 격리 파일에 대한 오탐 검토가 7일 이내 완료되었다
- [ ] 월간 운영 보고서가 CISO에 제출되었다

## 6. 인터페이스 부서

- **SecOps 팀**: 탐지 이벤트 분석·SIEM 연동, 심층 침해 에스컬레이션 연계
- **개발팀**: 의료기기 AV 설치 가이드 제공, 임상 기능 영향 검토
- **CISO**: AV 정책 승인, 탐지 대응 예외 승인, 월간 보고서 수신
- **CSIRT**: 랜섬웨어·심층 침해 탐지 시 침해사고 대응 연계

## 7. 주의사항 / 예외 처리

### 7.1 AV 설치 불가 자산(예외 처리)

의료기기 제조사가 AV 설치를 금지하거나 임베디드 OS로 인해 설치가 불가한 경우:
- 해당 자산을 [[TMP-OPS-030_AV_운영_현황_대장_v0.1]]의 "예외 자산" 섹션에 등록하고 CISO 승인을 받는다.
- 대체 통제로 네트워크 분리(DMZ 또는 의료기기 전용 VLAN), USB 포트 물리 차단, 외부 파일 반입 절차 강화를 적용한다.
- 예외 자산에 대해서는 6개월마다 제조사 지침 변경 여부를 재확인한다.

### 7.2 AV 오탐으로 인한 임상 운영 중단 위험

의료기기 또는 임상 시스템에서 AV가 정상 의료 소프트웨어를 악성코드로 오탐하여 격리할 경우 환자 안전 문제로 직결될 수 있다. 이 경우:
- AV 에이전트의 의료 소프트웨어 설치 경로를 제외 목록(Exclusion)으로 등록하되, 제외 범위를 최소화한다.
- 제외 목록 변경은 개발팀·QA팀의 검토를 거쳐 CISO가 승인한다.
- 오탐으로 격리된 파일을 복구하는 경우, 해당 파일의 무결성을 해시값으로 검증한 후 복구한다.

### 7.3 업무 시간 내 긴급 패턴 업데이트

제로데이 위협 등으로 인한 긴급 패턴 업데이트가 필요한 경우, 업무 시간 내 강제 배포를 실행할 수 있다. 배포 전 인프라팀장이 시스템 성능 영향을 평가하고, 영향이 큰 경우 단계적으로 배포한다. 긴급 배포 기준은 CISO가 판단하고 구두 승인 후 24시간 이내 서면으로 확인한다.
