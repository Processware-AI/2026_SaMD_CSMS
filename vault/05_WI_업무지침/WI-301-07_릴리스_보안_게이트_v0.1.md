# 릴리스 보안 게이트 업무지침 (WI-201-07)

> 상위 절차: [[PRO-MDCS-301_보안_SDLC_절차_v1.0]] · §5 단계 12

## 1. 업무 목적

SaMD 제품 릴리스 승인 전 보안 관련 모든 산출물의 완비성·품질·잔여 위험 수용 여부를 단일 체크리스트로 확인하여 릴리스 이후 중대 취약점 유입을 방지한다.

## 2. 수행 주체

- **주 수행자**: PSO
- **검토자**: QA 리드, 오동석, PSC
- **승인자**: VP of R&D, 오동석 공동

## 3. 범위

- 신규·개정 릴리스
- 긴급 핫픽스(간소화 게이트)
- 장기 지원 버전의 소급 검증

## 4. 입력 자료 / 산출물

- **Input**
  - SAST/DAST/Fuzzing/펜테스트 리포트
  - 위협 모델링·위험 평가 보고서
  - SBOM·CVE 매칭 결과
- **Output**
  - [[TMP-SDLC-003_릴리스_보안_게이트_체크리스트_v0.1]] 완성본
  - 릴리스 승인 기록 또는 차단 사유

## 5. 수행 절차

### 5.1 사전 준비

1. PSO 가 릴리스 게이트 기준을 유지한다(아래 §5.2 3번).
2. 게이트 체크리스트를 릴리스 정의(DoD) 에 포함.
3. 긴급 핫픽스 간소화 기준을 오동석 와 합의.

### 5.2 수행 단계

1. **산출물 수집**
   - 최신 SAST·DAST·Fuzzing·펜테스트 리포트.
   - 위협 모델링·위험 평가 보고서.
   - SBOM 과 CVE 매칭 결과.
   - 잔여 위험 수용 기록(PSC/CEO).
2. **완비성 점검**
   - 체크리스트 50개 항목 각각 충족 증빙 확인.
3. **임계 기준 적용**
   - Critical SAST/DAST 결함 0건.
   - High 결함은 승인된 완화 계획 보유.
   - SBOM 의 모든 Critical CVE 는 해소 또는 수용 기록.
   - 위협 모델링에서 식별된 High 위협의 통제 구현 완료.
   - 펜테스트 고위험 발견 모두 조치 또는 수용.
4. **심의**
   - PSC 에서 잔여 위험을 검토.
   - 위기 등급은 CEO 에 보고.
5. **승인·차단**
   - VP of R&D + 오동석 공동 승인 시 릴리스 진행.
   - 미충족 시 개선 요청으로 반려, 재심의 일정 지정.
6. **릴리스 후 게이트 기록**
   - 배포 이력과 체크리스트를 연결 보관.
   - 사후 발견 취약점은 본 기록에 역참조하여 프로세스 개선.

### 5.3 완료 조건

- [ ] 모든 보안 산출물(SAST·DAST·Fuzz·펜테스트·위협 모델·위험 평가·SBOM) 이 제출되었다
- [ ] 체크리스트 50개 항목이 충족되거나 예외 승인이 기록되었다
- [ ] Critical 결함 0건이 확인되었다
- [ ] High 결함에 대한 완화 계획이 승인되었다
- [ ] 잔여 위험 수용이 PSC/CEO 기록과 일치한다
- [ ] VP of R&D + 오동석 공동 승인이 문서화되었다

## 6. 인터페이스 부서

- **PSC**: 잔여 위험 심의
- **QA**: 테스트 커버리지 보증
- **RA**: 규제 요구사항 대응 확인
- **SBOM 관리자**: CVE 매칭 최신 유지

## 7. 주의사항 / 예외 처리

### 7.1 긴급 핫픽스 간소화

환자 안전 위협 긴급 패치 시 간소화 게이트(SAST·DAST 결과·핵심 테스트·SBOM 증분만) 로 운영. 배포 후 72시간 이내 전체 게이트 소급 수행.

### 7.2 예외 승인

특정 항목 미충족 시 PSC 예외 승인을 조건부로 부여(기한·모니터링·롤백 계획 필수). 예외는 릴리스 후 다음 릴리스까지 해소.

### 7.3 게이트 우회 시도

비공식 경로로 게이트 우회 시도가 탐지되면 PSO 가 즉시 릴리스 중단 요청. 경영진 보고 및 프로세스 제재 검토.

### 7.4 규제 심사 대응

식약처 심사·점검 대응 시 게이트 기록을 증적으로 제공. 심사 피드백은 게이트 기준 개선에 반영.
