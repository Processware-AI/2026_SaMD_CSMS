# 이동식매체 통제 업무지침 (WI-402-02)

> 상위 절차: [[PRO-MDCS-402_보안_통신_및_암호화_관리_절차_v1.0]] · §5 단계 3

## 1. 업무 목적

USB 드라이브·외장 HDD·SD 카드 등 이동식 저장매체를 통한 악성코드 유입, 비인가 데이터 반출·유출을 예방하기 위해 매체 사용 전 신청·승인 절차를 수행하고, 사용 중 자동실행 차단·암호화 적용, 사용 후 잔존 데이터 삭제를 일관되게 수행한다. SaMD-CSMS 제04조 제2호의 이동식 저장매체 통제 요건을 충족한다.

## 2. 수행 주체

- **주 수행자**: 인프라팀 보안담당 (포트 통제 설정, 화이트리스트 관리, 삭제 검증)
- **신청·사용자**: 이동식매체 사용이 필요한 임직원·외주·협력업체 인력
- **승인자**: 라인매니저(업무 필요성), 오동석(민감 데이터 관련 또는 외부 반출 시)
- **검토자**: SecOps (이상 탐지 모니터링 연계)

## 3. 범위

- 조직 소유 또는 개인 소유 이동식 저장매체(USB 플래시드라이브, 외장 HDD/SSD, SD·MicroSD 카드, CD/DVD, 테이프 등)
- 운영 환경 내 모든 단말(서버, 워크스테이션, 의료기기 연결 PC)의 USB·PCMCIA·eSATA·SD 포트
- 내부 반입(외부 → 내부) 및 외부 반출(내부 → 외부) 모두 적용
- 예외: 조직 발급·암호화된 매체를 사용하는 표준 백업 프로세스 (별도 백업 절차 적용)

## 4. 입력 자료 / 산출물

- **Input**
  - [[TMP-INF-011_이동식매체_사용_신청서_v0.1]] (신청자·소속·목적·데이터 분류·사용 기간·매체 식별정보)
  - 단말 보안 정책(EDR/DLP 설정 현황)
  - 화이트리스트 매체 등록 대장
- **Output**
  - [[TMP-INF-012_이동식매체_사용_대장_v0.1]] (승인 이력, 사용 시작/종료 시각, 삭제 완료 확인)
  - EDR·DLP 연결 이벤트 로그
  - 잔존 데이터 삭제 완료 증적 (해시 비교 또는 삭제 도구 실행 리포트)

## 5. 수행 절차

### 5.1 사전 준비

1. 인프라팀 보안담당은 모든 운영 단말의 USB 포트를 기본 **비활성화(disabled)** 또는 **읽기 전용(read-only)** 상태로 GPO/MDM 정책에 설정하고, 분기 1회 설정 준수 여부를 점검한다.
2. 조직이 공식 지급한 암호화 USB(예: BitLocker To Go 또는 하드웨어 암호화 모델)만 화이트리스트(SerialNumber 기반)에 등록하며, 등록 목록을 [[TMP-INF-012_이동식매체_사용_대장_v0.1]] 에 유지한다.
3. 사용 신청 전 신청자는 전송할 데이터의 민감도 분류(개인정보·기밀·내부·공개)를 확인하고, 민감 데이터 포함 시 DPO에게 사전 통보한다.
4. 자동실행(Autorun/AutoPlay)이 GPO·레지스트리 설정을 통해 전 단말에 비활성화되어 있음을 월 1회 스캐닝으로 검증한다.

### 5.2 수행 단계

1. **사용 신청**
   - 신청자가 [[TMP-INF-011_이동식매체_사용_신청서_v0.1]] 를 작성한다. 필수 항목: 신청자·소속·사용 목적·대상 데이터 분류·사용 기간(시작~종료)·매체 제조사·시리얼번호.
   - 라인매니저가 업무 필요성을 확인하여 1차 승인한다.
   - 민감 데이터(개인정보·기밀) 반출 또는 외부 반입 매체 사용 시, 오동석의 2차 승인을 받는다.
2. **매체 등록 및 포트 임시 활성화**
   - 인프라팀 보안담당이 승인된 신청서를 확인하고 해당 매체의 시리얼번호를 화이트리스트에 등록한다.
   - 해당 단말에 한해 승인 기간 동안 쓰기 가능 상태로 포트를 임시 활성화한다. 기간 외 자동 차단 정책을 설정한다.
3. **사용 중 통제**
   - 사용자는 매체 연결 즉시 EDR 에이전트가 연결 이벤트를 기록함을 인지하고, DLP 정책에 따른 파일 복사 로그가 생성됨을 동의한다.
   - 암호화되지 않은 개인 소유 매체를 연결하면 EDR이 자동 차단하고 SecOps에 알림을 발송한다.
   - 사용 중 악성코드 스캔이 자동 실행되며 위협 탐지 시 즉시 격리(quarantine) 조치한다.
4. **사용 완료 및 데이터 삭제 검증**
   - 사용자가 작업 완료 후 매체에 남은 데이터를 안전 삭제 도구(DoD 5220.22-M 3-pass 이상 또는 단일 덮어쓰기 + 검증)로 삭제한다.
   - 인프라팀 보안담당이 삭제 완료 리포트(도구 실행 로그)를 [[TMP-INF-012_이동식매체_사용_대장_v0.1]] 에 첨부하고 완료 일시를 기록한다.
5. **포트 재비활성화 및 대장 등록 마감**
   - 승인 기간 종료 시 인프라팀 보안담당이 해당 단말 포트를 비활성화 상태로 복원한다.
   - 사용 대장에 종료 서명(전자 승인)을 완료한다.
6. **정기 감사**
   - SecOps가 월 1회 EDR·DLP 이벤트 로그와 사용 대장을 대조하여 미등록 매체 연결 시도, 승인 기간 초과 사용, 삭제 미이행 건을 식별하고 오동석에 보고한다.

### 5.3 완료 조건

- [ ] 신청서에 라인매니저(및 해당 시 오동석) 승인이 기록되어 있다
- [ ] 매체 시리얼번호가 화이트리스트에 등록·해제 이력이 남아 있다
- [ ] 사용 기간 동안 EDR 연결 이벤트 로그가 사용 대장과 일치한다
- [ ] 사용 완료 후 안전 삭제 도구 실행 리포트가 대장에 첨부되었다
- [ ] 포트가 승인 기간 종료 후 비활성화 상태로 복원되었다
- [ ] 미등록 매체 연결 시도 이벤트가 월간 감사에서 검토·처리되었다

## 6. 인터페이스 부서

- **라인매니저**: 사용 필요성 1차 승인
- **오동석**: 민감 데이터 반출·외부 반입 매체 2차 승인, 예외 승인
- **SecOps**: EDR·DLP 이상 이벤트 모니터링, 월간 감사 보고
- **DPO**: 개인정보 포함 데이터 반출 사전 통보 수신, 위반 시 개인정보보호 조치 연계
- **IT헬프데스크**: 암호화 매체 발급·수령, 초기 등록 지원

## 7. 주의사항 / 예외 처리

### 7.1 긴급 운영 상황 (네트워크 단절 복구)

네트워크 장애 등 긴급 상황에서 패치·펌웨어 배포를 위해 이동식매체가 필요한 경우, 오동석 구두 승인 후 인프라팀 보안담당이 임시 활성화하며, 24시간 이내 사후 신청서를 제출한다. 사용 매체는 조직 발급 암호화 USB에 한하며, 작업 완료 즉시 포트를 재비활성화하고 사용 내역을 사후 대장에 기재한다.

### 7.2 개인 소유 매체 반입 시도

직원이 개인 소유 비암호화 매체를 연결하면 EDR이 즉시 차단하고 SecOps가 인프라팀 보안담당에게 알림을 발송한다. 인프라팀 보안담당이 사용자에게 정책 위반을 고지하고, 반복 위반 시 인사팀·오동석에 보고하여 징계 절차를 개시한다.

### 7.3 외주·협력업체 인력의 매체 사용

외주·협력업체 인력은 개인 소유 매체 사용이 원칙적으로 금지된다. 업무상 불가피한 경우 조직 발급 암호화 매체를 대여(대여 대장 기록)하며, 계약 종료 시 즉시 반납하고 포렌식 삭제를 수행한다.

### 7.4 의료기기 연결 단말 특별 통제

의료기기와 직접 연결된 단말은 이동식매체 연결을 원칙적으로 영구 금지하며, 반드시 하드웨어 포트 봉인(물리적 봉인 스티커·잠금 장치)을 병행한다. 봉인 상태는 월 1회 현장 점검으로 확인한다.
