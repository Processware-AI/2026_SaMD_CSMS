# 통신 암호화 업무지침 (WI-402-03)

> 상위 절차: [[PRO-MDCS-402_보안_통신_및_암호화_관리_절차_v1.0]] · §5 단계 4~5

## 1. 업무 목적

디지털의료기기와 운영 서버·클라이언트·외부 시스템 간의 모든 통신 구간에 TLS 1.2 이상 또는 IPsec/SSH 를 일관되게 적용하고, 취약 암호 알고리즘(DES/MD5/SHA-1 등) 을 금지하여 데이터 가로채기·변조를 예방한다.

## 2. 수행 주체

- **주 수행자**: 인프라팀 엔지니어
- **검토자**: 네트워크팀, KMS Admin
- **승인자**: CISO

## 3. 범위

- 제품 서버·클라이언트·API 게이트웨이 간 HTTPS/TLS
- 원격 접속 VPN(IPsec/SSL VPN), SSH
- 의료기기와 백엔드 간 통신(DICOM over TLS 등)
- 인증서 발급·갱신·폐기

## 4. 입력 자료 / 산출물

- **Input**
  - 통신 구간 목록(아키텍처 도면)
  - 승인된 암호 알고리즘 목록(AES-256, SHA-256/512, ECDHE 등)
- **Output**
  - TLS 설정 기록(암호 스위트·프로토콜 버전·인증서)
  - [[TMP-NET-002_TLS_인증서_인벤토리_v0.1]] 대장
  - 취약 암호 스캔 리포트(SSL Labs 등)

## 5. 수행 절차

### 5.1 사전 준비

1. KMS Admin 이 승인 암호 알고리즘 목록을 최신화하여 공유한다.
2. 인증서 발급 체계(내부 CA 또는 공용 CA) 를 확정한다.

### 5.2 수행 단계

1. **프로토콜·암호 스위트 설정**
   - TLS 1.2 이상 강제, TLS 1.0/1.1 비활성.
   - ECDHE 기반 Forward Secrecy 암호 스위트를 우선 사용한다.
   - 대칭키 AES-256-GCM, 해시 SHA-256/SHA-384 사용.
   - DES/3DES/RC4/MD5/SHA-1 은 금지한다.
2. **인증서 관리**
   - 공개 서비스는 공용 CA, 내부 서비스는 내부 CA 로 발급한다.
   - 모든 인증서를 [[TMP-NET-002_TLS_인증서_인벤토리_v0.1]] 에 등록한다. 등록 필드: 도메인·발급기관·발급일·만료일·소유팀.
   - 만료 30일 전 자동 알림(메일·SIEM) 을 발생시킨다.
3. **VPN·SSH 구성**
   - IPsec IKEv2 + AES-256 + SHA-256.
   - SSH 는 OpenSSH 최신, Root 직접 로그인 차단, 키 기반 인증 기본.
   - 약한 Cipher/KEX 는 `sshd_config` 에서 비활성.
4. **의료기기 프로토콜 암호화**
   - DICOM·HL7 FHIR 등 의료 전용 프로토콜은 TLS 위에 실행한다.
   - 평문 구간이 존재하는 경우 보안 게이트웨이·서비스 메시로 터널링.
5. **점검·스캔**
   - 분기별 외부 포트에 대해 SSL Labs 또는 내부 스캐너로 TLS 설정을 검증한다.
   - A등급 미만 서비스는 30일 이내 교정한다.
6. **이상 탐지**
   - 인증서 변조·비정상 발급 탐지는 SIEM 룰로 자동화한다.

### 5.3 완료 조건

- [ ] TLS 1.0/1.1 및 취약 암호가 모든 서비스에서 비활성화되었다
- [ ] 인증서 인벤토리가 최신 상태로 유지된다
- [ ] 인증서 만료 30일 전 알림이 작동한다
- [ ] SSH·VPN 의 Root 직접 접속 차단·키 인증 강제가 설정되었다
- [ ] 분기 TLS 점검 리포트가 A등급 이상을 유지한다
- [ ] 의료기기 프로토콜 통신이 TLS 위에 실행된다

## 6. 인터페이스 부서

- **KMS Admin**: 키·알고리즘 적정성 검토 ([[WI-402-04_암호키_수명주기_v0.1]])
- **네트워크팀**: 방화벽·VPN 게이트웨이 연동
- **SecOps**: TLS 이상 탐지 SIEM 룰

## 7. 주의사항 / 예외 처리

### 7.1 인증서 긴급 교체

개인키 유출 의심 발생 시 인프라팀장이 즉시 해당 인증서를 폐기(Revoke) 하고 신규 발급한다. CRL/OCSP 업데이트를 확인하고, 유출 범위·영향을 CSIRT 에 에스컬레이션한다.

### 7.2 레거시 의료기기 TLS 1.2 미지원

구형 기기가 TLS 1.0/1.1 만 지원하는 경우, 해당 기기를 격리 VLAN 에 배치하고 보안 게이트웨이에서 TLS 1.2 로 업그레이드 변환한다. 교체 계획을 [[WI-303-03_레거시_취약점_패치_협력_v0.1]] 에 등록한다.

### 7.3 만료 직전 갱신 지연

인증서 자동 갱신이 실패하고 만료 48시간 이내인 경우, 인프라팀장이 긴급 수동 갱신을 수행하고 동시에 자동화 파이프라인 장애 원인을 분석한다.

### 7.4 포스트 양자 암호(PQC) 전환

NIST PQC 표준화 확정 시 KMS Admin 이 전환 로드맵을 수립하고, 인프라팀은 하이브리드 암호 스위트(X25519+ML-KEM 등) 를 우선 적용한다.
