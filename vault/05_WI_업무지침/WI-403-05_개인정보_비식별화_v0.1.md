# 개인정보 비식별화 업무지침 (WI-403-05)

> 상위 절차: [[PRO-MDCS-403_데이터_및_파일_보안_관리_절차_v1.0]] · §5 단계 6

## 1. 업무 목적

디지털의료기기가 수집·처리하는 개인정보·민감정보에 대해 가명처리·익명처리·총계처리·마스킹 등 비식별화 조치를 일관된 기준으로 적용하고, 재식별 위험 평가를 주기적으로 수행하여 개인정보보호법 준수와 환자 프라이버시 보호를 달성한다.

## 2. 수행 주체

- **주 수행자**: DPO 와 데이터 처리 담당자
- **검토자**: CISO, 법무
- **승인자**: CISO

## 3. 범위

- 환자 식별정보(성명·주민번호·연락처)
- 의료 영상·진단 보고서
- 센서·로그 데이터에 내포된 개인 식별 요소
- 연구·AI 학습 목적의 2차 활용 데이터

## 4. 입력 자료 / 산출물

- **Input**
  - 데이터 카탈로그·분류 결과
  - 처리 목적(진료·연구·품질관리)
  - 재식별 위험 평가 방법론(K-익명성·L-다양성 등)
- **Output**
  - 비식별화 조치 결과물
  - [[TMP-PRIV-001_비식별화_조치_내역서_v0.1]]
  - 재식별 위험 평가 보고서

## 5. 수행 절차

### 5.1 사전 준비

1. DPO 가 데이터 카탈로그에서 개인정보 포함 여부·민감도 등급을 확정한다.
2. 처리 목적별 적정 기법(가명 vs 익명) 을 결정한다. 진료·청구 목적은 식별 필요, 연구·AI 는 익명 우선.
3. 재식별 위험 평가 도구(ARX·sdcMicro 등) 를 도입한다.

### 5.2 수행 단계

1. **식별자·준식별자 분류**
   - 직접 식별자(주민번호·성명) 와 준식별자(생년월일·성별·진료일) 를 분리한다.
2. **가명 처리**
   - 암호학적 해시(키드 해시) 또는 매핑 테이블로 가명을 부여한다.
   - 매핑 테이블은 별도 보안 저장소에 접근 통제된 상태로 보관한다.
3. **익명 처리 (2차 활용·연구)**
   - 일반화(예: 생년월일 → 연령대), 삭제, 잡음 추가를 조합 적용한다.
   - K-익명성(K≥5) 및 L-다양성 기준 달성 여부를 검증한다.
4. **총계 처리**
   - 통계·보고 목적은 집계 값만 공개하고 개인 수준 데이터 노출 없이 제공한다.
5. **마스킹**
   - 화면·로그 표시 시 성명·연락처 등에 마스킹을 적용한다(예: 홍*동, 010-****-1234).
6. **재식별 위험 평가**
   - 외부 제공·공개 전 반드시 위험 평가를 수행한다.
   - [[TMP-PRIV-001_비식별화_조치_내역서_v0.1]] 에 적용 기법·K값·잔여 위험을 기록한다.
7. **정기 재평가**
   - 반기 1회 샘플 데이터에 재식별 시도 테스트를 수행하여 지속적 안전성을 검증한다.

### 5.3 완료 조건

- [ ] 개인정보 포함 데이터에 대해 직접·준식별자가 분리 식별되었다
- [ ] 처리 목적별 적정 기법(가명/익명/총계/마스킹) 이 적용되었다
- [ ] 가명 매핑 테이블이 별도 접근 통제 저장소에 보관된다
- [ ] 외부 제공 전 재식별 위험 평가(K-익명성 등) 가 수행되었다
- [ ] [[TMP-PRIV-001_비식별화_조치_내역서_v0.1]] 가 최신 버전으로 유지된다
- [ ] 반기 재식별 시도 테스트 결과가 기록되었다

## 6. 인터페이스 부서

- **법무**: 개인정보보호법·의료법 준수 검토
- **개발팀**: 가명·마스킹 로직 구현
- **SecOps**: 비식별화 데이터 접근 감사
- **연구·AI 팀**: 2차 활용 시 익명 데이터 요청

## 7. 주의사항 / 예외 처리

### 7.1 가명 키 유출 시 전면 재가명

가명 해시키가 유출된 경우 DPO 가 즉시 신규 키로 전면 재가명을 지시하고, 유출 기간에 외부 제공된 데이터의 회수 가능성을 검토한다. 개인정보 유출 신고 의무 요건 충족 시 [[PRO-MDCS-502_침해사고_대응_및_신고_절차_v1.0]] 로 연계한다.

### 7.2 소수 표본 (K<5) 익명 불가

특정 질병의 희귀성으로 K-익명성 충족이 불가한 데이터는 외부 제공을 중단하거나 합성 데이터(Synthetic Data) 생성으로 전환한다. 연구 목적 외부 제공은 IRB 승인과 안전한 분석 환경(Secure Enclave) 를 병행한다.

### 7.3 의료 영상의 메타데이터

DICOM 헤더의 환자 식별정보(PatientName·PatientID 등) 는 DICOM PS3.15 Basic Profile 등에 따라 익명화하고 원본은 별도 보관한다. 연구용 공개 시 체형·주름 등 잠재 식별자도 검토한다.

### 7.4 AI 학습 재식별 공격 방어

모델이 학습 데이터 재식별(Membership Inference) 에 취약할 수 있으므로 DP-SGD 등 차등 프라이버시 기법 적용을 검토하고, [[WI-301-06_AI_데이터_모델_위협평가_v0.1]] 와 연계한다.
