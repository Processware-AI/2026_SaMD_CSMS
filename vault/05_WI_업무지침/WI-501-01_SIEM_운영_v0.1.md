# SIEM 운영 업무지침 (WI-501-01)

> 상위 절차: [[PRO-MDCS-501_보안_모니터링_및_로그_관리_절차_v1.0]] · §5 단계 4~6

## 1. 업무 목적

디지털의료기기 운영 환경의 보안 이벤트를 SIEM 에 수집·상관 분석하여 이상 행위를 조기 탐지하고, 플레이북 기반 초기 대응을 자동화하여 MTTD(평균 탐지 시간) 를 60분 이내로 유지한다.

## 2. 수행 주체

- **주 수행자**: SecOps 엔지니어
- **검토자**: CSIRT, PSO
- **승인자**: 오동석

## 3. 범위

- 운영 서버·네트워크 장비·엔드포인트·클라우드 자원의 로그
- 방화벽·IPS·AV·FIM·IAM 이벤트
- 탐지 룰 개발·튜닝·버전 관리
- SOAR 플레이북 운영

## 4. 입력 자료 / 산출물

- **Input**
  - 자산 목록·위협 인텔 피드
  - [[TMP-LOG-001_SIEM_룰_관리대장_v0.1]] 양식
- **Output**
  - 수집 구성(로그 소스·파서)
  - 탐지 룰 세트와 변경 이력
  - 월간 탐지 리포트·이상 탐지 케이스 레코드

## 5. 수행 절차

### 5.1 사전 준비

1. SecOps 가 자산·서비스별 로그 소스와 필수 필드를 목록화한다.
2. 시간 동기화(NTP) 를 모든 로그 소스에 적용한다.
3. 위협 인텔 피드(상업 + 오픈소스) 를 SIEM 에 연계한다.

### 5.2 수행 단계

1. **로그 수집**
   - 소스별 파서·정규화(CEF·ECS) 를 구성한다.
   - 수집 상태 대시보드로 누락 소스를 일일 점검한다.
2. **탐지 룰 개발**
   - MITRE ATT&CK 매핑을 포함해 룰을 작성한다.
   - 의료기기 특화 룰(대량 환자 데이터 조회·DICOM 비정상 통신) 을 추가한다.
   - [[TMP-LOG-001_SIEM_룰_관리대장_v0.1]] 에 룰 ID·목적·심각도·억제 조건을 등록한다.
3. **알림·분류**
   - 심각도 Critical/High/Medium/Low 4단계로 분류한다.
   - Critical 은 Oncall 즉시 호출, High 는 4시간 이내 대응, 이하 차일.
4. **1차 분석**
   - SecOps 가 False Positive 여부를 판단하고 티켓을 분류한다.
   - 침해 의심 시 [[PRO-MDCS-502_침해사고_대응_및_신고_절차_v1.0]] 로 에스컬레이션한다.
5. **SOAR 플레이북**
   - 반복 대응 패턴(로그인 실패 급증 → 계정 잠금, 악성 IP → 방화벽 차단) 을 플레이북으로 자동화한다.
   - 분기별 플레이북 효과 검증.
6. **룰 튜닝·유지**
   - False Positive 가 주간 10% 초과 룰은 튜닝 대상으로 지정한다.
   - 버전 관리(Git) 로 룰 변경 이력을 보존한다.

### 5.3 완료 조건

- [ ] 모든 핵심 자산·서비스의 로그가 SIEM 에 수집된다
- [ ] 탐지 룰셋이 MITRE ATT&CK 매핑을 포함한다
- [ ] Critical 알림에 대한 Oncall 대응 체계가 작동한다
- [ ] SOAR 플레이북이 최소 5개 이상 운영된다
- [ ] 룰 변경이 Git 에 버전 관리되며 변경 이력이 대장에 기록된다
- [ ] 월간 탐지 리포트가 작성·배포된다

## 6. 인터페이스 부서

- **CSIRT**: 침해 판정·대응 인계
- **인프라팀**: 로그 수집 에이전트 배포
- **IAM 관리자**: 계정 관련 이벤트 규칙 협업
- **PSO**: 의료기기 특화 위협 반영

## 7. 주의사항 / 예외 처리

### 7.1 로그 소스 장애

핵심 로그 소스(방화벽·AD) 가 60분 이상 수집 중단되면 SecOps 가 P1 티켓을 발행하고 임시 수집 경로(rsyslog 이중화) 를 가동한다. 수집 공백 기간은 월간 리포트에 명시한다.

### 7.2 False Positive 과다로 경보 피로

주간 False Positive 가 임계 초과 시 해당 룰을 자동 검토 큐에 배치한다. 2주 이내 튜닝 완료하지 못한 룰은 detect-only 모드로 임시 전환.

### 7.3 위협 인텔 누락으로 0-day 미탐

Zero-day 위협이 탐지에서 누락된 사례가 사후 발견되면 SecOps 가 근본 원인을 분석하고 신규 룰 또는 UEBA(행위 기반 탐지) 룰을 추가한다. 결과는 분기 탐지 성숙도 리포트에 반영.

### 7.4 SIEM 업그레이드·전환

SIEM 플랫폼 업그레이드·교체 시 구 룰셋을 신 플랫폼 문법으로 마이그레이션하고, 병행 운영 기간(최소 2주)을 두어 누락을 방지한다. 전환 완료 후 구 환경을 읽기 전용으로 30일 유지.
