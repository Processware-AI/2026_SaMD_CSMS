# STRIDE 위협모델링 업무지침 (WI-301-01)

> 상위 절차: [[PRO-MDCS-201_사이버보안_리스크_관리_절차_v1.0]] · §5 단계 2~3

## 1. 업무 목적

SaMD 제품의 설계 초기 단계부터 STRIDE(Spoofing·Tampering·Repudiation·Information Disclosure·Denial of Service·Elevation of Privilege) 프레임으로 위협과 공격 경로를 체계적으로 식별·문서화하고, 각 위협에 대응 통제를 매핑한다.

## 2. 수행 주체

- **주 수행자**: PSO, R&D 리드
- **검토자**: PSC, 임상 담당자
- **승인자**: CISO

## 3. 범위

- 신규 제품·주요 아키텍처 개정 시점
- AI 적용 기기의 경우 [[WI-301-06_AI_데이터_모델_위협평가_v0.1]] 와 통합 실시
- 시판 후 중대 변경(서비스·프로토콜 추가) 시 재수행

## 4. 입력 자료 / 산출물

- **Input**
  - 아키텍처 도면·데이터 흐름도(DFD)
  - 자산 목록·신뢰 경계
- **Output**
  - [[TMP-RISK-001_STRIDE_워크시트_v0.1]]
  - 위협 우선순위·통제 매핑 결과

## 5. 수행 절차

### 5.1 사전 준비

1. PSO 가 STRIDE 세션 진행 매뉴얼과 사례 라이브러리를 유지.
2. R&D 가 DFD 최신본을 준비(Level 0~2).
3. 세션 참여자는 PSO·R&D·QA·Safety·Clinical 최소 1인 이상.

### 5.2 수행 단계

1. **자산·진입점·신뢰경계 식별**
   - 자산(데이터·기능·키) 과 공격 진입점(네트워크·UI·파일·USB) 을 문서화.
2. **DFD 검토**
   - Level 2 DFD 에서 각 트러스트 바운더리 교차 지점을 식별.
3. **STRIDE 적용**
   - 각 교차 지점에 대해 S/T/R/I/D/E 6개 항목별로 위협을 도출.
4. **위협 기술**
   - 위협 기술: 공격자·공격 경로·목표·영향·가능성 기재.
   - 예: "외부 공격자가 웹 API 에 SQL Injection 으로 환자 데이터 Tampering, CVSS 8.5".
5. **통제 매핑**
   - 각 위협에 기존/계획 통제를 매핑.
   - 미통제 항목은 통제 설계로 이관.
6. **우선순위·수용**
   - DREAD 또는 CVSS + 임상 영향으로 우선순위 산정.
   - 잔여 위험을 [[WI-301-04_잔여_위험_수용_v0.1]] 로 연계.

### 5.3 완료 조건

- [ ] 자산·진입점·신뢰경계가 문서화되었다
- [ ] DFD 에서 모든 트러스트 바운더리 교차 지점이 검토되었다
- [ ] STRIDE 6개 항목별 위협이 식별되었다
- [ ] 각 위협에 공격자·경로·목표·영향이 기술되었다
- [ ] 위협-통제 매핑과 미통제 항목이 식별되었다
- [ ] 잔여 위험 수용 프로세스로 연계되었다

## 6. 인터페이스 부서

- **R&D**: 아키텍처·DFD 업데이트
- **Safety**: 임상 영향 평가
- **QA**: 테스트 설계에 위협 반영

## 7. 주의사항 / 예외 처리

### 7.1 DFD 누락·불일치

DFD 가 실 구현과 불일치하거나 누락 영역이 있는 경우 세션 중단 후 아키텍트가 재작성. 재세션 일정 확정.

### 7.2 AI 기기 추가 위협

AI 기기의 경우 STRIDE 외 데이터 중독·모델 회피 공격을 [[WI-301-06_AI_데이터_모델_위협평가_v0.1]] 로 병행 실시.

### 7.3 외부 IP 포함 제품

외부 라이선스 IP 포함 시 해당 IP 의 위협 기여 분석은 Black-box 접근. 공급사 문서에서 보안 가정을 추출하고 격리·모니터링 보완 통제 강화.

### 7.4 세션 시간 부족

대규모 시스템은 2~3일 집중 세션으로 분할. 중간 결과 백업과 참여자 순환 고려.
