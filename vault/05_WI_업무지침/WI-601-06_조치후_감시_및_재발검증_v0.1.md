# 조치 후 감시 및 재발검증 업무지침 (WI-601-06)

> 상위 절차: [[PRO-MDCS-601_취약점_감시_공개_조치_절차_v1.0]] · §5 단계 11~12 (조치 후 감시, 문서화·종결)

## 1. 업무 목적

취약점 패치·완화 조치 배포 완료 후, 동일 취약점 또는 변종 취약점의 재발 여부를 체계적으로 모니터링·검증함으로써 조치의 실효성을 확인하고, 검증 결과를 근거로 취약점 티켓을 종결하며 전 과정을 문서화하여 SaMD-CSMS 규제 요건을 충족한다.

## 2. 수행 주체

- **주 수행자**: PSO (감시 총괄, 종결 판단), SecOps 팀 (재발검증 스캔 실행, 로그 모니터링)
- **검토자**: 개발팀 (패치 기술 적합성 재확인), QA팀 (배포본 무결성·회귀 테스트 지원)
- **승인자**: CISO (취약점 종결 최종 승인)

## 3. 범위

- WI-601-05(패치 및 업데이트 배포)를 통해 배포 완료된 모든 취약점 조치 건
- Critical/High/Medium 심각도 취약점의 조치 후 검증 (Low는 다음 정기 스캔에서 병합 처리)
- SBOM 기반으로 동일 컴포넌트를 사용하는 다른 제품에 대한 영향 범위 확인
- 악용 징후 발생 시 PRO-MDCS-502(침해사고 대응) 연계

## 4. 입력 자료 / 산출물

- **Input**
  - 취약점 티켓 (접수번호·CVSS·영향 컴포넌트·패치 배포 완료 기록)
  - WI-601-05 배포 완료 기록 (배포 버전·배포 일시·대상 범위)
  - SBOM (동일 컴포넌트 사용 범위 확인)
  - 취약점 스캐너 정책 설정 (재발검증 스캔 프로파일)
- **Output**
  - [[TMP-VDP-060_조치후_감시_보고서_v0.1]] (심각도별 감시 기간·이상 이벤트 요약)
  - [[TMP-VDP-061_재발검증_스캔_기록_v0.1]] (스캔 일시·도구·결과·잔류 취약점 여부)
  - [[TMP-VDP-062_취약점_종결_확인서_v0.1]] (CISO 승인 포함)
  - 악용 징후 발견 시 PRO-MDCS-502 연계 IR 티켓

## 5. 수행 절차

### 5.1 사전 준비

1. PSO는 패치 배포 완료 기록을 수신한 후, 취약점 심각도에 따라 감시 기간 및 재발검증 스캔 일정을 확정한다.
   - Critical: 배포 후 7일 이내 1차 검증 스캔, 30일간 집중 감시
   - High: 배포 후 14일 이내 1차 검증 스캔, 60일간 감시
   - Medium: 배포 후 30일 이내 1차 검증 스캔, 90일간 감시
2. SecOps 팀은 취약점에 매핑된 SIEM 탐지 룰이 배포 후에도 활성화 상태임을 확인한다 (패치가 완료되더라도 동일 벡터의 재공격 시도를 탐지하기 위해 룰을 유지).
3. SBOM을 조회하여 패치된 컴포넌트와 동일 버전을 사용하는 다른 제품·시스템 목록을 확인하고, 필요 시 해당 제품에도 검증 스캔 대상을 확대한다.

### 5.2 수행 단계

1. **재발검증 스캔 실행**
   - SecOps 팀이 취약점 스캐너(Nessus, OpenVAS, Tenable 등)에 해당 취약점 CVE를 포함한 검증 스캔 프로파일을 구성하고, 배포 대상 시스템 전체에 대해 검증 스캔을 실행한다.
   - 스캔 결과를 [[TMP-VDP-061_재발검증_스캔_기록_v0.1]]에 기재한다. 항목: 스캔 일시·도구·대상 시스템·스캔 프로파일·결과(취약점 없음/재발 감지).
   - 패치가 적용되지 않은 잔류 시스템이 발견된 경우, 즉시 인프라팀에 통보하여 추가 배포를 완료하고 재스캔을 실행한다.

2. **SIEM 기반 악용 징후 모니터링**
   - 감시 기간 동안 SecOps 팀은 SIEM에서 해당 취약점과 관련된 탐지 룰의 알림 이벤트를 매일 검토한다.
   - 이상 이벤트(동일 CVE 관련 익스플로잇 시도, 비정상 트래픽 패턴) 발생 시:
     - 즉시 PSO에게 통보한다.
     - 취약점 티켓을 "재발 의심" 상태로 변경한다.
     - 악용이 확인된 경우 PRO-MDCS-502(침해사고 대응)를 트리거하고, 취약점 종결을 보류한다.

3. **변종 취약점 감시**
   - 감시 기간 동안 해당 취약점 컴포넌트에서 신규 CVE가 발표된 경우, SBOM 기반 영향 범위를 즉시 재평가하고 PRO-MDCS-601 단계 3(영향 분석)으로 돌아간다.
   - NVD, MITRE, 벤더 보안 공지 등 외부 취약점 데이터베이스를 주 1회 이상 조회하여 관련 컴포넌트의 신규 취약점 발표 여부를 확인한다.

4. **SBOM 기반 영향 확산 검증**
   - PSO는 패치된 컴포넌트가 SBOM에서 다른 제품에도 사용되고 있는지 재확인하고, 미패치 제품에 대한 조치 완료 현황을 취합한다.
   - 동일 컴포넌트를 사용하는 모든 제품에서 조치가 완료된 경우에만 해당 취약점을 종결 검토 대상으로 전환한다.

5. **조치 후 감시 보고서 작성**
   - 감시 기간 종료 시점에 PSO가 [[TMP-VDP-060_조치후_감시_보고서_v0.1]]를 작성한다.
   - 보고서 항목: 취약점 ID·패치 배포 일시·감시 기간·이상 이벤트 건수·재발검증 스캔 결과·SBOM 확산 범위 확인 결과·종결 권고 여부.

6. **종결 기준 확인 및 CISO 승인**
   - PSO가 다음 종결 기준을 모두 충족하였는지 체크하고 [[TMP-VDP-062_취약점_종결_확인서_v0.1]]를 작성한다.
     - 검증 스캔 결과 해당 CVE가 재검출되지 않았다
     - 감시 기간 동안 SIEM에서 관련 악용 징후 이벤트가 없었다
     - SBOM 기반 동일 컴포넌트 사용 제품 전체에서 조치가 완료되었다
     - 홈페이지 공개 및 식약처 통보가 완료되었다 (PRO-MDCS-601 단계 4, 9 연계)
   - CISO가 [[TMP-VDP-062_취약점_종결_확인서_v0.1]]를 검토하고 최종 승인하면 취약점 티켓을 "종결"로 처리한다.

7. **문서화 및 아카이브**
   - 취약점 관련 모든 기록(접수 대장·영향 분석·배포 기록·검증 스캔·종결 확인서)을 취약점 티켓에 링크하여 완결된 이력을 구성한다.
   - 아카이브된 기록은 최소 5년간 보관한다.

### 5.3 완료 조건

- [ ] 심각도별 기한 내 재발검증 스캔이 실행되고 [[TMP-VDP-061_재발검증_스캔_기록_v0.1]]이 작성되었다
- [ ] 감시 기간 동안 SIEM 이상 이벤트가 없었거나, 이벤트 발생 건에 대한 처리 기록이 존재한다
- [ ] SBOM 기반 영향 제품 전체에서 조치 완료가 확인되었다
- [ ] [[TMP-VDP-060_조치후_감시_보고서_v0.1]]가 PSO 작성 완료되었다
- [ ] [[TMP-VDP-062_취약점_종결_확인서_v0.1]]의 모든 종결 기준이 충족되고 CISO 승인이 기재되어 있다
- [ ] 취약점 티켓이 "종결" 상태로 처리되었다
- [ ] 관련 기록이 아카이브되었다

## 6. 인터페이스 부서

- **SecOps 팀**: 재발검증 스캔 실행, SIEM 감시, 악용 징후 탐지 및 에스컬레이션
- **개발팀**: 패치 적용 기술 확인, SBOM 업데이트
- **QA팀**: 검증 스캔 환경 지원, 기능 회귀 미발생 재확인
- **CISO**: 취약점 종결 최종 승인
- **RA (규제 담당)**: 식약처 조치 결과 통보 완료 확인
- **CSIRT**: 악용 징후 확인 시 침해사고 대응 연계

## 7. 주의사항 / 예외 처리

### 7.1 감시 기간 중 재발 탐지

검증 스캔 또는 SIEM 모니터링에서 동일 취약점이 재발한 것으로 확인된 경우, PSO는 즉시 개발팀과 원인을 분석하고 패치 실패 여부 또는 새로운 공격 벡터 출현 여부를 판단한다. 재발로 확인되면 WI-601-05(패치 및 업데이트 배포)를 재실행하고 감시 기간을 초기화한다. 악용이 수반된 경우 PRO-MDCS-502를 트리거한다.

### 7.2 레거시 제품의 패치 불가

레거시 제품에서 해당 취약점에 대한 패치가 기술적으로 불가하거나 제조사 지원이 종료된 경우, PSO는 CISO와 협의하여 다음 중 하나를 결정한다.
- 네트워크 분리·접근 통제 강화로 위험 완화 후 종결
- 완화 불가 시 식약처에 리스크 수용 사유를 포함한 추가 통보 후 종결
이 경우 [[TMP-VDP-062_취약점_종결_확인서_v0.1]]에 완화 조치 및 식약처 통보 내용을 별도 기재한다.

### 7.3 종결 후 동일 컴포넌트 신규 CVE 발표

취약점을 종결 처리한 후 동일 컴포넌트에서 신규 CVE가 발표된 경우, 이는 별도의 신규 취약점으로 PRO-MDCS-601 처음 단계(감시·접수)부터 재진행한다. 이전 종결 건과 연관성이 높은 경우 영향 분석에 이전 분석 결과를 참조 자료로 활용한다.

### 7.4 감시 기간 중 SIEM 룰 비활성화 요청

운영 부하 경감을 이유로 감시 기간 중 해당 탐지 룰의 비활성화를 요청하는 경우, PSO와 SecOps 팀장이 공동으로 검토하고 감시 기간 종료 전까지 룰 유지를 원칙으로 한다. 부득이한 경우 CISO 승인 아래 대체 모니터링 방안을 적용 후 룰을 조정한다.
