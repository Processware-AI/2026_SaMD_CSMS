## 7. 결론·권고

**결론**: 공격자는 피싱 또는 크리덴셜 스터핑을 통해 탈취한 VPN 계정으로 초기 침투하였으며, Mimikatz 변종으로 도메인 관리자 권한을 탈취한 후 LockBit 3.0 변종 랜섬웨어를 배포하였다. 환자 데이터 외부 유출 증거는 현재까지 확인되지 않았으나, 파일 서버 접근 이력이 존재하므로 유출 가능성을 배제할 수 없다.

**주요 권고사항**:
1. 탈취된 모든 계정 즉시 비활성화 및 전사 비밀번호 재설정
2. VPN 접속에 MFA(다중인증) 강제 적용
3. 내부 네트워크 마이크로 세그멘테이션 강화
4. LSASS 보호 모드(Protected Process Light) 활성화
5. 환자 데이터 접근 로그 전수 분석 후 유출 여부 최종 확인

## 작성 요령

- **증거 체인 오브 커스터디**: 모든 증거물에 해시값(sha256 이상)을 기재하고 봉인자와 인수인을 명확히 구분합니다. 해시값은 분석 전후로 일치해야 증거 무결성이 입증됩니다.
- **공격 타임라인 재구성**: 각 이벤트에 출처 로그와 확신도(높음/중간/낮음)를 기재합니다. 확신도가 낮은 항목은 추정 근거를 비고에 기재하세요.
- **MITRE ATT&CK 매핑**: Tactic과 Technique ID를 정확히 기재합니다. 공식 ATT&CK 사이트에서 ID를 조회하여 정확한 번호를 사용하세요.
- **IOC**: IP 주소와 도메인은 직접 클릭·접속되지 않도록 도메인은 `[.]`으로 치환하여 기재합니다. IOC는 KISA 등 외부 기관과 공유할 수 있습니다.
