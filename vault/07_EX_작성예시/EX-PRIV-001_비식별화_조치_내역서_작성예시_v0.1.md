# 비식별화 조치 내역서 작성예시 (EX-PRIV-001)

> 원본 양식: [[TMP-PRIV-001_비식별화_조치_내역서_v0.1]]

> **이 문서는 교육용 예시로, 실제 기록(REC)이 아닙니다.**

---

## 1. 데이터셋 개요

| 항목 | 내용 |
|---|---|
| 데이터셋명 | 혈당 연속 모니터링 로그 (CGM-LOG-2026-Q1) |
| 처리 목적 | AI 학습 (혈당 예측 모델 v2 훈련) |
| 레코드 수 | 약 1,200,000건 (환자 3,840명 × 평균 312일치 5분 간격 데이터) |
| 처리 기간 | 2026-04-01 ~ 2026-04-30 |
| 담당자 | 이다빈 (Data Science팀, Privacy Officer 위임) |

## 2. 식별자 분류

| 컬럼 | 분류 | 조치 기법 | 비고 |
|---|---|---|---|
| patient_id | 직접 식별자 | 가명(HMAC-SHA256 해시) | 매핑 테이블 별도 보관(KMS) |
| name | 직접 식별자 | 삭제 | 학습에 불필요 |
| phone | 직접 식별자 | 삭제 | 학습에 불필요 |
| birth_date | 준식별자 | 일반화 (연도만 유지) | 1985-07-23 → 1985 |
| zip_code | 준식별자 | 일반화 (앞 3자리 유지) | 06130 → 061** |
| glucose_value | 일반 | 유지 | 핵심 학습 피처 |
| timestamp | 준식별자 | 잡음 추가 (±5분 랜덤) | 시계열 패턴은 보존 |
| device_serial | 준식별자 | 가명(SHA-256 해시) | 기기 매핑 분리 보관 |
| diagnosis_code | 민감 | 마스킹 (3자리 이상 코드 → 대분류 2자리) | E11.x → E11 |

## 3. 적용 기법 상세

| 기법 | 파라미터 | 대상 컬럼 | 비고 |
|---|---|---|---|
| 가명(해시·매핑) | HMAC-SHA256, 키 ID: KMS-KEY-2026-03, 매핑 저장소: S3-internal-vault | patient_id, device_serial | 재식별 방지를 위해 매핑 테이블은 에어갭 스토리지 보관 |
| 일반화 | 생년 구간: 연도 단위; 우편번호 구간: 3자리 | birth_date, zip_code | K=4 충족 기준으로 설정 |
| 삭제 | 해당 컬럼 전체 드롭 | name, phone | 학습에 미사용, 규정상 필수 삭제 |
| 잡음 추가 | 정규분포 N(0, 5분), seed=42 | timestamp | 시계열 왜곡 최소화 |
| 마스킹 | 진단 코드 소분류(소숫점 이하) 제거 | diagnosis_code | ICD-10 대분류 수준 유지 |

## 4. 재식별 위험 평가

| 항목 | 값 |
|---|---|
| K-익명성 | K=5 (최소 5개 레코드가 동일 준식별자 조합) |
| L-다양성 | L=3 (진단코드 기준 민감 속성 최소 3가지 다양) |
| T-근접성(해당 시) | 미적용 (진단코드 마스킹 후 분포 왜곡 미미) |
| 차등 프라이버시 ε(해당 시) | ε=1.0 (타임스탬프 잡음 추가 적용) |
| 잔여 위험 수준 | 낮음 (Low) |
| 재식별 시도 테스트 결과 | 외부 DB(건강보험공단 샘플) 연결 재식별 시도 → 0건 매칭 (2026-04-10 완료) |

## 작성 요령

- **데이터셋 개요**: 레코드 수는 정확한 수치를 기재하고, 처리 목적은 진료/청구/연구/AI학습/통계 중 선택한다. 담당자는 Privacy Officer 또는 위임받은 담당자를 명시한다.
- **식별자 분류**: 모든 컬럼을 직접 식별자·준식별자·민감·일반로 분류하고, 조치 기법을 구체적으로 기재한다. 유지(미처리) 컬럼도 반드시 기재하여 의도적 결정임을 증명한다.
- **재식별 위험 평가**: K-익명성은 최소값 기준으로 기재하며, 외부 DB 연결 공격을 실제로 시도한 결과를 기재한다. 수치가 없으면 감사 통과가 어렵다.
- **승인**: 법적 검토와 오동석 최종 승인 모두 필수이며, 결과란에 '승인/반려/조건부승인'을 명확히 기재한다.

## 잘못된 기재 사례

### NG 사례 — 조치 기법 미기재·재식별 테스트 생략

| 컬럼 | 분류 | 조치 기법 | 비고 |
|---|---|---|---|
| patient_id | 직접 식별자 | 가명화 | |
| birth_date | 준식별자 | 일반화 | |
| name | 직접 식별자 | 삭제 | |

재식별 위험 평가: K-익명성 — 미측정, 재식별 시도 테스트 — 미수행

**문제점**: 가명화·일반화의 파라미터(해시 알고리즘, 구간 크기)가 없어 재현·검증 불가. K-익명성 미측정 및 재식별 시도 미수행 시 규제 요건(개인정보보호법 §28의2) 미충족.

**올바른 예시**: 기법 파라미터(예: HMAC-SHA256, 키 ID, 일반화 구간)를 명기하고, K-익명성 수치와 외부 연결 재식별 시도 결과를 반드시 기재한다.
