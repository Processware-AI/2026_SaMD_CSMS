# 침해사고 협조 요청서 작성예시 (EX-MSP-004)

> 원본 양식: [[TMP-MSP-004_침해사고_협조_요청서_v0.1]]

> **이 문서는 교육용 예시로, 실제 기록(REC)이 아닙니다.**

## 1. 작성 개요

서울메디컬센터에서 발생한 RadiAI 서버 랜섬웨어 침해사고(MSP-INC-2026-001)의 원인 분석을 위해, 제조업자등에게 기기 로그 분석 지원, 원격 접근 권한, 현장 기술 지원을 요청하는 사례이다. 초기 신고 완료 후 원인 분석 단계에서 제조업자등의 전문 지원이 필요하여 본 협조 요청서를 발송한다.

|---|---|

| 요청 일시 | 2026-02-21 10:00 KST |
| 요청 기관(병원명) | 서울메디컬센터 |
| 작성자 / 직위 | 정수아 / 의료정보보안팀장 |
| 연락처(전화/이메일) | 02-1234-5678 / security@seoulmc.example.kr |
| 관련 침해사고 신고 번호 | MSP-INC-2026-001 |

### 2.2 침해사고 요약

| 항목 | 내용 |

| 대상 기기/시스템 명칭 | RadiAI Diagnostic Assistant 서버 |

| 현재 대응 단계 | 원인분석 |

| 번호 | 협조 유형 | 요청 상세 | 희망 일정 |
|---|---|---|---|

| 2 | 원격지원 | 격리된 서버에 대한 원격 포렌식 분석 지원 -- 기기 내부 로그 구조 해석 및 악성코드 감염 경로 분석 | 2026-02-23~25 (3일간) |
| 3 | 현장지원 | 복구 단계 시 현장 방문하여 기기 재설치/무결성 검증 지원 | 원인 분석 완료 후 협의 |

### 2.4 접근 권한 부여 사항

| 항목 | 내용 |
|---|---|
| 접근 대상 시스템/영역 | RadiAI 서버 격리 환경 (포렌식 분석 전용 VLAN) |
| 접근 허용 기간 | 2026-02-23 09:00 ~ 2026-02-25 18:00 KST (3일간) |
| 접근 방식 | VPN (병원 제공 전용 VPN 계정) |
| 병원 측 안내 담당자 | 김보안 주임 (의료정보보안팀, 내선 3456) |
| 보안 준수 조건 | (1) 병원 정보보안 서약서 서명 필수, (2) 분석 데이터 외부 반출 금지, (3) 작업 종료 후 VPN 계정 즉시 폐기, (4) 환자 개인정보 접근 금지 |

## 작성 요령

1. 관련 침해사고 신고 번호(MSP-INC-)를 반드시 기재하여 선행 신고와의 연계를 명확히 한다.
2. 협조 유형별로 요청 사항을 분리하여 기재한다. 하나의 요청에 여러 유형을 혼합하면 제조업자등의 담당 배분이 어렵다.
3. 접근 권한 부여 시 **기간, 방식, 보안 조건**을 구체적으로 명시한다. 개방형 접근(기간 미정, 범위 무제한)은 금지한다.
4. 보안 준수 조건에는 정보보안 서약서, 데이터 반출 금지, 계정 폐기 일정을 포함한다.
5. 현장 방문 시 병원 측 안내 담당자를 사전 지정하여 보안 구역 출입 관리에 차질이 없도록 한다.

## 잘못된 기재 사례

| 구분 | 사례 |
|---|---|
| NG | 접근 권한을 "필요 시 상시 접근 가능"으로 기재 -- 기간/범위 미한정 |
| 올바른 예시 | "2026-02-23 09:00 ~ 2026-02-25 18:00 KST"로 기간 한정, 전용 VLAN만 접근 허용 |
| NG | 관련 침해사고 신고 번호 미기재로 선행 맥락 파악 불가 |
| 올바른 예시 | "MSP-INC-2026-001" 명기하여 신고서와 연계 |
