## 2. 요약(Summary)

MediGuard CGM App(혈당 연속 모니터링 앱) 클라우드 API 서버에서 JWT(JSON Web Token) 서명 알고리즘 검증 미흡으로 인해 인증을 우회하여 다른 환자의 혈당 데이터에 비인가 접근이 가능한 취약점이 발견되었습니다. 당사는 2026-03-05 외부 보안 연구자 신고로 해당 취약점을 인지하였으며, 2026-03-15 핫픽스 패치를 배포하여 조치를 완료하였습니다.

현재까지 실제 악용 사례는 확인되지 않았습니다.

## 4. 상세

| 항목 | 내용 |
|---|---|
| 취약점 설명(비악용 수준 요약) | JWT 라이브러리 설정 오류로 인해 `alg: none` (서명 없음) 토큰을 유효한 인증 토큰으로 처리하는 문제. 공격자가 임의 사용자 ID를 포함한 무서명 JWT를 생성하여 API에 전송할 경우 인증 절차를 우회할 수 있음. |
| CWE 분류 | CWE-347: Improper Verification of Cryptographic Signature |
| CVSS 기본 점수·벡터 | 9.8 / AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H |
| 악용 조건·범위 | 인터넷을 통해 API 서버에 접근 가능한 공격자. 특별한 권한 또는 사용자 상호작용 불필요. 영향 범위: 모든 환자 혈당 데이터 열람·위변조 가능. |

## 작성 요령

- **비악용 수준 요약**: 취약점 설명은 재현에 필요한 구체적 페이로드, 공격 코드를 포함하지 않는다. 취약점의 유형·영향 범위를 설명하되 공격 방법론은 생략한다.
- **CVSS 벡터**: 점수만이 아닌 전체 벡터 문자열을 기재하여 평가 근거를 투명하게 공개한다. 점수와 벡터가 일치하지 않으면 신뢰성이 훼손된다.
- **임시 완화(Workaround)**: 패치 적용이 즉시 불가능한 의료 환경을 고려하여, 기능 제한이 있더라도 즉시 적용 가능한 완화 방안을 제시한다.
- **개정 이력**: 공개 후 내용이 변경될 경우 버전을 올리고 변경 내용을 기재한다. 오탈자 수정도 이력에 포함한다.

## 3. 영향 제품 (NG)

| 제품 | 영향 버전 | 안전 버전 |
|---|---|---|
| MediGuard CGM App | 구버전 | 최신 버전 |

## 4. 상세 (NG)

| 항목 | 내용 |
|---|---|
| 취약점 설명 | JWT alg none을 이용해 인증 우회 가능. 공격자는 다음과 같이 토큰을 생성한다: `{"alg":"none"}`.`{"sub":"patient-1"}`.` ` |
| CVSS 기본 점수·벡터 | 9.8 |

**문제점**: 영향 버전 '구버전'·안전 버전 '최신 버전'은 사용자가 자신의 앱이 영향 버전인지 판단 불가. 취약점 설명에 공격자가 활용 가능한 토큰 생성 방법 노출(악용 수준 위반). CVSS 벡터 미기재.

**올바른 예시**: 영향 버전 `2.0.0 ~ 2.2.9`, 안전 버전 `2.3.0 (2026-03-15 배포)`, 취약점 설명은 '알고리즘 검증 미흡'으로 요약(구체적 페이로드 제외), CVSS 벡터 전체 문자열 기재.
