# 취약점 통보서 작성예시 (EX-MSP-005)

> 원본 양식: [[TMP-MSP-005_취약점_통보서_v0.1]]

> **이 문서는 교육용 예시로, 실제 기록(REC)이 아닙니다.**

## 1. 작성 개요

서울메디컬센터 의료정보보안팀이 정기 보안 점검 중 혈당 모니터링 SaMD 게이트웨이에서 기본 관리자 계정(admin/admin)이 변경되지 않은 상태로 운영되고 있음을 발견하였다. 해당 취약점은 비인가 접근을 통한 기기 설정 변경 및 환자 데이터 노출 위험이 있어, 제조업자등에게 즉시 통보한다.

| 항목 | 내용 |
|---|---|
| 대상 기기/시스템 명칭 | GlucoLink Gateway (혈당 모니터링 데이터 수집 게이트웨이) |
| 대상 기기 모델/버전 | GL-GW200 v1.3.0 (S/N: GL-2025-00412) |
| 소프트웨어/펌웨어 버전 | 펌웨어 v1.3.0, 관리 웹UI v1.1.2 |
| 취약점 발견 일시 | 2026-05-12 10:30 KST |
| 취약점 발견 경위 | 정기점검 (분기별 기기 보안 점검 시 기본 계정 변경 여부 확인 항목에서 발견) |
| 관련 CVE 번호 | 해당 없음 (제품 고유 설정 이슈) |
| 취약점 유형 | 인증우회 (기본 관리자 계정 미변경) |
| 취약점 상세 설명 | 관리 웹UI(포트 8080)에 기본 계정(admin/admin)으로 접근 가능. 초기 설치 시 강제 비밀번호 변경 메커니즘이 없어 기본 계정이 그대로 유지됨. 해당 계정으로 기기 설정 변경, 데이터 조회, 펌웨어 업데이트 모두 가능. |
| 재현 조건 | 관리 웹UI(https://{기기IP}:8080)에 admin/admin으로 로그인 시도 시 즉시 접근 가능 |

### 2.3 영향 평가(MSP 측 판단)

| 항목 | 내용 |
|---|---|
| 심각도(MSP 판단) | 높음 -- 네트워크 내부 접근 가능한 모든 사용자가 관리자 권한 획득 가능 |
| 환자 안전 영향 | 있음 -- 기기 설정 임의 변경 시 혈당 데이터 수집 오류로 임상 판단 오류 가능 |
| 악용 가능성 | 높음 -- 기본 계정은 널리 알려진 공격 벡터 |
| 현재 임시 대응 조치 | (1) 관리 웹UI 포트(8080) 방화벽 차단(병원 보안팀만 접근 허용), (2) 관리자 비밀번호 수동 변경 완료 |

## 작성 요령

1. 취약점 통보는 발견 즉시 수행한다. 정기점검 중 발견된 경우에도 점검 완료를 기다리지 말고 해당 항목을 먼저 통보한다.
2. CVE 번호가 없는 제품 고유 취약점의 경우 "해당 없음"으로 명기하고, 취약점 상세 설명과 재현 조건을 더욱 구체적으로 기재한다.
3. MSP 측 임시 대응 조치를 기재하여, 제조업자등이 근본 원인 해결(초기 설정 강제 변경 기능 등) 우선순위를 판단하는 데 참고하도록 한다.
4. 심각도 판단은 MSP 관점이므로 "MSP 판단"을 명시하며, 제조업자등의 최종 평가와 다를 수 있음을 인지한다.

## 잘못된 기재 사례

| 구분 | 사례 |
|---|---|
| NG | "비밀번호가 약한 것 같음" -- 구체적 내용 없이 모호하게 기재 |
| 올바른 예시 | "기본 관리자 계정(admin/admin)이 변경되지 않은 상태로 관리 웹UI 접근 가능"으로 구체적 기재 |
| NG | 취약점 발견 후 2주 경과 후 통보하면서 지연 사유 미기재 |
| 올바른 예시 | 발견 즉시(당일 이내) 통보하며, 부득이 지연 시 사유 명시 |
| NG | 재현 조건을 생략하여 제조업자등이 취약점 확인 불가 |
| 올바른 예시 | 접근 URL, 계정 정보, 재현 절차를 단계별로 기재 |
