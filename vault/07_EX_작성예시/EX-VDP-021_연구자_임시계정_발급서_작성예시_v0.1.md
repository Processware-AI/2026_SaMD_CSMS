## 작성 요령

- **계정명**: 발급 대상자의 실명이 드러나지 않는 가명 형식(`vdp-researcher-###`)을 사용한다. 계정명과 실제 신고자 매핑 정보는 PSO만 접근 가능한 별도 파일로 관리한다.
- **접속 범위**: 허용 환경(스테이징)과 접근 가능 엔드포인트를 구체적으로 명시한다. '스테이징 전체'처럼 광범위한 기재는 의도치 않은 내부 API 노출로 이어질 수 있다.
- **만료일**: 발급 후 14일이 원칙이며, 검증 기간이 더 필요한 경우 PSO 승인 후 1회 연장(최대 7일) 가능하다. 만료일 도래 시 자동 비활성화와 수동 확인을 병행한다.
- **회수 확인**: 자동 만료만으로는 충분하지 않다. 인프라팀이 계정 상태를 직접 확인하고 비활성화 완료를 서명으로 기재한다.
