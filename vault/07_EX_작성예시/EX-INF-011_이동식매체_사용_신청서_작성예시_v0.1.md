## 작성 요령

- **매체 정보**: 매체 고유 식별번호가 없는 매체는 자산 스티커를 부착하고 번호를 기재합니다. 암호화 USB 사용을 원칙으로 하며, 비암호화 매체 사용 시 조건부 승인 또는 반려 사유가 됩니다.
- **사용 계획**: 사용 기간은 최소화하고, 당일 반납을 원칙으로 합니다. 반출 데이터 범위는 파일명·버전까지 구체적으로 기재하세요.
- **보안 조치**: 보안 검사는 반드시 사전에 수행합니다. 악성코드가 탐지된 경우 즉시 반려하고 보안팀에 보고해야 합니다.
- **승인**: 팀장 검토 후 보안 담당자가 최종 승인합니다. 기밀 데이터 반출 시 오동석 승인이 추가로 필요할 수 있습니다.

## 잘못된 기재 사례

### NG 사례: 암호화 미적용 매체로 기밀 데이터 반출 신청

| 항목 | 내용 |
|---|---|
| 반출 데이터 분류 | 기밀 |
| 데이터 암호화 여부 | 미적용 |
| 승인 결과 | 승인 |

**문제점**: 기밀 데이터를 비암호화 매체로 반출하는 것은 데이터 유출 위험이 매우 높습니다. 이 경우 보안 담당자는 반드시 반려하고 암호화 매체 사용을 요구해야 합니다.

### 올바른 예시

| 항목 | 내용 |
|---|---|
| 반출 데이터 분류 | 기밀 |
| 데이터 암호화 여부 | 적용 (AES-256 하드웨어 암호화 USB) |
| 승인 결과 | 조건부승인 |
| 반려/조건부 사유 | 암호화 매체 사용 및 사용 직후 데이터 삭제 확인 조건 |
