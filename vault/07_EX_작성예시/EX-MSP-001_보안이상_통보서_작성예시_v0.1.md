# 보안이상 통보서 작성예시 (EX-MSP-001)

> 원본 양식: [[TMP-MSP-001_보안이상_통보서_v0.1]]

> **이 문서는 교육용 예시로, 실제 기록(REC)이 아닙니다.**

## 1. 작성 개요

서울메디컬센터 의료정보보안팀이 원격 환자 모니터링 시스템(SaMD)에서 비정상적인 외부 통신 시도를 감지하여, 해당 디지털의료기기 제조업자등에게 보안 이상을 통보하는 사례이다. 2026-03-15 새벽 SIEM 경보를 통해 이상 징후를 인지하였으며, 해당 기기를 네트워크에서 격리 후 제조업자등에 통보한다.

## 2. 작성 내용

| 항목 | 내용 |
|---|---|
| 통보 번호 | MSP-SEC-2026-003 |
| 통보 일시 | 2026-03-15 08:30 KST |
| 통보 기관(병원명) | 서울메디컬센터 |
| 작성자 / 직위 | 정수아 / 의료정보보안팀장 |
| 연락처(전화/이메일) | 02-1234-5678 / security@seoulmc.example.kr |
| 제조업자등 수신 담당 | 오동석 오동석 (보안팀, 010-9876-5432) |

### 2.2 보안 이상 내역

| 항목 | 내용 |
|---|---|
| 이상 감지 일시 | 2026-03-15 03:17 KST |
| 대상 기기/시스템 명칭 | CardiWatch Pro 원격 환자 모니터링 시스템 |
| 대상 기기 모델/버전 | CW-Pro v3.2.1 (S/N: CW-2025-00847) |
| 이상 유형 | 통신이상 |
| 이상 현상 상세 설명 | SIEM에서 해당 기기가 미등록 외부 IP(203.0.113.55, 포트 8443)로 5분 간격 반복 접속을 시도하는 경보 발생. 정상 통신 목록에 없는 대상이며, 기기 업데이트 일정도 없는 시간대에 발생. 총 23회 접속 시도 감지됨. |
| 감지 수단 | 모니터링시스템 (SIEM Critical 경보) |
| 환자 안전 영향 여부 | 미확인 -- 기기 격리 후 대체 모니터링 장비로 전환 완료 |
| 현재 기기 가동 상태 | 제한운영 (네트워크 격리, 로컬 모드 전환) |

## 작성 요령

1. 통보 번호는 병원 내부 보안 이벤트 관리 체계에 따라 연도별 순번으로 부여한다. 중복 번호가 발생하지 않도록 관리대장과 대조한다.
2. 이상 현상 상세 설명에는 **시각, 대상 IP/포트, 횟수, 패턴** 등 제조업자등이 원인 분석에 활용할 수 있는 정량적 정보를 포함한다.
3. 환자 안전 영향이 "미확인"인 경우, 현재 수행한 대체 조치(대체 장비 전환 등)를 함께 기재하여 환자 안전이 확보되었음을 명시한다.
4. 첨부 자료에 환자 개인정보가 포함되지 않도록 비식별화 처리 후 제공한다.
5. 제조업자등의 회신 기한은 사안의 긴급도에 따라 합리적으로 설정하되, 영업일 기준으로 명시한다.
