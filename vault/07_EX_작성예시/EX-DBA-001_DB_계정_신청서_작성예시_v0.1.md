## 작성 요령

- **신청 ID**: `DBA-REQ-{YYYY}-{###}` 형식을 반드시 준수한다. 번호는 DB 보안 담당자가 순번 채번하여 부여한다.
- **권한 범위**: `SELECT/INSERT/UPDATE/DELETE/DDL/DBA/기타` 중 실제 필요한 권한만 선택한다. 복수 권한이 필요한 경우 각 권한별 사유를 개별 기재한다.
- **접속 IP**: 단일 IP(/32)가 원칙이다. 동적 IP 환경이라면 VPN 출구 IP 또는 NAT IP를 기재하고 보안 검토 의견에 사유를 명시한다.
- **반출 데이터 범위**: 분석 목적이라도 개인식별 원시 데이터 반출은 별도 개인정보 영향평가를 거쳐야 한다. 집계값만 반출하는 경우 명확히 기재한다.

## 잘못된 기재 사례

### NG 사례 — 권한 범위를 DBA로 신청

| 항목 | 잘못된 기재 | 올바른 기재 |
|---|---|---|
| 권한 범위 | DBA | SELECT (audit_log_view, incident_summary_view 한정) |

DBA 권한은 DB 전체 제어 권한으로, 분석 목적에는 절대 부적합하다. 업무에 필요한 최소 권한(뷰 단위 SELECT)으로 신청해야 하며, DBA 권한 신청은 오동석 별도 승인이 필요하다.

### NG 사례 — 사용 기간 미기재

사용 기간 란을 공란으로 두면 만료 자동 회수 설정이 불가능하고, 해당 계정이 영구적으로 존재하게 된다. 업무 종료 예정일 또는 최대 1년 이내의 기간을 반드시 기재한다.
