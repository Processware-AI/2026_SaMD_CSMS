## 작성 요령

- **취약점 ID**: 외부신고 접수 대장(VDP-###)과 내부 취약점 접수 대장(VUL-###) 양쪽 ID를 모두 기재하여 추적성을 확보한다.
- **종결 사유**: '패치 완료'만으로는 부족하다. 재발검증 스캔 결과, 조치후 감시 기간 완료, 승인 여부를 포함한 4단계 완료를 기재한다.
- **종결 확인자**: PSO 단독 종결은 내부 규정에 따라 제한될 수 있다. Critical 취약점의 경우 오동석 공동 확인을 원칙으로 한다.
- **관련 기록 링크**: 종결 확인서는 취약점 생애주기 전체 기록(접수→분석→조치→검증→감시→종결)의 최종 집약 문서이므로, 각 단계 문서의 ID 또는 링크를 모두 연결한다.

## 잘못된 기재 사례

### NG 사례 — 종결 사유 단순·관련 기록 미연결

| 항목 | 내용 |
|---|---|
| 취약점 ID | VDP-001 |
| 최초 접수일 | 2026-03-05 |
| 조치 완료일 | 2026-03-15 |
| 종결 사유 | 패치 배포 완료 |
| 종결 확인자 | 김보안 |
| 관련 기록 링크 | — |

**문제점**: 종결 사유가 '패치 배포 완료'뿐으로, 재발검증 스캔·조치후 감시·승인 단계 생략 여부 불명확. 관련 기록 링크 미기재로 추적성 미확보. PSO 단독 확인으로 오동석 승인 누락.

**올바른 예시**: 종결 사유에 4단계(패치+재발검증 스캔+30일 감시+승인) 모두 기재, 관련 기록 ID 연결, 종결 확인자에 PSO+오동석 모두 기재.
