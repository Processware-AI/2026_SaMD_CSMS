# 재발검증 스캔 기록 작성예시 (EX-VDP-061)

> 원본 양식: [[TMP-VDP-061_재발검증_스캔_기록_v0.1]]

> **이 문서는 교육용 예시로, 실제 기록(REC)이 아닙니다.**

---

## 1. 스캔 기본 정보

| 항목 | 내용 |
|---|---|
| 취약점 ID | VDP-2026-001 / VUL-2026-001 (JWT alg:none 인증 우회) |
| 검증 도구 | OWASP ZAP 2.14.0 + 커스텀 스크립트 (JWT alg:none 탐지 룰) |
| 스캔 일시 | 2026-03-16 10:00 ~ 11:15 KST |
| 스캔 범위 | MediGuard CGM API 서버 스테이징 (api.mediguard-stg.local:443) — /v2/* 엔드포인트 전체 |
| 검증자 | 정보안 (보안팀 선임) |

## 2. 스캔 결과

| 항목 | 내용 |
|---|---|
| 취약점 재탐지 여부 | 탐지 안됨 |
| 결과 판정 | 검증 완료 |

---

## 작성 요령

- **검증 도구**: 최초 취약점 탐지에 사용한 도구와 동일하거나 동등 수준의 도구를 사용한다. 도구명과 버전, 특히 커스텀 룰이 있는 경우 버전·파일명도 기재한다.
- **스캔 범위**: 취약점이 발생한 구체적인 엔드포인트와 환경(운영/스테이징)을 명기한다. 스테이징에서 검증 후 운영 배포 전 운영 환경 재검증이 필요하다면 그 계획도 기재한다.
- **결과 판정**: '탐지 안됨 → 검증 완료', '탐지됨 → 재조치 필요' 두 가지만 기재하며, 모호한 표현(예: '대체로 안전함')은 사용하지 않는다.
- **연계**: 스캔 기록은 조치후 감시 보고서(EX-VDP-060)와 취약점 종결 확인서(EX-VDP-062)에서 참조 문서로 활용된다. 스캔 기록 ID(예: VDP-SCN-2026-001)를 부여하여 추적을 용이하게 한다.

---

## 잘못된 기재 사례

### NG 사례 — 스캔 도구 미기재·결과 모호

| 항목 | 내용 |
|---|---|
| 취약점 ID | VDP-2026-001 |
| 검증 도구 | 스캐너 |
| 스캔 일시 | 2026-03-16 |
| 스캔 범위 | API 서버 |
| 검증자 | 정보안 |

| 항목 | 내용 |
|---|---|
| 취약점 재탐지 여부 | 괜찮음 |
| 결과 판정 | 완료 |

**문제점**: 검증 도구 '스캐너'는 어떤 도구인지 특정 불가. 스캔 범위 'API 서버'로 엔드포인트 미특정. 재탐지 여부 '괜찮음'은 표준 표현이 아니어서 의미가 불명확. 이 기록으로는 감사 추적이 불가능하다.

**올바른 예시**: 검증 도구 `OWASP ZAP 2.14.0`, 스캔 범위 `/v2/* 엔드포인트`, 재탐지 여부 `탐지 안됨`, 결과 판정 `검증 완료` 기재.
