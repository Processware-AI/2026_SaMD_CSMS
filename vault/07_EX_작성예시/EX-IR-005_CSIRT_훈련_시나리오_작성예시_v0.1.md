## 2. 시나리오 스토리

### 2.1 배경

AI 판독 보조 제품의 추론 API 에 사용 중인 오픈소스 컴포넌트(ProtoLib v1.2.3) 에 대한 0-day RCE 취약점이 보안 커뮤니티에 공개됨. CVE 발급은 아직 없으며 CVSS 환산 9.1 Critical. 악용 PoC 가 수 시간 내 공개될 것으로 예상.

### 2.2 트리거 이벤트(시각별)

| T | 이벤트 | 참여자에 제공되는 정보 |
|---|---|---|
| T+0 | 위협 인텔 피드 경보 수신 | CVE 미발급 상태, GitHub Issue 링크·PoC 스니펫 |
| T+30m | SBOM 매칭 결과 — 당사 3개 제품에 해당 컴포넌트 포함 | 영향 제품·버전 리스트, MSP 17곳 |
| T+1h | SIEM 이상 트래픽 탐지(의심 exploit 시도) | IP 2개, 대상 서버 2대, 시도 500회 |
| T+2h | 관리자 계정 1개의 비정상 로그인 성공 | 계정명, 로그인 시각, 관리자 콘솔 API 호출 이력 |
| T+4h | 언론 보도 문의 접수 | 기자 질의 내용 요약 |

## 작성 요령

- 시나리오는 **실재 가능성 있는 현업 스토리** 로 작성. 추상적 서술 금지.
- 평가 기준은 정량 지표(분·건수·%) 중심.
- 훈련 결과는 반드시 WI·PRO 개선 과제로 이관.
- 훈련 실시 전 시나리오 내용은 **퍼실리테이터 외 비공개** 로 관리한다. 시나리오 파일은 훈련 종료 48시간 후에만 참가자에게 공유하며, 사전 유출 여부를 훈련 계획서에 기록한다. 이를 통해 참가자의 즉흥 판단력과 절차 준수 여부를 실질적으로 평가한다.
