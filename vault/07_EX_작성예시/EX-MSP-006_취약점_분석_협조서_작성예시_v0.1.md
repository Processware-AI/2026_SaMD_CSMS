# 취약점 분석 협조서 작성예시 (EX-MSP-006)

> 원본 양식: [[TMP-MSP-006_취약점_분석_협조서_v0.1]]

> **이 문서는 교육용 예시로, 실제 기록(REC)이 아닙니다.**

## 1. 작성 개요

서울메디컬센터에서 통보한 HeartAI ECG Analyzer 인증 우회 취약점(CVE-2026-10234, MSP-VUL-2026-003)에 대해, 제조업자등이 패치 개발 및 테스트를 위한 협조를 요청하였다. 서울메디컬센터는 테스트 환경 접근 권한 제공, 기기 운영 로그 공유, 패치 적용 테스트 지원 등의 협조 내역을 정리하여 제출한다.

## 작성 요령

1. 관련 취약점 통보 번호(MSP-VUL-)와 제조업자등 요청 번호를 모두 기재하여 양방향 추적성을 확보한다.
2. 제조업자등에 제공하는 로그, 환경 정보에는 환자 개인정보가 포함되지 않도록 **비식별화 처리 후** 제공한다. 비식별화 완료 여부를 협조 제공 내역에 명시한다.
3. 패치 적용 일정은 임상 업무 영향을 최소화하는 시간대(야간/주말)를 우선 검토하고, 해당 진료과와 사전 협의한다.
4. 롤백 계획은 반드시 수립하며, 롤백 시 임시 완화 조치 재적용 계획까지 포함한다.
5. 접근 권한 부여 후 제조업자등의 작업 내역을 병원 측 안내 담당자가 확인하고 기록한다.

| NG | 패치 적용 일정을 평일 진료 시간대(09:00~17:00)로 설정하여 임상 업무 중단 위험 |
| 올바른 예시 | 주말 새벽(06:00~08:00) 등 임상 영향 최소 시간대 선정 |
| NG | 롤백 계획 없이 패치 적용 -- 문제 발생 시 대응 불가 |
| 올바른 예시 | 사전 백업 + 30분 내 롤백 기준 + 임시 완화 조치 재적용 계획 수립 |
