# 조치후 감시 보고서 작성예시 (EX-VDP-060)

> 원본 양식: [[TMP-VDP-060_조치후_감시_보고서_v0.1]]

> **이 문서는 교육용 예시로, 실제 기록(REC)이 아닙니다.**

---

## 1. 감시 기본 정보

| 항목 | 내용 |
|---|---|
| 취약점 ID | VDP-2026-001 / VUL-2026-001 (JWT alg:none 인증 우회) |
| 패치 배포일 | 2026-03-15 (MediGuard CGM App v2.3.0 핫픽스 배포) |
| 감시 기간 | 2026-03-15 ~ 2026-04-15 (패치 후 30일) |
| 담당자 | 김보안 (PSO) |

## 2. 감시 결과

| 항목 | 내용 |
|---|---|
| 재발 탐지 여부 | 없음 |
| 검증 스캔 결과 | DAST 재스캔 (ZAP 2026-03-16 실행): JWT alg:none 취약점 미탐지. 재발검증 스캔 기록 VDP-SCN-2026-001 참조. |
| SIEM 이상 여부 | 없음 — 감시 기간 중 `/v2/*` 엔드포인트에서 alg:none 토큰 사용 시도 0건 (SIEM 룰 #SIEM-JWT-001 모니터링 결과) |

---

## 작성 요령

- **감시 기간**: 패치 배포 후 최소 30일 감시를 권장한다. Critical 취약점의 경우 90일 감시를 수행하고 기간을 기재한다.
- **재발 탐지 여부**: 단순 '없음' 기재로는 부족하다. SIEM 룰 번호, 검증 스캔 결과 문서 ID 등 객관적 근거를 연결하여 기재한다.
- **검증 스캔 결과**: 재발검증 스캔 기록(EX-VDP-061)과 연번을 연결하여 별도 상세 기록을 참조할 수 있도록 한다. 스캔 날짜와 도구 버전도 간략히 기재한다.
- **SIEM 이상 여부**: SIEM 모니터링 결과를 '없음'으로 기재 시 해당 기간 동안 SIEM 룰이 정상 동작했는지 확인하고, 룰 ID를 기재하여 추적 가능하게 한다.

## 잘못된 기재 사례

### NG 사례 — 감시 기간 미기재·근거 없는 결과

| 항목 | 내용 |
|---|---|
| 취약점 ID | VDP-2026-001 |
| 패치 배포일 | 2026-03-15 |
| 감시 기간 | — |
| 담당자 | 김보안 |

| 항목 | 내용 |
|---|---|
| 재발 탐지 여부 | 없음 |
| 검증 스캔 결과 | 이상 없음 |
| SIEM 이상 여부 | 없음 |

**문제점**: 감시 기간 미기재로 패치 후 얼마나 감시했는지 확인 불가. 검증 스캔 결과 '이상 없음'에 스캔 문서 ID 연결 없어 근거 불충분. SIEM 룰 번호 없어 실제 모니터링 수행 여부 확인 불가.

**올바른 예시**: 감시 기간 `2026-03-15 ~ 2026-04-15`, 검증 스캔 결과에 `VDP-SCN-2026-001 참조`, SIEM 이상 여부에 `SIEM 룰 #SIEM-JWT-001 모니터링 결과` 연결.
