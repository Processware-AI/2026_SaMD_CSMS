# 침해사고 신고 절차서 작성예시 (EX-MSP-003)

> 원본 양식: [[TMP-MSP-003_침해사고_신고_절차서_v0.1]]

> **이 문서는 교육용 예시로, 실제 기록(REC)이 아닙니다.**

## 1. 작성 개요

서울메디컬센터에서 운영 중인 AI 기반 영상진단 보조 SaMD 서버에 랜섬웨어 감염이 확인되었다. 환자 영상 데이터 접근이 차단되어 임상 업무에 직접 영향이 발생하였으며, 의료정보보안팀은 침해행위 발생 사실을 식품의약품안전처와 제조업자등에게 즉시 신고한다.

## 작성 요령

1. 식품의약품안전처와 제조업자등에 대한 신고는 가능한 한 동시에 수행하되, 신고 방법(전화/이메일/온라인)과 정확한 신고 일시를 각각 기록한다.
2. 침해 경위 요약은 **시간순**으로 핵심 이벤트를 나열하되, 추정과 확인된 사실을 명확히 구분한다 ("조사 중", "확인됨" 등 표기).
3. 개인정보 유출 여부는 포렌식 분석 전까지 "미확인"으로 유지하고, 확정 판단을 기재하지 않는다.
4. 환자 안전 영향이 있는 경우 대체 운영 조치를 반드시 함께 기재하여 환자 진료 연속성이 확보되었음을 명시한다.
5. 신고 후 식품의약품안전처 및 제조업자등의 후속 지시/회신은 별도 기록(REC)으로 관리한다.

## 잘못된 기재 사례

| 구분 | 사례 |
|---|---|
| NG | 식품의약품안전처 신고 없이 제조업자등에만 통보 |
| 올바른 예시 | 식품의약품안전처와 제조업자등에 동시 신고하고 각각의 신고 일시/방법 기록 |
| NG | 개인정보 유출 여부를 "없음"으로 확정 기재 (포렌식 미완료 상태) |
| 올바른 예시 | "미확인 -- 포렌식 분석 필요"로 기재 |
| NG | 침해 인지 후 12시간 경과 후 신고하면서 지연 사유 미기재 |
| 올바른 예시 | 즉시 신고하되, 부득이 지연 시 사유와 경과 시간을 비고란에 명시 |
