# AI 데이터·모델 위협평가 워크시트 작성예시 (EX-RISK-003)

> 원본 양식: [[TMP-RISK-003_AI_위협평가_워크시트_v0.1]]

> **이 문서는 교육용 예시로, 실제 기록(REC) 이 아닙니다.**

## 1. 모델 개요

| 항목 | 내용 |
|---|---|
| 모델명·버전 | RadiologyCNN v2.0 |
| 임상 용도 | 흉부 X-ray 결핵 의심 병변 보조 판독 |
| 학습 프레임워크 | PyTorch 2.1, Torchserve |
| 학습 데이터 규모 | 120만 영상(학습) / 20만(검증) |
| 추론 방식 | 하이브리드(병원 엣지 1차 + 클라우드 2차) |
| 재학습 주기 | 6개월 정기 + 임상 피드백 기반 수시 |

## 2. 데이터 위협

| 항목 | 위협 | 대응 통제 | 잔여 위험 |
|---|---|---|---|
| 데이터 중독 | 악성 병원 커뮤니티가 편향 데이터 기여 | 소스 신뢰성 계약 + 이상 샘플 감지 + 적대적 학습 | Low(3.0) ← 기여 소스 제한적 |
| 데이터 유출 | 학습 데이터셋 저장소 탈취 | KMS 암호화 + 최소권한 + 접근 감사 | Medium(5.1) |
| 데이터 무결성 | 학습 배치 체크섬 불일치 | SHA-256 해시·디지털 서명 | Low(2.5) |
| 라벨 공격 | 어노테이션 도구 탈취로 라벨 조작 | 어노테이터 MFA·2인 검토·라벨 이력 보존 | Medium(4.7) |

## 3. 모델 위협

| 항목 | 위협 | 대응 통제 | 잔여 위험 |
|---|---|---|---|
| 모델 추출 | API 반복 질의로 모델 복제 | API 호출 제한(분당 100) + 모델 워터마크 | Medium(4.3) |
| 적대적 회피 | 교란 입력으로 오진 유발 | 적대적 학습 + 입력 이상 탐지 | Medium(5.2) ← 주기적 재평가 |
| 반복 질의 | 자동화된 쿼리로 결정 경계 추론 | 동일 IP 쿼리 제한, 레이트 리미터 | Medium(4.0) |
| 멤버십 추론 | 특정 환자 데이터 학습 포함 여부 추정 | DP-SGD(ε=3.0, δ=1e-5) | Medium(4.8) |
| 속성 추론 | 환자 민감 속성 추론 | 비식별화 + DP 노이즈 | Low(3.2) |
| Prompt Injection(LLM) | 해당 없음 — 비-LLM 모델 | — | N/A |

## 4. Fail-safe / 대안 운영 모드

| 항목 | 설계 | 검증 결과 |
|---|---|---|
| 이상 피드백 화면 | 신뢰도 60% 미만 시 경고 배너 표시 | 통과 ← 임상 검증 100건 표본 |
| 이용자 안내 메시지 | "참고 보조 정보로만 활용" 고정 문구 | 통과 |
| 안전 중단(Fail-safe) | 모델 연속 실패 3회 시 자동 중단 | 통과 |
| 대안 운영 모드 | 모델 중단 시 '수동 판독 모드' 전환 | 통과 |
| 임상 안전 모드 | 응급 케이스 규칙 기반 결정 트리 제공 | 통과 |

## 5. 프라이버시 보호

| 항목 | 적용 여부 | 파라미터 |
|---|---|---|
| DP-SGD (ε, δ) | 예 | ε=3.0, δ=1e-5 ← 연구 기준 중간 보호 |
| 차등 프라이버시 | 예 | DP-SGD 와 동일 |
| 합성 데이터 | 부분 — 외부 공개용 벤치마크에만 | 합성 비율 30% |

## 작성 요령

- LLM 미적용 제품에서는 Prompt Injection 칸에 "N/A" 명시.
- DP-SGD 파라미터는 임상 성능과 프라이버시의 균형점 기록(ε·δ 값 명시).
- Fail-safe 검증 결과는 반드시 임상 담당자 표본 리뷰 포함.
- 재학습 이벤트(정기·수시) 발생 시마다 이 워크시트를 **재평가** 하고 버전 번호를 증가시킨다. 재학습으로 인해 위협 점수나 잔여 위험이 변동된 항목은 변경 사유(학습 데이터 추가·하이퍼파라미터 변경 등)를 해당 행 비고에 기재하여 모델 이력과 위협 이력을 연결한다.
