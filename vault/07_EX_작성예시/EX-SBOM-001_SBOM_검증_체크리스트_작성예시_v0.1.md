## 작성 요령

- "Unknown 공급자" 가 있으면 사유(지역 오픈소스·맞춤 패키지 등) 를 비고에 기재.
- 이전 릴리스 Diff 는 자동 생성 결과를 그대로 붙이지 말고, 보안·라이선스 관점 요약을 추가.
- Critical CVE 0건이 출시 필수. 1건 이상이면 즉시 [[WI-601-05_패치_및_업데이트_배포_v0.1]] 로 이관.
- SBOM 생성 도구는 버전을 명시하고, 빌드마다 동일 버전의 도구를 사용했는지 CI/CD 파이프라인 로그로 확인한다. 도구 버전이 변경된 경우 Diff 섹션에 변경 사유와 재검증 결과를 기재하여 SBOM 의 재현성을 보장한다.
