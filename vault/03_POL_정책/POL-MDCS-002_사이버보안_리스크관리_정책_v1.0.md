# 사이버보안 리스크 관리 정책 (POL-MDCS-002)

> 상위 방침: [[POL-MDCS-001_의료기기_사이버보안_기본방침_v1.0]]

## 1. 목적

본 정책은 디지털의료기기 수명주기 전반에서 **사이버보안 위협·취약점을 식별·평가·통제**하고, **임상 안전(Safety)과 제품 보안(Security)을 연계**하여 환자에게 발생 가능한 위해를 허용 수준 이하로 관리하기 위한 리스크 거버넌스 체계를 정의한다.

## 2. 적용 범위

- 디지털의료기기의 **설계·개발·검증·출시·운영·유지·레거시·폐기** 전체 수명주기
- 자사 개발 구성요소와 **제3자·오픈소스 소프트웨어**(SBOM 기반)
- **AI 적용 기기**의 경우 데이터 공격·모델 공격 등 AI 특화 위협 포함 (SaMD-CSMS 제12조)
- 시판 후 취약점·공격 기법 변화에 따른 **Post-market 지속 감시**

## 3. 정책 원칙

1. **Safety-Security 통합** — ISO 14971 기반 안전 리스크 관리와 사이버보안 리스크 평가를 **단일 프레임워크**로 연계하며, 보안 위협이 환자 안전에 미치는 영향을 최우선 평가 기준으로 삼는다.
2. **위협 모델링 의무화** — 설계 초기 단계부터 STRIDE 등 체계적 위협 모델링을 수행하며, 공격 표면·진입점·자산·신뢰 경계를 문서화한다.
3. **정량·정성 평가 병행** — 심각도(CVSS) 및 임상 영향·악용 가능성·탐지 용이성을 종합하여 위험 등급을 판정한다.
4. **잔여 위험의 경영 수용** — 통제 조치 후 잔여 위험은 PSC 가 공식 수용하며, 수용 불가 시 추가 통제·기능 제약·출시 연기를 결정한다.
5. **지속 감시와 재평가** — 시판 후에도 취약점 감시·침해 사례·신규 공격 기법을 반영하여 **연 1회 이상** 위험 평가를 갱신한다.

## 4. 역할과 책임

| 역할 | 책임 |
|---|---|
| **CEO** | 본 정책 승인, 중대 잔여 위험의 최종 수용 결정 |
| **오동석** | 본 정책 유지, 리스크 평가 방법론 유효성 관리 |
| **PSC** | 위험 수용·예외·완화 계획 심의, 분기별 리스크 포트폴리오 검토 |
| **Product Security Officer** | 제품별 위협 모델링·위험 평가 수행, Safety 팀과의 연계 보증 |
| **R&D / 개발팀** | 위협 모델링 참여, 통제 설계·구현 |
| **Safety / QA** | ISO 14971 안전 리스크와 연계 평가 |
| **Regulatory Affairs** | 시판 후 감시 결과의 규제 보고·갱신 |

## 5. 준수 기준

- 모든 제품은 **출시 전**에 위협 모델링·위험 평가·잔여 위험 수용 결정을 완료한다.
- SBOM 기반의 CVE 추적을 주기적으로 수행하여 취약점을 위험 평가 대상에 반영한다.
- 위험 평가 보고서는 위협·취약점·통제·잔여 수준·수용 근거를 포함하여 [[PRO-MDCS-201_사이버보안_리스크_관리_절차_v1.0]] 에 따라 작성한다.
- 시판 후 **중대 신규 위협** 발견 시 14일 이내 긴급 재평가를 실시하고 필요 시 사용자 공지·패치를 수행한다.
- AI 적용 기기는 데이터·모델에 대한 위협 시나리오를 위험 평가에 포함한다.
