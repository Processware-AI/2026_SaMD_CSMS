# 전자적 침해행위 대응 정책 (POL-MDCS-005)

> 상위 방침: [[POL-MDCS-001_의료기기_사이버보안_기본방침_v1.0]]

## 1. 목적

본 정책은 디지털의료기기 및 운영 환경에 대한 **전자적 침해행위(랜섬웨어·데이터 유출·서비스 마비·악성코드 감염 등)** 를 조기에 탐지하고, 법령이 요구하는 시한 내 신고·대응·사후 재발 방지 조치를 일관되게 수행하기 위한 거버넌스를 정의한다.

## 2. 적용 범위

- 당사 제품과 운영 환경(개발·운영 시스템, 고객 배포 제품 포함)
- 침해행위 **탐지 → 분석 → 대응 → 복구 → 사후 조치** 전 단계
- **24시간 비상 연락 체계** 및 식품의약품안전처·의료서비스제공자·이용자 통지
- 의료서비스제공자(MSP)와의 **협력·공동 대응** 활동

## 3. 정책 원칙

1. **상시 탐지(Always-On Detection)** — 운영 환경 전반에 보안 모니터링을 구현하고, 이상 징후·비정상 접근·잠재 위협을 실시간 탐지한다.
2. **신속·투명 신고(Timely & Transparent Notification)** — 침해행위 인지 즉시 식품의약품안전처장·의료서비스제공자·이용자에게 지체 없이 알리며, 조치 상황을 지속 공유한다.
3. **환자 안전 우선(Patient Safety First)** — 대응 조치 결정 시 임상 안전·제품 사용 연속성(BCP)·환자 피해 최소화를 최우선 기준으로 한다.
4. **증거 보전과 포렌식(Preserve Evidence)** — 침해 대응 초기 단계부터 로그·이미지·메모리 덤프 등 증거를 변조 방지 상태로 보전한다.
5. **학습과 개선(Learn & Improve)** — 모든 침해사고는 원인 분석·재발 방지 대책·교훈을 문서화하고 이후 대응 계획·제품 설계·교육에 반영한다.

## 4. 역할과 책임

| 역할 | 책임 |
|---|---|
| **CEO** | 본 정책 승인, 중대 침해 대외 의사결정 |
| **오동석** | 본 정책 유지, CSIRT 운영 감독 |
| **CSIRT 리더** | 침해행위 지휘 통제, 초기 분석·격리·복구 결정 |
| **CSIRT 팀원** | 24시간 비상 대기, 탐지·포렌식·복구 실행 |
| **SecOps** | SIEM·로그·안티바이러스·백업 운영, 이상 탐지 1차 대응 |
| **Regulatory Affairs** | 식품의약품안전처 신고, 규제 보고서 작성 |
| **홍보/고객지원** | 이용자·의료서비스제공자 커뮤니케이션 |
| **Privacy Officer (DPO)** | 개인정보 유출 여부 판단·개인정보보호위원회 신고 연계 |

## 5. 준수 기준

- **보안 모니터링**: 불필요 서비스·포트 식별·비활성화, 이상 탐지 구현, 주요 운영 데이터 정기 백업·무결성 검증·주기적 복구 모의훈련.
- **로그 관리**: 보안 활동·시스템 이벤트 로그를 변조 방지 상태로 보관·정기 분석.
- **안티바이러스**: 운영 환경 컴퓨터에 최신 버전 AV 설치·자동 업데이트·정기 스캔.
- **대응 계획 수립**: 시나리오 기반 대응 계획을 문서화하며 신고 방안·조치 방안·위험 분석·CSIRT 구성·BCP·훈련 계획을 포함한다.
- **신고 시한**: 침해행위 인지 즉시 식품의약품안전처장·MSP·이용자에게 알리며, 24시간 비상 연락 체계를 유지한다.
- **초기 신고 정보**: 침해 유형, 발생 시간, 인지 시점, 초기 영향 범위, 현재 조치 상황을 표준 양식으로 제출한다.
- **지속 공유**: 신고 이후에도 조치 상황·추가 분석·재발 방지 대책을 관련 대상에게 계속 공유한다.
- **BCP 연계**: 제품 사용 연속성을 위해 비상 운영·백업·대체 시스템 가동·AI 기기 Fail-safe 를 실행한다.
- **훈련**: Tabletop·Simulation 훈련을 주기적으로 실시하고 결과를 계획 개선에 반영한다.
