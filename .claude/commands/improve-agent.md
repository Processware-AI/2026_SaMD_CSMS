# /improve-agent — 에이전트 개선 슬래시 커맨드

## 목적
`.claude/agents/` 의 에이전트 정의 파일에서 **재발한 문제 유형**을 찾아 규칙을 보강한다.
개선 대상 에이전트와 문제 증상을 인자로 받아, 현재 정의를 읽고 → 근본원인을 분석하고 → 규칙을 추가/수정한다.

## 사용법

```
/improve-agent <에이전트명> "<증상 또는 누락된 행동 설명>"
```

예시:
```
/improve-agent wi-tmp-writer "WI와 TMP를 일부만 생성하고 나머지를 누락함"
/improve-agent process-designer "PRO 번호가 상위 POL 번호와 불일치하게 부여됨"
/improve-agent traceability-mapper "MAT 파일을 갱신하지 않고 완료 보고함"
```

## 수행 절차

### Step 1. 에이전트 현재 정의 읽기
`.claude/agents/<에이전트명>.md` 파일을 Read한다.

### Step 2. 근본원인 분석
증상을 보고 현재 정의의 어느 규칙이 **부재·모호·우선순위 미정의** 상태인지 찾는다.
분석 항목:
- 완전성 강제 규칙 존재 여부 (전체 생성 vs 일부 생성)
- 완료 검증 단계 존재 여부 (생성 후 카운트 대조)
- 예외·누락 시 복구 규칙 존재 여부
- 출력물 명명 규칙 명확도 (버전 접미사, 경로 패턴 등)
- 선행/후행 조건 명시 여부

### Step 3. 보강 규칙 초안 작성
근본원인별로 추가할 규칙을 초안으로 작성한다.
규칙 작성 원칙:
- **단언형**: "~한다", "~금지" — 판단 여지를 남기지 않음
- **검증 가능**: 수치(건수, 파일 개수)나 명령어(ls | wc -l, grep)로 확인 가능한 조건
- **위치 명시**: 어느 Phase/Step 앞뒤에 삽입할지 지정
- **재발 방지**: 문제가 재발했을 때 에이전트 스스로 잡을 수 있는 자가 검증 포함

### Step 4. 에이전트 파일 Edit
확정된 규칙을 `.claude/agents/<에이전트명>.md` 에 Edit한다.

### Step 5. 변경 요약 보고
수정된 섹션과 추가된 규칙을 표로 정리해서 보고한다.
